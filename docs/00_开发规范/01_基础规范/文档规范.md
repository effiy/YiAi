## 代码文档规范

### 模块文档

每个模块文件都应该有模块文档：

```python
"""
用户服务模块。

提供用户相关的业务逻辑，包括用户的增删改查、
权限验证、数据验证等功能。
"""

import logging
from typing import Optional

logger = logging.getLogger(__name__)
```

### 类文档

公共类必须有文档字符串：

```python
class UserService:
    """
    用户服务类。

    提供用户的增删改查等功能。

    Args:
        db: 数据库连接实例
        cache: 缓存实例（可选）

    Examples:
        >>> service = UserService(db, cache)
        >>> user = await service.get_by_id("123")
    """

    def __init__(self, db, cache=None):
        self.db = db
        self.cache = cache
```

### 函数文档

公共函数必须有文档字符串：

```python
async def get_user(
    user_id: str,
    include_profile: bool = False
) -> Optional[dict]:
    """
    通过 ID 获取用户信息。

    Args:
        user_id: 用户 ID，不能为空
        include_profile: 是否包含用户详细资料

    Returns:
        用户信息字典，如果用户不存在则返回 None

    Raises:
        ValueError: 当 user_id 为空时
        BusinessException: 当数据库查询失败时

    Examples:
        >>> user = await get_user("123")
        >>> print(user["name"])
        Alice

        >>> user = await get_user("123", include_profile=True)
        >>> print(user["profile"]["bio"])
        Hello, world!
    """
    if not user_id:
        raise ValueError("user_id 不能为空")

    user = await db.find_one("users", {"id": user_id})
    if not user:
        return None

    if include_profile:
        profile = await db.find_one("profiles", {"user_id": user_id})
        user["profile"] = profile

    return user
```

## Markdown 文档规范

### 标题层级

使用 ATX 标题样式，最多 6 级：

```markdown
# 一级标题

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题
```

### 列表

#### 无序列表

```markdown
- 项目一
- 项目二
- 项目三
```

#### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

#### 嵌套列表

```markdown
- 项目一
  - 子项目 1.1
  - 子项目 1.2
- 项目二
  1. 子项目 2.1
  2. 子项目 2.2
```

### 链接

```markdown
# 内联链接
[显示文本](URL)

# 引用链接
[显示文本][引用标签]

[引用标签]: URL

# 相对链接
[README](./README.md)
[API 文档](../API端点.md)
```

## 文档更新规范

### 及时更新

代码变更时同步更新文档：

- API 变更 → 更新 API 文档
- 架构变更 → 更新架构文档
- 规范变更 → 更新规范文档

### 文档审查

提交前检查文档：

- 拼写检查
- 格式检查
- 链接检查
- 内容准确性检查
