# 编码规范

本文档定义 YiAi 项目的 Python 代码编写规范。

---

## 代码风格

### 遵循 PEP 8

所有 Python 代码必须遵循 [PEP 8](https://peps.python.org/pep-0008/) 风格指南。

### 代码格式化

使用 **black** 进行代码格式化：

```bash
black src/
```

### 导入排序

使用 **isort** 进行导入排序：

```bash
isort src/
```

---

## 命名规范

### 文件命名
- 使用小写字母和下划线
- 例如：`user_service.py`, `api_routes.py`

### 类命名
- 使用大驼峰命名法（CapWords）
- 例如：`UserService`, `DatabaseManager`

### 函数和方法命名
- 使用小写字母和下划线
- 例如：`get_user()`, `process_data()`

### 常量命名
- 使用全大写字母和下划线
- 例如：`MAX_RETRIES`, `DEFAULT_TIMEOUT`

### 变量命名
- 使用小写字母和下划线
- 例如：`user_id`, `result_list`

### 私有成员
- 使用单下划线前缀
- 例如：`_internal_method()`, `_private_var`

## 类型注解

所有函数必须添加类型注解：

```python
# 正确示例
from typing import List, Optional, Dict, Any

def get_user(user_id: str) -> Optional[Dict[str, Any]]:
    """获取用户信息"""
    pass

async def process_items(items: List[str]) -> List[Dict[str, Any]]:
    """处理项目列表"""
    pass
```

## 文档字符串

所有公共函数、类和模块必须包含文档字符串：

```python
def calculate_total(price: float, quantity: int) -> float:
    """
    计算总价格。

    Args:
        price: 单价
        quantity: 数量

    Returns:
        总价格

    Raises:
        ValueError: 当价格或数量为负数时
    """
    if price < 0 or quantity < 0:
        raise ValueError("价格和数量必须为正数")
    return price * quantity
```

## 异步编程

### 优先使用异步

所有 I/O 操作必须使用异步方式：

```python
# 正确示例
async def fetch_data(url: str) -> str:
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()

# 避免示例
def fetch_data(url: str) -> str:
    response = requests.get(url)
    return response.text
```

### 异步函数命名

异步函数不需要特殊命名，但在文档中要注明是异步函数。

## 错误处理

### 使用自定义异常

优先使用项目定义的业务异常：

```python
from core.exceptions import BusinessException
from core.error_codes import ErrorCode

async def process_request(data: dict):
    if not data.get("id"):
        raise BusinessException(ErrorCode.INVALID_PARAMS, message="缺少必需的 ID 参数")
```

### 异常捕获

只捕获预期的异常，避免空的 except：

```python
# 正确示例
try:
    result = await risky_operation()
except SpecificError as e:
    logger.error(f"操作失败: {e}")
    raise BusinessException(ErrorCode.INTERNAL_ERROR) from e

# 避免示例
try:
    result = await risky_operation()
except:
    pass
```

## 日志规范

### 使用项目日志器

使用项目统一的日志器：

```python
import logging

logger = logging.getLogger(__name__)

async def process_task(task_id: str):
    logger.info(f"开始处理任务: {task_id}")
    try:
        # 处理逻辑
        logger.debug(f"任务处理完成: {task_id}")
    except Exception as e:
        logger.error(f"任务处理失败: {task_id}, 错误: {e}", exc_info=True)
        raise
```

### 日志级别

- **DEBUG** - 调试信息，开发环境使用
- **INFO** - 一般信息，正常流程记录
- **WARNING** - 警告信息，不影响流程但需要注意
- **ERROR** - 错误信息，需要处理的错误

## 代码组织

### 函数长度

函数不宜过长，一般不超过 50 行。过长的函数应该拆分成多个小函数。

### 模块组织

相关的功能应该放在同一个模块中，模块大小要适中。
