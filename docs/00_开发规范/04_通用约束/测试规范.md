# 测试规范

本文档定义 YiAi 项目的测试编写规范。

---

## 测试类型

### 单元测试

测试单个函数或类：

```python
# tests/services/test_user_service.py
import pytest

@pytest.mark.asyncio
async def test_get_user():
    """测试获取用户"""
    service = UserService()
    user = await service.get_by_id("test-id")
    assert user is not None
    assert user["id"] == "test-id"
```

### 集成测试

测试多个组件的协作：

```python
# tests/api/test_users.py
from fastapi.testclient import TestClient
from src.main import app

client = TestClient(app)

def test_create_user():
    """测试创建用户 API"""
    response = client.post(
        "/users",
        json={
            "name": "Test User",
            "email": "test@example.com"
        }
    )
    assert response.status_code == 201
    data = response.json()
    assert data["code"] == 0
```

### E2E 测试

端到端测试整个业务流程：

```python
# tests/e2e/test_user_flow.py
@pytest.mark.asyncio
async def test_complete_user_flow():
    """测试完整的用户流程"""
    # 1. 创建用户
    user = await user_service.create({"name": "Test", "email": "test@example.com"})

    # 2. 获取用户
    retrieved = await user_service.get_by_id(user["id"])
    assert retrieved["name"] == "Test"

    # 3. 更新用户
    updated = await user_service.update(user["id"], {"name": "Updated"})
    assert updated["name"] == "Updated"

    # 4. 删除用户
    result = await user_service.delete(user["id"])
    assert result is True
```

## 测试组织

### 测试目录结构

```
tests/
├── conftest.py              # pytest 配置

│   └── users.py            # 用户数据夹具
├── unit/                   # 单元测试

│   │   └── test_order_service.py
│   └── utils/
│       └── test_helpers.py
├── integration/             # 集成测试
│   ├── api/
│   │   ├── test_users.py
│   │   └── test_orders.py
│   └── db/
```

### 测试文件命名

测试文件以 `test_` 开头：

```
test_user_service.py
test_api_endpoints.py
test_database_operations.py
```

### 测试函数命名

测试函数以 `test_` 开头，描述测试内容：

```python
# 正确示例
def test_get_user_by_id():
    pass

def test_create_user_with_valid_data():
    pass

def test_create_user_with_invalid_email():
    pass

# 避免示例
def test_user1():
    pass

def test_func():
    pass
```

## pytest 规范

### 使用 pytest

使用 pytest 作为测试框架：

```python
import pytest

def test_example():
    assert 1 + 1 == 2
```

### 标记测试

使用标记分类测试：

```python
import pytest

@pytest.mark.unit
def test_unit_test():
    pass

@pytest.mark.integration
def test_integration_test():
    pass

@pytest.mark.asyncio
async def test_async_function():
    pass

@pytest.mark.slow
def test_slow_test():
    pass
```

### 运行测试

```bash
# 运行所有测试
pytest

# 运行特定文件
pytest tests/unit/services/test_user_service.py

# 运行特定标记的测试
pytest -m unit
pytest -m "not slow"

# 显示详细输出
pytest -v

# 显示打印输出
pytest -s

# 生成覆盖率报告
pytest --cov=src --cov-report=html
```

## 测试夹具

### 使用夹具

使用 pytest fixtures 复用测试代码：

```python
# tests/conftest.py
import pytest
from core.database import db

@pytest.fixture
async def test_db():
    """测试数据库夹具"""
    await db.initialize()
    yield db
    # 清理
    await db.db["test_users"].delete_many({})

@pytest.fixture
def sample_user():
    """示例用户数据"""
    return {
        "id": "test-001",
        "name": "Test User",
        "email": "test@example.com"
    }
```

### 使用夹具

```python
# tests/services/test_user_service.py
@pytest.mark.asyncio
async def test_create_user(test_db, sample_user):
    """测试创建用户"""
    service = UserService(test_db)
    user = await service.create(sample_user)
    assert user["id"] == sample_user["id"]
```

## Mock 规范

### 使用 unittest.mock

使用 mock 隔离外部依赖：

```python
from unittest.mock import Mock, patch, AsyncMock

@pytest.mark.asyncio
async def test_user_service_with_mock():
    """使用 mock 测试用户服务"""
    # Mock 数据库
    mock_db = Mock()
    mock_db.find_one = AsyncMock(return_value={"id": "123", "name": "Test"})

    service = UserService(mock_db)
    user = await service.get_by_id("123")

    assert user["name"] == "Test"
    mock_db.find_one.assert_called_once_with("users", {"id": "123"})
```

### 使用 patch

```python
@pytest.mark.asyncio
@patch("services.user_service.send_email")
async def test_user_registration(mock_send_email):
    """测试用户注册"""
    mock_send_email.return_value = True

    await user_service.register({"email": "test@example.com"})

    mock_send_email.assert_called_once_with(
        "test@example.com",
        "欢迎注册"
    )
```

## 断言规范

### 使用 assert

使用 pytest 的断言：

```python
# 基本断言
assert value == expected
assert value is not None
assert value is True

# 集合断言
assert item in list
assert key in dict

# 异常断言
with pytest.raises(ValueError):
    function_that_raises()

# 异常信息断言
with pytest.raises(ValueError, match="无效的参数"):
    function_that_raises()
```

### 近似比较

使用 pytest.approx：

```python
import pytest

def test_floating_point():
    """浮点数比较"""
    assert 0.1 + 0.2 == pytest.approx(0.3)
```

## 测试数据管理

### 测试数据文件

使用 JSON/YAML 文件存储测试数据：

```
tests/fixtures/data/
├── users.json
├── orders.json
└── products.yaml
```

### 加载测试数据

```python
import json
from pathlib import Path

def load_test_data(filename: str) -> dict:
    """加载测试数据"""
    path = Path(__file__).parent / "fixtures" / "data" / filename
    with open(path, encoding="utf-8") as f:
        return json.load(f)
```

## 测试覆盖率

### 覆盖率目标

- 核心业务逻辑：> 80%
- 工具函数：> 90%
- API 端点：> 70%

```bash

pytest --cov=src --cov-report=term --cov-report=html

# 只检查核心模块
pytest --cov=src/services --cov=src/api
```

## 测试最佳实践

### 测试独立性

测试之间应该相互独立：

```python
# 正确示例
@pytest.fixture(autouse=True)
async def clean_database():
    """每个测试前清理数据库"""
    await db.db["test_users"].delete_many({})
    yield
    await db.db["test_users"].delete_many({})
```

### 测试描述性

测试应该描述行为，而不是实现：

```python
# 正确示例
def test_user_cannot_register_with_duplicate_email():
    pass

# 避免示例
def test_user_registration_2():
    pass
```

### 每个测试一个断言

每个测试应该只验证一个行为：

```python
# 正确示例
def test_create_user_returns_correct_id():
    user = create_user({"id": "123"})
    assert user["id"] == "123"

def test_create_user_sets_created_at():
    user = create_user({"id": "123"})
    assert "created_at" in user

# 避免示例
def test_create_user():
    user = create_user({"id": "123"})
    assert user["id"] == "123"
    assert "created_at" in user
    assert user["status"] == "active"
```
