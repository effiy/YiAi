## 安全约束

### 输入验证

所有外部输入必须进行验证：

```python
from pydantic import BaseModel, Field, validator

class UserRequest(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    email: str = Field(...)

    @validator("email")
    def validate_email(cls, v):
        if not re.match(r"^[\w\.-]+@[\w\.-]+\.\w+$", v):
            raise ValueError("无效的邮箱格式")
        return v
```

### SQL 注入防护

使用参数化查询，避免字符串拼接：

```python
# 正确示例
async def get_user(name: str):
    return await db.find_one("users", {"name": name})

# 避免示例
async def get_user(name: str):
    # 不要这样做！存在注入风险
    return await db.find_one("users", {"name": {"$eq": name}})
```

### 路径遍历防护

验证文件路径，防止路径遍历攻击：

```python
import os
from core.exceptions import BusinessException

def safe_path(base_dir: str, filename: str) -> str:
    """安全的路径拼接"""
    path = os.path.normpath(os.path.join(base_dir, filename))
    if not path.startswith(os.path.normpath(base_dir) + os.sep):
        raise BusinessException(ErrorCode.INVALID_PARAMS, "非法路径")
    return path
```

### XSS 防护

输出用户输入时进行转义：

```python
import html

def escape_html(content: str) -> str:
    """转义 HTML 内容"""
    return html.escape(content)
```

## 性能约束

### 查询优化

避免 N+1 查询问题：

```python
# 正确示例 - 使用批量查询
async def get_users_with_orders(user_ids: List[str]):
    users = await db.find_many("users", {"id": {"$in": user_ids}})
    orders = await db.find_many("orders", {"user_id": {"$in": user_ids}})
    # 组合数据...

# 避免示例 - N+1 查询
async def get_users_with_orders(user_ids: List[str]):
    users = []
    for user_id in user_ids:
        user = await db.find_one("users", {"id": user_id})
        user["orders"] = await db.find_many("orders", {"user_id": user_id})
        users.append(user)
```

### 分页查询

大数据量必须使用分页：

```python
async def list_users(page: int = 1, size: int = 20):
    """用户列表 - 分页查询"""
    skip = (page - 1) * size
    users = await db.find_many(
        "users",
        limit=size,
        skip=skip
    )
    total = await db.count("users")
    return {"items": users, "total": total, "page": page, "size": size}
```

### 异步 I/O

所有 I/O 操作必须使用异步方式：

```python
# 正确示例
async def fetch_data(url: str):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()

# 避免示例
def fetch_data(url: str):
    response = requests.get(url)
    return response.text
```

## 代码质量约束

### 代码复杂度

函数圈复杂度不应超过 10：

```python
# 正确示例 - 低复杂度
async def process_order(order: dict):
    if not validate_order(order):
        return error("无效订单")

    await reserve_inventory(order)
    await charge_payment(order)
    await create_order_record(order)
    await send_notification(order)
    return success("订单处理完成")

# 避免示例 - 高复杂度
async def process_order(order: dict):
    if order.get("type") == "A":
        if order.get("status") == "pending":
            if order.get("amount") > 100:
                # ... 多层嵌套
    elif order.get("type") == "B":
        # ... 更多分支
```

函数不应超过 50 行：

```python
# 正确示例 - 拆分长函数
async def process_order(order: dict):
    await validate_order(order)
    await process_payment(order)
    await update_inventory(order)

async def validate_order(order: dict):

# 避免示例 - 长函数
async def process_order(order: dict):
    # 100+ 行的逻辑...
```

## 可维护性约束

### 注释规范

复杂逻辑必须添加注释：

```python
async def calculate_ranking(users: List[dict]) -> List[dict]:
    """
    计算用户排名。

    算法说明：
    1. 按积分从高到低排序
    2. 积分相同则按创建时间排序（先创建的靠前）
    3. 添加排名序号，积分相同排名相同
    """
    # 先按积分降序，再按创建时间升序
    sorted_users = sorted(
        users,
        key=lambda u: (-u["score"], u["created_at"])
    )

    # 计算排名
    current_rank = 1
    prev_score = None
    for i, user in enumerate(sorted_users):
        if i > 0 and user["score"] != prev_score:
            current_rank = i + 1
        user["rank"] = current_rank
        prev_score = user["score"]

    return sorted_users
```

### TODO 注释

临时方案必须添加 TODO 注释：

```python
async def get_data():
    """获取数据"""
    # TODO: 使用缓存优化性能，当前每次都查数据库
    return await db.find_many("data", {})
```

## 日志约束

### 敏感信息

不要在日志中记录敏感信息：

```python
# 正确示例
logger.info(f"用户登录: user_id={user_id}")

# 避免示例
logger.info(f"用户登录: user={user}, password={password}")
```

### 日志级别

合理使用日志级别：

```python
logger.debug("调试信息 - 开发环境使用")
logger.info("一般信息 - 正常流程记录")
logger.warning("警告信息 - 需要注意但不影响流程")
logger.error("错误信息 - 需要处理的错误", exc_info=True)
```
