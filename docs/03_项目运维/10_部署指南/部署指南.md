# 部署指南

## 📋 概述

本文档详细说明 YiAi 项目的部署流程，包括环境准备、依赖安装、配置管理和生产环境部署等内容。

## 🚀 快速开始

### 本地开发环境

```bash
# 1. 克隆代码
git clone <repository-url>
cd YiAi

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境
cp config.yaml.example config.yaml
# 编辑 config.yaml 配置文件

# 4. 启动服务
python main.py
```

服务将在 `http://localhost:8000` 启动，API 文档可通过 `/docs` 访问。

## ⚙️ 配置管理

### 配置文件结构

```yaml
# config.yaml
server:
  host: "0.0.0.0"
  port: 8000

mongodb:
  url: "mongodb://localhost:27017"
  db_name: "yiai"

oss:
  access_key: ""
  secret_key: ""
  bucket: ""
  endpoint: ""

rss:
  scheduler_enabled: true
  scheduler_interval: 3600

ollama:
  url: "http://localhost:11434"
  model: "llama2"

middleware:
  auth_enabled: false
  auth_token: ""

module:
  allowlist: ["*"]

static:
  base_dir: "./static"
```

### 环境变量覆盖

所有配置项都可以通过环境变量覆盖，格式为大写+下划线：

```bash
export SERVER_HOST="127.0.0.1"
export SERVER_PORT=8000
export MONGODB_URL="mongodb://localhost:27017"
export MONGODB_DB_NAME="yiai"
export MIDDLEWARE_AUTH_ENABLED="true"
export MIDDLEWARE_AUTH_TOKEN="your-secret-token"
```

## 🌐 生产环境部署

### 使用 systemd 服务

创建 `/etc/systemd/system/yiai.service`：

```ini
[Unit]
Description=YiAi FastAPI Service
After=network.target mongod.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/YiAi
Environment="PATH=/var/www/YiAi/venv/bin"
ExecStart=/var/www/YiAi/venv/bin/python main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载 systemd 配置
sudo systemctl daemon-reload

# 启用并启动服务
sudo systemctl enable yiai
sudo systemctl start yiai

# 查看状态
sudo systemctl status yiai

# 查看日志
sudo journalctl -u yiai -f
```

### 使用 Docker

创建 `Dockerfile`：

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["python", "main.py"]
```

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:6
    restart: always
    volumes:
      - mongodb_data:/data/db
    environment:
      MONGO_INITDB_DATABASE: yiai

  yiai:
    build: .
    restart: always
    ports:
      - "8000:8000"
    depends_on:
      - mongodb
    environment:
      - MONGODB_URL=mongodb://mongodb:27017
      - MONGODB_DB_NAME=yiai
    volumes:
      - ./config.yaml:/app/config.yaml
      - ./static:/app/static

volumes:
  mongodb_data:
```

启动容器：

```bash
docker-compose up -d
```

### 使用 Nginx 反向代理

创建 `/etc/nginx/sites-available/yiai`：

```nginx
upstream yiai_backend {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 50M;

    location / {
        proxy_pass http://yiai_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 支持
        proxy_buffering off;
        proxy_cache off;
        proxy_set_header Connection '';
        proxy_http_version 1.1;
        chunked_transfer_encoding off;
    }

    location /static {
        alias /var/www/YiAi/static;
        expires 30d;
    }

    location /docs {
        proxy_pass http://yiai_backend;
        proxy_set_header Host $host;
    }
}
```

启用站点：

```bash
sudo ln -s /etc/nginx/sites-available/yiai /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 🔐 安全配置

### 启用认证

编辑 `config.yaml`：

```yaml
middleware:
  auth_enabled: true
  auth_token: "your-secure-token-here"
```

客户端请求时添加 Header：

```bash
curl -H "X-Token: your-secure-token-here" http://localhost:8000/execution
```

### HTTPS 配置

使用 Let's Encrypt 证书：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 模块白名单

限制可执行的模块：

```yaml
module:
  allowlist:
    - "services.rss"
    - "services.ai"
    - "services.storage"
```

## 🔄 升级部署

### 零停机升级（使用 systemd）

```bash
# 1. 拉取最新代码
cd /var/www/YiAi
git pull

# 2. 安装新依赖
source venv/bin/activate
pip install -r requirements.txt

# 3. 优雅重启
sudo systemctl reload yiai
# 或
sudo systemctl restart yiai
```

### 数据库迁移

如有数据结构变更，执行迁移脚本：

```bash
python scripts/migrate.py
```
