# 监控维护

## 📋 概述

本文档定义 YiAi 项目的监控指标、日常维护任务和日志管理规范，确保系统稳定运行。

## 📊 监控指标

### 应用层监控

| 指标 | 监控方法 | 正常阈值 | 告警阈值 |
|------|---------|---------|---------|
| 服务状态 | HTTP GET /docs | 200 OK | != 200 |
| 响应时间 | API 延迟统计 | < 500ms | > 2s |
| 错误率 | 5xx 响应占比 | < 1% | > 5% |
| 并发连接数 | netstat/ss | < 1000 | > 5000 |
| 请求速率 | QPS | 根据业务 | 突增/突降 |

### 系统层监控

| 指标 | 监控方法 | 正常阈值 | 告警阈值 |
|------|---------|---------|---------|
| CPU 使用率 | top/ps | < 70% | > 90% |
| 内存使用率 | free | < 80% | > 95% |
| 磁盘使用率 | df -h | < 80% | > 90% |
| 磁盘 I/O | iostat | < 80% util | > 90% util |
| 网络流量 | iftop/nload | 根据业务 | 突增/突降 |

### 数据库监控

| 指标 | 监控方法 | 正常阈值 | 告警阈值 |
|------|---------|---------|---------|
| MongoDB 连接数 | db.serverStatus().connections | < 500 | > 1000 |
| 操作延迟 | db.serverStatus().opLatencies | < 100ms | > 500ms |
| 磁盘空间 | db.stats().dataSize | 根据业务 | > 80% |
| 复制延迟 | rs.printSlaveReplicationInfo() | < 5s | > 30s |
| 锁等待 | db.serverStatus().globalLock | < 1% | > 5% |

## 📜 日志管理

### 日志位置

| 日志类型 | 路径 |
|---------|------|
| 应用日志 | `/var/log/yiai/app.log` |
| 访问日志 | `/var/log/yiai/access.log` |
| 错误日志 | `/var/log/yiai/error.log` |
| 系统日志 | `journalctl -u yiai` |

### 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG | 调试信息 | 详细的函数调用、参数值 |
| INFO | 一般信息 | 服务启动、配置加载 |
| WARNING | 警告信息 | 重试操作、非关键错误 |
| ERROR | 错误信息 | 异常捕获、请求失败 |
| CRITICAL | 严重错误 | 服务崩溃、数据丢失 |

### 日志格式

标准日志格式：

```
2026-03-22 10:30:45,123 - INFO - [main] Application started
2026-03-22 10:30:46,456 - DEBUG - [api.routes.execution] Executing module: services.ai
2026-03-22 10:30:47,789 - ERROR - [core.database] MongoDB connection failed: Connection refused
```

### 日志轮转配置

创建 `/etc/logrotate.d/yiai`：

```
/var/log/yiai/*.log {
    daily
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 www-data www-data
    sharedscripts
    postrotate
        systemctl reload yiai > /dev/null 2>&1 || true
    endscript
}
```

### 日志查询常用命令

```bash
# 查看最近 100 行日志
tail -n 100 /var/log/yiai/app.log

# 实时跟踪日志
tail -f /var/log/yiai/app.log

# 搜索错误日志
grep ERROR /var/log/yiai/app.log

# 查看特定时间段的日志
journalctl -u yiai --since "2026-03-22 09:00:00" --until "2026-03-22 10:00:00"

# 查看错误级别的日志
journalctl -u yiai -p err -n 50
```

## 🚨 告警配置

### 告警规则示例

| 告警项 | 触发条件 | 告警级别 | 通知方式 |
|--------|---------|---------|---------|
| 服务宕机 | 服务不可达 > 1分钟 | P0 | 电话、短信 |
| 高 CPU | CPU > 90% 持续 5分钟 | P1 | 邮件、钉钉 |
| 高内存 | 内存 > 95% 持续 5分钟 | P1 | 邮件、钉钉 |
| 磁盘满 | 磁盘 > 90% | P2 | 邮件 |
| 错误率高 | 5xx > 5% 持续 1分钟 | P2 | 邮件 |
| 数据库延迟 | MongoDB 延迟 > 500ms | P2 | 邮件 |

### 简单监控脚本

创建 `/usr/local/bin/yiai-monitor.sh`：

```bash
#!/bin/bash

# YiAi 监控脚本

YIAI_URL="http://localhost:8000/docs"
ALERT_EMAIL="admin@example.com"

check_service() {
    if ! curl -s -o /dev/null -w "%{http_code}" "$YIAI_URL" | grep -q "200"; then
        echo "YiAi service is down!" | mail -s "ALERT: YiAi Service Down" "$ALERT_EMAIL"
    fi
}

check_disk() {
    usage=$(df -h / | awk 'NR==2 {print $5}' | sed 's/%//')
    if [ "$usage" -gt 90 ]; then
        echo "Disk usage is ${usage}%!" | mail -s "ALERT: High Disk Usage" "$ALERT_EMAIL"
    fi
}

# 执行检查
check_service
check_disk
```

添加到 crontab：

```bash
# 每 5 分钟执行一次监控
*/5 * * * * /usr/local/bin/yiai-monitor.sh
```

## 🧹 清理任务

### 清理未使用的图片

使用内置 API：

```bash
curl -X POST http://localhost:8000/cleanup-unused-images \
  -H "X-Token: your-token"
```

### 清理旧日志

```bash
# 保留最近 30 天的日志
find /var/log/yiai -name "*.log.*" -mtime +30 -delete

# 压缩旧日志
gzip /var/log/yiai/app.log.*
```

### 清理数据库旧数据

```javascript
// 删除 90 天前的 RSS 文章
db.rss.deleteMany({ fetched_at: { $lt: new Date(Date.now() - 90 * 24 * 60 * 60 * 1000) } });

// 删除 30 天前的聊天记录（按需）
db.chat_records.deleteMany({ created_at: { $lt: new Date(Date.now() - 30 * 24 * 60 * 60 * 1000) } });
```
