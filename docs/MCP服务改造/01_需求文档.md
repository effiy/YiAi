## 功能概述

YiAi 当前是一个基于 FastAPI 的 REST API 服务，提供文件管理、动态模块执行、RSS 管理、AI 聊天等功能。本功能的目标是将 YiAi 的 FastAPI 端点通过 `fastapi-mcp` 自动暴露为 MCP（Model Context Protocol）工具，使 Claude 等 AI 客户端能够通过 MCP 协议直接调用 YiAi 的全部 API 能力，无需编写自定义 HTTP 调用代码。范围覆盖 YiAi 现有全部 API 端点的 MCP 化；非目标包括修改业务逻辑、新增 API 功能或移除现有 REST 接口。

**核心价值**
- 🎯 AI 客户端可直接发现和使用 YiAi 的全部 API 能力，无需手动封装 HTTP 请求
- ⚡ 零配置自动转换，基于现有 FastAPI 路由和 Pydantic 模型自动生成 MCP 工具描述
- 📖 保留现有 API 文档和请求/响应模式，确保 MCP 工具与 REST API 语义一致

## 功能详情

> 只写"要做什么/不做什么"，不写实现细节。

### MCP 服务器集成

- **功能说明**：在 YiAi 的 FastAPI 应用中集成 `fastapi-mcp`，将现有 REST API 端点自动转换为 MCP 工具。MCP 服务器通过 SSE 传输协议与客户端通信，默认挂载到 `/mcp` 路径。
- **边界与异常**：不修改现有路由处理逻辑；静态文件服务（`/static`）不暴露为 MCP 工具；OPTIONS 预检请求不暴露为工具。
- **价值/动机**：使 AI 客户端能够直接发现和调用 YiAi 功能，降低集成成本。

### 工具命名与描述

- **功能说明**：MCP 工具名称使用 FastAPI 路由的 `operation_id`，描述保留原始路由的 docstring 和 Pydantic 模型的 schema 信息。对于未设置 `operation_id` 的路由，FastAPI 会自动生成名称，但建议为所有路由显式设置 `operation_id` 以获得清晰的工具名。
- **边界与异常**：自动生成的 operation_id 可能晦涩（如 `read_user_users__user_id__get`）；需要评估现有路由的命名质量。
- **价值/动机**：确保 AI 客户端看到的工具名称直观、描述准确，提升可用性。

### 端点过滤控制

- **功能说明**：支持通过 `include_operations`、`exclude_operations`、`include_tags`、`exclude_tags` 参数控制哪些端点暴露为 MCP 工具。例如，可以排除 `/cleanup-unused-images` 等维护端点。
- **边界与异常**：不能同时使用 include 和 exclude 同类过滤器；组合过滤时采用贪婪匹配策略。
- **价值/动机**：提供安全边界，防止敏感或内部端点被外部 AI 客户端访问。

### 多客户端连接支持

- **功能说明**：MCP 服务器支持通过 SSE 直接连接（Cursor 等），也支持通过 `mcp-proxy` 转为 stdio 连接（Claude Desktop 等）。
- **边界与异常**：mcp-proxy 需要额外安装；不同客户端的配置方式不同。
- **价值/动机**：覆盖主流 AI 客户端的连接方式，最大化兼容性。

## 使用场景示例

#### 📋 场景一：Cursor 中直接调用 YiAi 功能

> **背景**：开发者在 Cursor 中编写代码，需要上传一个文件到 YiAi。
>
> **操作**：在 Cursor 设置中配置 YiAi MCP 服务器（`http://localhost:8000/mcp`），然后在对话中请求"帮我把这个文件上传到 YiAi"。
>
> **结果**：Cursor 自动发现并调用 `upload_file` MCP 工具，完成文件上传。

#### 📋 场景二：Claude Code 中执行动态模块

> **背景**：开发者需要在 Claude Code 中执行一个 YiAi 白名单中的模块方法。
>
> **操作**：配置 YiAi MCP 服务器后，在 Claude Code 对话中请求"执行 services.ai.chat_service 的 send_message 方法"。
>
> **结果**：Claude Code 通过 MCP 调用 `execute_module_via_post` 工具，传入正确的参数，获取执行结果。

#### 📋 场景三：运维人员控制暴露范围

> **背景**：生产环境中，运维人员不希望维护端点（如 cleanup-unused-images）暴露给 AI 客户端。
>
> **操作**：在 MCP 服务器配置中设置 `exclude_tags=["Maintenance"]`。
>
> **结果**：AI 客户端只能看到 Upload、Execution、WeWork 等标签的端点，维护端点被隐藏。
