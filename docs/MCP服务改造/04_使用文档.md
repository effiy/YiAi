## 功能介绍

通过 `fastapi-mcp` 库，YiAi 的 FastAPI 端点可以自动暴露为 MCP（Model Context Protocol）工具。这意味着 Claude、Cursor 等 AI 客户端可以直接发现并调用 YiAi 的文件上传、动态模块执行、企业微信消息发送等功能，无需手动编写 HTTP 请求代码。

**核心特性**
- 🎯 **零配置自动转换**：现有 FastAPI 路由自动变为 MCP 工具
- ⚡ **ASGI 内部通信**：MCP 调用直接走 ASGI，无需额外 HTTP 开销
- 🔧 **安全可控**：支持按标签或操作 ID 过滤暴露的端点

**适用人群**
- 希望在 AI 客户端中直接使用 YiAi 能力的开发者
- 需要为团队提供 AI 集成能力的运维人员

## 快速开始

### 前置条件

- [ ] YiAi 服务已部署并运行
- [ ] AI 客户端支持 MCP（Cursor、Claude Desktop、Claude Code 等）
- [ ] 网络可达（YiAi 服务地址可访问）

### 30 秒上手

1. **确保 YiAi MCP 已启用**
   ```bash
   python main.py
   ```
   启动日志中应包含 MCP 服务器挂载信息。

2. **获取 MCP 端点地址**
   - 本地开发：`http://localhost:8000/mcp`
   - 生产环境：`https://api.effiy.cn/mcp`

3. **在 Cursor 中配置**
   - 打开 Cursor 设置 → MCP
   - 添加 SSE 类型服务器，URL 填 `http://localhost:8000/mcp`

4. **开始使用**
   - 在 Cursor 对话中输入："帮我把这个文件上传到 YiAi"
   - Cursor 自动发现并调用 `upload_file` 工具

## 操作场景

### 场景一：Cursor 中使用 YiAi MCP

**适用情况**：在 Cursor 编辑器中需要通过 AI 调用 YiAi 功能。

**操作步骤**：
1. 启动 YiAi 服务：`python main.py`
2. 打开 Cursor，进入设置 → MCP
3. 点击「添加服务器」，选择 SSE 类型
4. 填写 URL：`http://localhost:8000/mcp`
5. 保存后 Cursor 自动发现并列出工具
6. 在对话中输入自然语言指令，如"读取 static/test.md 文件"

**预期结果**：Cursor 自动调用 `read_file` MCP 工具，返回文件内容。

**注意事项**：
- ✅ 确保 YiAi 服务已启动且 `/mcp` 可访问
- ✅ Cursor 需要能访问 YiAi 服务的网络地址
- ❌ 如果启用了认证，需要确保 MCP 端点在白名单中

### 场景二：Claude Desktop 中使用 YiAi MCP

**适用情况**：Claude Desktop 不支持 SSE，需要通过 mcp-proxy 使用 stdio 连接。

**操作步骤**：
1. 安装 mcp-proxy：
   ```bash
   uv tool install mcp-proxy
   # 或
   pip install mcp-proxy
   ```

2. 找到 mcp-proxy 可执行文件路径：
   ```bash
   which mcp-proxy
   ```

3. 编辑 Claude Desktop 配置文件：
   - macOS：`~/Library/Application Support/Claude/claude_desktop_config.json`
   - Windows：`%APPDATA%/Claude/claude_desktop_config.json`

4. 添加配置：
   ```json
   {
     "mcpServers": {
       "yiai": {
         "command": "/Full/Path/To/mcp-proxy",
         "args": ["http://localhost:8000/mcp"]
       }
     }
   }
   ```

5. 重启 Claude Desktop，自动发现工具

**预期结果**：Claude Desktop 中可以使用 YiAi 的所有 MCP 工具。

**注意事项**：
- ✅ macOS 上需要使用 mcp-proxy 的完整路径
- ✅ Claude Desktop 重启后才能加载新配置
- ❌ Windows 上路径格式需使用反斜杠或转义

### 场景三：Claude Code 中使用 YiAi MCP

**适用情况**：在终端使用 Claude Code 时调用 YiAi 功能。

**操作步骤**：
1. 确保 YiAi 服务运行中
2. 在 Claude Code 的 MCP 配置中添加 YiAi：
   ```json
   {
     "mcpServers": {
       "yiai": {
         "url": "http://localhost:8000/mcp",
         "transport": "sse"
       }
     }
   }
   ```
3. 启动 Claude Code，AI 自动发现 YiAi 工具
4. 输入指令如："执行 services.rss.feed_service 的 parse_feed 方法"

**预期结果**：Claude Code 通过 MCP 调用 `execute_module_post`，传入正确的模块路径和方法名。

**注意事项**：
- ✅ Claude Code 支持 SSE 直连，无需 mcp-proxy
- ✅ 动态模块执行功能特别适合 AI 客户端使用
- ❌ 执行白名单模块需确保模块在 `module.allowlist` 中

### 场景四：控制暴露的端点

**适用情况**：生产环境中需要限制 AI 客户端可访问的端点。

**操作步骤**：
1. 编辑 `src/main.py`
2. 修改 FastApiMCP 配置：
   ```python
   mcp = FastApiMCP(
       app,
       name="YiAi MCP",
       exclude_tags=["Maintenance"],
       # 或仅包含特定标签
       # include_tags=["Upload", "Execution"]
   )
   ```
3. 重启 YiAi 服务
4. AI 客户端重新连接，工具列表自动更新

**预期结果**：被排除的端点不再出现在 AI 客户端的工具列表中。

**注意事项**：
- ✅ 过滤配置仅影响 MCP 工具列表，不影响原有 REST API
- ✅ 可以组合使用 include_operations 和 include_tags
- ❌ 不能同时使用 include 和 exclude 同类过滤器

## 常见问题

### 💡 基础问题

**Q1：MCP 是什么？**
> MCP（Model Context Protocol）是 Anthropic 推出的开放协议，用于 AI 模型与外部工具和数据源之间的标准化通信。通过 MCP，AI 客户端可以自动发现和使用外部工具。

**Q2：fastapi-mcp 会修改我的 API 吗？**
> 不会。fastapi-mcp 仅添加一个 `/mcp` 端点，不会修改任何现有路由的行为。原有 REST API 完全保持不变。

**Q3：所有 API 端点都会暴露为 MCP 工具吗？**
> 默认情况下是的，但你可以通过 `include_tags`、`exclude_tags` 等参数控制暴露范围。静态文件服务（`/static`）不会被暴露。

### ⚙️ 高级问题

**Q4：如何自定义 MCP 工具的名称？**
> 在 FastAPI 路由装饰器中添加 `operation_id` 参数：
> ```python
> @router.post("/upload", operation_id="upload_file")
> ```
> 如果不设置，FastAPI 会自动生成名称，但可能比较晦涩。

**Q5：MCP 调用和直接 HTTP 调用有什么区别？**
> MCP 调用通过 ASGI 内部通信直接转发到 FastAPI 路由，不经过网络层。性能更高，且保留了 Pydantic 模型的完整 schema 信息。

### 🔧 故障排查

**Q6：AI 客户端无法连接到 `/mcp`？**
> - 检查 YiAi 服务是否已启动
> - 检查 `/mcp` 路径是否正确
> - 检查认证中间件是否拦截了 MCP 请求（需要将 `/mcp` 加入白名单）
> - 检查防火墙或网络代理是否阻止了 SSE 连接

**Q7：工具列表为空或缺少某些端点？**
> - 检查路由是否设置了 `operation_id`
> - 检查过滤配置是否排除了该端点
> - 检查路由是否在 MCP 服务器创建后才注册（需要调用 `mcp.setup_server()` 刷新）

**Q8：工具调用返回 401 未授权？**
> - 认证中间件可能拦截了 MCP 请求
> - 在 `src/core/middleware.py` 中将 `/mcp` 加入白名单
> - 或在 MCP 客户端配置中传递认证令牌

## 技巧与提示

### 💡 实用技巧

1. **工具命名规范**：为所有路由显式设置 `operation_id`，使用动词+名词格式（如 `upload_file`、`execute_module`）
2. **标签管理**：合理使用 FastAPI 的 `tags` 参数，便于后续通过标签批量控制暴露范围
3. **本地测试**：使用 `curl http://localhost:8000/mcp` 测试 SSE 端点是否可访问

### 📚 最佳实践

1. **开发环境**：启用全部端点，便于调试和测试
2. **生产环境**：通过 `exclude_tags` 排除 Maintenance 等内部端点，最小化攻击面
3. **文档同步**：修改路由后执行 `mcp.setup_server()` 刷新 MCP 工具列表
