# 日志规范

本文档定义 YiAi 项目的日志规范。

---

## 日志系统

### 日志配置

项目使用 Python 标准库 `logging` 模块，并在 `src/core/logger.py` 中统一配置。

### 日志输出

日志同时输出到两个位置：
1. **控制台** - 标准输出
2. **文件** - `logs/app.log`，按大小轮转

### 日志轮转

- 单个日志文件最大 10MB
- 最多保留 5 个备份文件
- 文件编码为 UTF-8

---

## 日志级别

### 级别定义

| 级别 | 数值 | 使用场景 |
|------|------|---------|
| DEBUG | 10 | 调试信息，开发环境使用 |
| INFO | 20 | 一般信息，正常流程记录 |
| WARNING | 30 | 警告信息，需要注意但不影响流程 |
| ERROR | 40 | 错误信息，需要处理的错误 |
| CRITICAL | 50 | 严重错误，系统无法继续运行 |

### 级别配置

通过 `config.yaml` 配置：

```yaml
logging:
  level: "INFO"
  format: "%(asctime)s - %(name)s - %(levelname)s - %(message)s"
  datefmt: "%Y-%m-%d %H:%M:%S"
```

## 日志使用规范

### 获取日志器

每个模块使用自己的日志器：

```python
# 正确示例
import logging

logger = logging.getLogger(__name__)

async def process_data():
    logger.info("开始处理数据")
    try:
        # 处理逻辑
        logger.debug(f"处理结果: {result}")
    except Exception as e:
        logger.error(f"处理失败: {e}", exc_info=True)
```

### 日志消息格式

日志消息应该清晰、准确、有意义：

```python
# 正确示例
logger.info("用户登录成功: user_id=12345")
logger.warning("配置项缺失，使用默认值: oss.bucket")
logger.error("数据库连接失败: url=mongodb://..., 错误=连接超时")

# 避免示例
logger.info("登录成功")
logger.warning("配置有问题")
logger.error("出错了")
```

### 日志参数

使用字符串格式化，避免直接拼接：

```python
# 正确示例
logger.info("处理完成: count=%d, duration=%.2fs", count, duration)
logger.info(f"处理完成: count={count}, duration={duration:.2f}s")

# 避免示例
logger.info("处理完成: count=" + str(count) + ", duration=" + str(duration))
```

## 各场景日志规范

### 应用启动和关闭

```python
# src/main.py
logger = logging.getLogger(__name__)

@asynccontextmanager
async def lifespan(app: FastAPI):
    logger.info("应用启动中...")

    # 初始化数据库
    if settings.startup_init_database:
        await db.initialize()
        logger.info("数据库连接已初始化")

    # 启动 RSS 调度器
    if settings.startup_init_rss_system and settings.rss_scheduler_enabled:
        scheduler.start()
        logger.info("RSS 调度器已启动")

    logger.info("应用启动完成")

    yield

    logger.info("应用关闭中...")

    if settings.rss_scheduler_enabled:
        scheduler.stop()
        logger.info("RSS 调度器已停止")

    if db.client:
        db.client.close()
        logger.info("数据库连接已关闭")

    logger.info("应用已关闭")
```

### API 请求日志

```python
# 中间件或端点中
logger = logging.getLogger(__name__)

@router.post("/execution")
async def execute_module_via_post(request: ExecuteRequest):
    logger.info(f"执行模块: {request.module_name}, 方法: {request.method_name}")
    logger.debug(f"参数: {request.parameters}")

    try:
        result = await execute_module(...)
        logger.info(f"模块执行完成: {request.module_name}.{request.method_name}")
        return success(data=result)
    except Exception as e:
        logger.error(f"模块执行失败: {e}", exc_info=True)
        raise
```

### 数据库操作日志

```python
logger = logging.getLogger(__name__)

async def find_one(collection_name: str, query: dict):
    logger.debug(f"查询 {collection_name}: {query}")
    result = await db.db[collection_name].find_one(query)
    logger.debug(f"查询结果: {'找到' if result else '未找到'}")
    return result

async def insert_one(collection_name: str, document: dict):
    logger.info(f"插入 {collection_name}: id={document.get('id')}")
    result = await db.db[collection_name].insert_one(document)
    logger.debug(f"插入完成，inserted_id={result.inserted_id}")
    return result.inserted_id
```

### 业务逻辑日志

```python
logger = logging.getLogger(__name__)

async def upload_image_to_oss(request: ImageUploadToOssRequest):
    filename = _normalize_no_spaces(request.filename)
    logger.info(f"开始上传图片: {filename}")

    # 尝试 OSS 上传
    try:
        result = await upload_bytes_to_oss(content, filename, directory=directory)
        logger.info(f"图片上传成功 (OSS): {filename}")
        return success(data=result)
    except Exception as e:
        logger.warning(f"OSS 上传失败，fallback 到本地存储: {e}")
        result = await _upload_to_local_storage(content, filename, directory)
        logger.info(f"图片上传成功 (本地): {filename}")
        return success(data=result)
```

### 错误日志

```python
logger = logging.getLogger(__name__)

try:
    # 业务逻辑
    await risky_operation()
except BusinessException as e:
    # 业务异常，记录警告
    logger.warning(f"业务异常: code={e.code}, message={e.message}")
    raise
except Exception as e:
    # 系统异常，记录错误和堆栈
    logger.error(f"系统异常: {e}", exc_info=True)
    raise BusinessException(ErrorCode.INTERNAL_ERROR) from e
```

## 敏感信息保护

### 禁止记录的信息

绝对不要在日志中记录以下信息：

```python
# 错误示例 - 不要这样做！
logger.info(f"用户登录: username={username}, password={password}")
logger.info(f"API 调用: token={api_token}")
logger.info(f"数据库连接: {mongodb_url_with_password}")
logger.info(f"用户数据: {user_ssn}, {user_credit_card}")

# 正确示例
logger.info(f"用户登录: username={username}")
logger.info("API 调用成功")
logger.info(f"数据库连接: {mongodb_url_without_password}")
logger.info(f"用户数据已保存: user_id={user_id}")
```

### 敏感字段过滤

如果必须记录对象，先过滤敏感字段：

```python
def safe_user_dict(user: dict) -> dict:
    """返回安全的用户字典，过滤敏感字段"""
    safe = user.copy()
    safe.pop("password", None)
    safe.pop("token", None)
    safe.pop("ssn", None)
    return safe

logger.info(f"用户数据: {safe_user_dict(user)}")
```

## 第三方库日志

### Uvicorn 日志

项目默认调整了 Uvicorn 的日志级别：

```python
logging.getLogger("uvicorn.access").setLevel(logging.WARNING)

```

## 日志最佳实践

### 1. 适当的日志级别

```python
# DEBUG - 详细的调试信息
logger.debug(f"查询参数: {query}, 选项: {options}")
logger.debug(f"缓存命中: key={key}")

# INFO - 重要的业务事件
logger.info(f"用户创建成功: user_id={user_id}")
logger.info(f"任务完成: task_id={task_id}, duration={duration}s")

# WARNING - 不影响流程但需要注意
logger.warning(f"配置项缺失，使用默认值: {key}={default_value}")
logger.warning(f"重试中: attempt={attempt}/{max_attempts}, error={error}")

# ERROR - 需要处理的错误
logger.error(f"数据库操作失败: {e}", exc_info=True)
logger.error(f"外部服务调用失败: {service_name}, 错误={e}")
```

### 2. 记录上下文信息

```python
# 正确示例 - 包含足够的上下文
logger.info(f"处理订单: order_id={order_id}, user_id={user_id}, amount={amount}")

# 避免示例 - 信息不足
logger.info("处理订单")
```

### 3. 异常时记录堆栈

```python
# 正确示例 - 记录异常堆栈
try:
    await risky_operation()
except Exception as e:
    logger.error(f"操作失败: {e}", exc_info=True)
    raise

# 避免示例 - 只记录消息
try:
    await risky_operation()
except Exception as e:
    logger.error(f"操作失败: {e}")
    raise
```

### 4. 避免过度日志

```python
# 避免示例 - 循环中过度日志
for item in large_list:
    logger.debug(f"处理项目: {item}")  # 列表很大时会有问题

# 正确示例 - 批量记录
logger.debug(f"开始处理 {len(large_list)} 个项目")
# ... 处理逻辑 ...
logger.debug(f"处理完成，成功: {success_count}, 失败: {fail_count}")
```
