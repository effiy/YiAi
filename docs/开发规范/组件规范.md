# 组件规范

本文档定义 YiAi 项目的组件设计和开发规范。

---

## 组件设计原则

### 单一职责

每个组件应该只有一个职责，只做一件事。

```python
# 正确示例
class UserService:
    """用户服务 - 只负责用户相关操作"""
    async def get_user(self, user_id: str):
        pass

    async def create_user(self, user_data: dict):
        pass

# 避免示例
class UserAndOrderService:
    """用户和订单服务 - 职责过多"""
    async def get_user(self, user_id: str):
        pass

    async def create_order(self, order_data: dict):
        pass
```

### 高内聚低耦合

组件内部应该高内聚，组件之间应该低耦合。

### 依赖注入

优先使用依赖注入，避免硬编码依赖：

```python
# 正确示例
class OrderService:
    def __init__(self, user_service: UserService, db: Database):
        self.user_service = user_service
        self.db = db

# 避免示例
class OrderService:
    def __init__(self):
        self.user_service = UserService()  # 硬编码依赖
        self.db = Database()                # 硬编码依赖
```

## 服务组件规范

### 服务类命名

使用 `{业务}Service` 格式：

```python
class UserService:
    """用户服务"""
    pass

class OrderService:
    """订单服务"""
    pass
```

### 服务方法

服务方法应该是异步的（如果涉及 I/O）：

```python
class UserService:
    async def get_by_id(self, user_id: str) -> Optional[User]:
        """通过 ID 获取用户"""
        pass

    async def create(self, data: dict) -> User:
        """创建用户"""
        pass

    async def update(self, user_id: str, data: dict) -> User:
        """更新用户"""
        pass

    async def delete(self, user_id: str) -> bool:
        """删除用户"""
        pass
```

## API 端点组件规范

### 路由定义

使用 FastAPI 的 APIRouter：

```python
from fastapi import APIRouter
from core.response import success

router = APIRouter()

@router.get("/users/{user_id}")
async def get_user(user_id: str):
    """获取用户详情"""
    user = await user_service.get_by_id(user_id)
    return success(data=user)
```

### 请求验证

使用 Pydantic 模型进行请求验证：

```python
from pydantic import BaseModel, Field

class UserCreateRequest(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    email: str = Field(..., pattern=r"^[\w\.-]+@[\w\.-]+\.\w+$")
    age: int = Field(..., ge=0, le=150)
```

### 响应格式

使用统一的响应格式：

```python
from core.response import success, error

# 成功响应
return success(data=user_data, message="操作成功")

# 错误响应
return error(code=ErrorCode.NOT_FOUND, message="用户不存在")
```

## 工具组件规范

### 工具函数

纯工具函数应该放在 `core/` 或单独的 `utils/` 模块中：

```python
# 正确示例 - 纯函数
def format_datetime(dt: datetime) -> str:
    """格式化日期时间"""
    return dt.strftime("%Y-%m-%d %H:%M:%S")

def generate_id() -> str:
    """生成唯一 ID"""
    return str(uuid.uuid4())
```

### 工具类

工具类应该是无状态的：

```python
class DateUtils:
    """日期工具类"""

    @staticmethod
    def format_date(dt: datetime) -> str:
        return dt.strftime("%Y-%m-%d")

    @staticmethod
    def parse_date(date_str: str) -> datetime:
        return datetime.strptime(date_str, "%Y-%m-%d")
```

## 组件测试

每个组件都应该有对应的测试：

```python
# tests/services/test_user_service.py
import pytest

@pytest.mark.asyncio
async def test_get_user():
    service = UserService()
    user = await service.get_by_id("test-id")
    assert user is not None
```
