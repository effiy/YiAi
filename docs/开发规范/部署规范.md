# 部署规范

本文档定义 YiAi 项目的部署规范。

---

## 环境配置

### 环境类型

| 环境 | 说明 | 配置文件 |
|------|------|---------|
| `development` | 开发环境 | config.yaml (开发配置) |
| `staging` | 预发布环境 | config.yaml (预发布配置) |
| `production` | 生产环境 | config.yaml (生产配置) |

### 环境变量

生产环境推荐使用环境变量覆盖敏感配置：

```bash
# 服务器配置
export SERVER_HOST=0.0.0.0
export SERVER_PORT=8000

# MongoDB 配置
export MONGODB_URL=mongodb://user:password@localhost:27017
export MONGODB_DB_NAME=yiai_prod

# OSS 配置
export OSS_ACCESS_KEY=your-access-key
export OSS_SECRET_KEY=your-secret-key

# 认证配置
export MIDDLEWARE_AUTH_ENABLED=true
export API_X_TOKEN=your-production-token

# 日志配置
export LOGGING_LEVEL=INFO
```

## 配置检查清单

部署前检查以下配置：

### 必须配置

- [ ] `mongodb.url` - MongoDB 连接地址
- [ ] `mongodb.db_name` - 数据库名称
- [ ] `static.base_dir` - 静态文件目录
- [ ] `static.base_url` - 静态文件访问 URL

### 可选配置

- [ ] `oss.*` - OSS 配置（如使用 OSS）
- [ ] `rss.*` - RSS 配置（如使用 RSS）
- [ ] `ollama.*` - Ollama 配置（如使用 AI 功能）

### 安全配置（生产环境）

- [ ] `middleware.auth_enabled` - 设为 `true`
- [ ] `middleware.auth_token` - 设置强密码
- [ ] `logging.level` - 设为 `INFO` 或 `WARNING`

## 依赖安装

### 使用 requirements.txt

```bash
# 安装依赖
pip install -r requirements.txt

# 升级 pip
pip install --upgrade pip

# 安装到虚拟环境（推荐）
python -m venv venv
```

## 目录准备

### 创建必要目录

```bash
# 日志目录
mkdir -p logs

# 静态文件目录
mkdir -p /var/www/static

mkdir -p $(yq '.static.base_dir' config.yaml)
```

## 运行应用

### 开发环境

```bash
# 使用 main.py 直接运行（推荐）
python main.py

# 或使用 uvicorn 直接运行
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

### 生产环境

```bash
# 使用 uvicorn 生产配置
uvicorn src.main:app \
  --host 0.0.0.0 \
  --port 8000 \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --access-logfile logs/access.log \
  --error-logfile logs/error.log
```

### 使用 systemd 管理

创建服务文件 `/etc/systemd/system/yiai.service`：

```ini
[Unit]
Description=YiAi API Service
After=network.target mongod.service

[Service]
Type=notify
User=www-data
WorkingDirectory=/var/www/YiAi
Environment="PATH=/var/www/YiAi/venv/bin"
ExecStart=/var/www/YiAi/venv/bin/uvicorn src.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

管理服务：

```bash
# 重载配置
systemctl daemon-reload

# 启动服务
systemctl start yiai

# 停止服务
systemctl stop yiai

# 重启服务
systemctl restart yiai

# 查看状态
systemctl status yiai

# 查看日志
journalctl -u yiai -f

# 开机自启
systemctl enable yiai
```

## 反向代理配置

### Nginx 配置

创建 `/etc/nginx/sites-available/yiai`：

```nginx
upstream yiai_backend {
    server 127.0.0.1:8000;
    keepalive 32;
}

server {
    listen 80;
    server_name api.your-domain.com;

    # 日志
    access_log /var/log/nginx/yiai-access.log;
    error_log /var/log/nginx/yiai-error.log;

    # 客户端上传大小限制
    client_max_body_size 50M;

    # 静态文件
    location /static/ {
        alias /var/www/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # API 代理
    location / {
        proxy_pass http://yiai_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

启用配置：

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/yiai /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
systemctl reload nginx
```

### HTTPS 配置

使用 Let's Encrypt 证书：

```bash
# 安装 certbot
apt install certbot python3-certbot-nginx

# 获取证书并自动配置
certbot --nginx -d api.your-domain.com

# 自动续期
certbot renew --dry-run
```

## Docker 部署

### Dockerfile

创建 `Dockerfile`：

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY src/ ./src/
COPY config.yaml .

# 创建必要目录
RUN mkdir -p logs /var/www/static

# 环境变量
ENV PYTHONUNBUFFERED=1

# 暴露端口
EXPOSE 8000

# 运行应用
CMD ["uvicorn", "src.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### docker-compose.yml

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:7
    container_name: yiai-mongodb
    restart: always
    volumes:
      - mongodb_data:/data/db
    environment:
      MONGO_INITDB_DATABASE: yiai

  app:
    build: .
    container_name: yiai-app
    restart: always
    ports:
      - "8000:8000"
    volumes:
      - ./config.yaml:/app/config.yaml:ro
      - ./logs:/app/logs
      - static_data:/var/www/static
    depends_on:
      - mongodb
    environment:
      MONGODB_URL: mongodb://mongodb:27017

volumes:
  mongodb_data:
  static_data:
```

运行：

```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f app

# 停止
docker-compose down
```

## 健康检查

### 健康检查端点

可以添加健康检查端点：

```python
@router.get("/health")
async def health_check():
    """健康检查"""
    # 检查数据库连接
    try:
        await db.db.command("ping")
        db_ok = True
    except:
        db_ok = False

    return {
        "status": "healthy" if db_ok else "unhealthy",
        "database": "ok" if db_ok else "error",
        "timestamp": datetime.now(timezone.utc).isoformat()
    }
```

### 使用健康检查

在 systemd 或 Docker Compose 中配置：

```yaml
# docker-compose.yml
services:
  app:
    # ...
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
```

## 日志管理

### 日志轮转

配置 logrotate，创建 `/etc/logrotate.d/yiai`：

```
/var/www/YiAi/logs/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
    create 0640 www-data www-data
    sharedscripts
    postrotate
        systemctl reload yiai > /dev/null 2>&1 || true
    endscript
}
```

## 备份策略

### 数据库备份

创建备份脚本 `scripts/backup-db.sh`：

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/backups/yiai"
mkdir -p $BACKUP_DIR

# 备份 MongoDB
mongodump --uri="mongodb://localhost:27017" \
  --db="yiai" \
  --out="$BACKUP_DIR/mongodump_$DATE"

# 压缩
tar -czf "$BACKUP_DIR/yiai_$DATE.tar.gz" \
  -C "$BACKUP_DIR" "mongodump_$DATE"

# 清理临时文件
rm -rf "$BACKUP_DIR/mongodump_$DATE"

# 保留最近 30 天
find $BACKUP_DIR -name "yiai_*.tar.gz" -mtime +30 -delete
```

添加到 crontab：

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 2 点备份
0 2 * * * /var/www/YiAi/scripts/backup-db.sh
```
