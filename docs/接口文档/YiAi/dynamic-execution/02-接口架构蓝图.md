## 调用链（以 POST / 为例）

```mermaid
sequenceDiagram
    participant Client
    participant Route as execution.py
    participant Exec as executor.py
    participant Guard as ReentrancyGuard
    participant Sandbox as sandbox_context
    participant Target as 目标函数
    participant DB as state_records

    Client->>Route: POST / {module_name, method_name, parameters}
    Route->>Exec: execute_module(m, f, params)
    Exec->>Guard: _acquire_guard()
    alt 深度超限
        Guard-->>Exec: BusinessException (code=1004)
        Exec-->>Route: 异常
        Route-->>Client: fail(code=1004)
    end
    Exec->>Exec: _check_whitelist(m, f)
    alt 不在白名单
        Exec-->>Route: BusinessException (code=1003)
        Route-->>Client: fail(code=1003)
    end
    Exec->>Exec: parse_parameters(params) → dict
    Exec->>Exec: _import_target_function(m, f) → callable
    Exec->>Sandbox: sandbox_context(fs_allowlist, net_allowlist)
    Sandbox->>Target: fn(parsed_params)
    Target-->>Sandbox: result
    Sandbox-->>Exec: result
    Exec->>DB: SkillRecorder.record_async() (fire-and-forget)
    Exec-->>Route: result
    alt async generator
        Route-->>Client: SSE stream
    else 普通值
        Route-->>Client: success(data=result)
    end
```

## 关键设计决策

### 1. 白名单机制

白名单以 `module_path:function_name` 格式存储在 `settings.module_allowlist`。`"*"` 通配表示全部放行。生产环境建议精确指定。

```
# config.yaml
module_allowlist:
  - "services.ai.chat_service:chat"
  - "services.rss.rss_scheduler:get_scheduler_status_info"
```

### 2. 沙箱隔离

通过 `contextmanager` 临时替换 `builtins.open`，在执行目标函数期间拦截文件访问。执行完毕后自动恢复原始 `open()`。

- 启用条件：`observer_sandbox_enabled: true`
- 白名单路径：`observer_sandbox_fs_allowlist`
- 违规行为：抛出 `SandboxViolation`，被 executor 捕获转为 `BusinessException`

### 3. 重入守卫

基于 `contextvars.ContextVar` 的异步安全深度计数器。每次 `execute_module` 调用 depth+1，超出 `max_depth`（默认 3）拒绝执行。

- 目的：防止模块 A → execute_module → 模块 B → execute_module → ... 无限递归

### 4. 流式分发策略

| 检测顺序 | 类型 | 处理 |
|---------|------|------|
| 1 | `isasyncgen(result)` 或 `__aiter__` | `StreamingResponse` + SSE |
| 2 | `isinstance(result, GeneratorType)` | `StreamingResponse` + SSE |
| 3 | 其他 | `success(data=result)` |

### 5. 执行录制

每次执行通过 `SkillRecorder.record_async()` 以 fire-and-forget 方式写入 MongoDB `state_records` 集合。录制失败不阻断主流程。
