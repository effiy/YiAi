## 配置参考

```yaml
# config.yaml — 执行引擎相关配置

# 模块白名单：允许通过 API 调用的 module:function
module_allowlist:
  - "services.ai.chat_service:chat"
  - "services.ai.chat_service:list_ollama_models"
  - "services.rss.rss_scheduler:parse_all_enabled_rss_sources"
  - "services.rss.rss_scheduler:get_scheduler_status_info"

# Observer 沙箱
observer:
  sandbox_enabled: true
  sandbox_fs_allowlist: "/tmp,/var/www/YiAi/static"
  sandbox_network_allowlist: ""

  guard_enabled: true
  guard_max_depth: 3

# 状态存储（录制执行结果）
state_store:
  enabled: true
  query_max_limit: 8000
```

## 调用示例

### cURL — AI 对话（非流式）

```bash
curl -X POST http://localhost:10086/ \
  -H "Content-Type: application/json" \
  -H "X-Token: your-token" \
  -d '{
    "module_name": "services.ai.chat_service",
    "method_name": "chat",
    "parameters": {
      "user": "What is FastAPI?",
      "model": "qwen3.5"
    }
  }'
```

响应：
```json
{
  "code": 0,
  "data": {
    "success": true,
    "model": "qwen3.5",
    "message": "FastAPI is a modern web framework..."
  }
}
```

### cURL — AI 对话（流式）

```bash
curl -N -X POST http://localhost:10086/ \
  -H "Content-Type: application/json" \
  -H "X-Token: your-token" \
  -d '{
    "module_name": "services.ai.chat_service",
    "method_name": "chat",
    "parameters": {
      "user": "Tell me a story",
      "model": "qwen3.5",
      "stream": true
    }
  }'
```

响应（SSE 流）：
```
data: {"data":{"message":"Once"}}

data: {"data":{"message":" upon"}}

data: {"data":{"message":" a time..."}}

data: {"done":true}
```

### cURL — 获取可用模型

```bash
curl "http://localhost:10086/?module_name=services.ai.chat_service&method_name=list_ollama_models&parameters=%7B%7D" \
  -H "X-Token: your-token"
```

### cURL — RSS 调度器状态

```bash
curl "http://localhost:10086/?module_name=services.rss.rss_scheduler&method_name=get_scheduler_status_info&parameters=%7B%7D" \
  -H "X-Token: your-token"
```

### Python — 调用示例

```python
import requests

resp = requests.post(
    "http://localhost:10086/",
    headers={"X-Token": "your-token"},
    json={
        "module_name": "services.ai.chat_service",
        "method_name": "chat",
        "parameters": {"user": "Hello", "model": "qwen3.5"},
    },
)
print(resp.json())
```

### Python — SSE 流式消费

```python
import requests

with requests.post(
    "http://localhost:10086/",
    headers={"X-Token": "your-token"},
    json={
        "module_name": "services.ai.chat_service",
        "method_name": "chat",
        "parameters": {"user": "Hello", "model": "qwen3.5", "stream": True},
    },
    stream=True,
) as resp:
    for line in resp.iter_lines():
        if line:
            print(line.decode())
```

## 错误码参考

| code | 含义 | 常见原因 |
|------|------|---------|
| 0 | 成功 | — |
| 1001 | 内部错误 | 目标函数执行时异常 |
| 1002 | 参数非法 | module/function 为空、JSON 非法、模块不存在 |
| 1003 | 权限拒绝 | 模块不在白名单 |
| 1004 | 服务端错误 | 重入深度超限 |

## 如何添加新的可调用函数

1. 在 `services/` 或任意模块下创建函数，签名为 `async def fn(params: dict) -> Any` 或 `def fn(params: dict) -> Any`
2. 将 `module_path:function_name` 加入 `config.yaml` 的 `module_allowlist`
3. 如果是流式响应，返回 `AsyncIterator` 或 `Iterator`
4. 重启服务

无需修改路由代码。

## 运行时依赖

| 组件 | 启动条件 | 失败行为 |
|------|---------|---------|
| MongoDB | `startup_init_database: true` | 启动失败，拒绝服务 |
| Ollama | 按需（调用 AI 函数时） | 执行时返回错误 |
| Observer 沙箱 | `observer.sandbox_enabled: true` | — |
| Observer 守卫 | `observer.guard_enabled: true` | — |
| State Store | `state_store.enabled: true` | 录制静默失败，不阻断执行 |
