## 2. 模块契约

### 2.1 配置系统 (`config.py`)

**单例**: `settings = Settings()` — 全应用唯一实例

**配置源优先级** (settings_customise_sources):
1. `init_settings` (构造参数)
2. 环境变量
3. `.env` 文件
4. `file_secret_settings`
5. `config.yaml` (扁平化)

**关键字段** (7 组, ~70 个 field):

| 组 | 前缀 | 关键字段 |
|----|------|---------|
| Server | `server_` | host, port, reload |
| MongoDB | `mongodb_` | url, db_name, pool_size |
| OSS | `oss_` | access_key, secret_key, endpoint, bucket |
| RSS | `rss_scheduler_` | enabled, interval |
| Observer | `observer_` | enabled, throttle_*, sampler_*, sandbox_*, guard_* |
| State Store | `state_store_` | enabled, default_ttl, query_max_limit |
| Ollama | `ollama_` | url, auth |

**列表解析**: 逗号分隔字符串 → `_to_list()` 自动拆分

**属性计算**: `static_max_zip_size` = `static_max_zip_size_mb * MB`

### 2.2 数据库层 (`database.py`)

**类型**: 线程安全单例 (`threading.Lock` + `__new__`)

**连接池参数**:
- `maxPoolSize`: 50 (可配置)
- `minPoolSize`: 10
- `maxIdleTimeMS`: 30000
- `waitQueueTimeoutMS`: 10000
- `retryWrites`: true
- `retryReads`: true

**自动索引**: `rss.link` (唯一索引)

**CRUD 包装**:
| 方法 | 自动行为 |
|------|---------|
| `insert_one` | 自动添加 `createdTime` (UTC) |
| `insert_many` | 每个文档自动添加 `createdTime` |
| `find_one` | 直接透传 Motor |
| `find` | 未包装 — 由调用方自行使用 `db.db[collection]` |

### 2.3 认证中间件 (`middleware.py`)

**中间件类型**: 纯 ASGI 中间件 (`app.middleware("http")`)

**处理流程**:
```
Request → OPTIONS? → skip
       → whitelist path? → skip
       → auth disabled? → skip
       → X-Token match? → pass
                       → 401 UNAUTHORIZED + CORS headers
```

**白名单路径** (硬编码):
- `/write-file`, `/read-file`, `/delete-file`, `/upload`
- `/static/*` (前缀匹配)

**Token 来源**: `os.getenv("API_X_TOKEN", settings.middleware_auth_token)`

**CORS 头注入**: 所有错误响应自动附加 CORS 头 (`*` origin)

### 2.4 响应包装 (`response.py`)

**数据结构**:
```json
{
  "code": 0,        // business error code
  "message": "success",
  "data": null       // payload
}
```

**success()**: code=0, http_code=200
**fail()**: code=ErrorCode.business, http_code=ErrorCode.http

### 2.5 异常处理链 (`exception_handler.py`)

| 优先级 | 异常类型 | Handler | 映射 |
|--------|---------|---------|------|
| 1 | `RequestValidationError` | `validation_exception_handler` | INVALID_PARAMS (1002) |
| 2 | `BusinessException` | `business_exception_handler` | 透传 error_code |
| 3 | `HTTPException` | `http_exception_handler` | `map_http_to_error_code()` |
| 4 | `Exception` | `general_exception_handler` | SERVER_ERROR (5000) |

**HTTP→Business 映射表**:
```
401 → UNAUTHORIZED (1009)
404 → DATA_NOT_FOUND (1004)
403 → PERMISSION_DENIED (1008)
400 → INVALID_REQUEST (1000)
429 → RATE_LIMITED (1003)
500 → SERVER_ERROR (5000)
```

### 2.6 应用工厂 (`main.py`)

**create_app() 签名**:
```python
def create_app(
    *,
    enable_auth: bool | None = None,
    init_db: bool | None = None,
    init_rss: bool | None = None,
) -> FastAPI
```

**注册顺序** (不可调整):
1. `setup_logging()`
2. `register_exception_handlers(app)`
3. Observer 中间件 (Sampler → Throttle, 按配置启用)
4. API 路由注册 (6 个 router)
5. CORS 中间件
6. 认证中间件
7. Static 文件挂载

## 3. 数据流

### 请求生命周期

```
HTTP Request
  → CORS (headers)
  → Observer:Sampler (记录开始时间)
  → Observer:Throttle (IP限流检查)
  → Auth Middleware (X-Token验证)
  → Exception Handlers (包裹)
  → Route Handler
    → Service Layer
      → db (MongoDB singleton)
    ← response
  ← Observer:Sampler (记录结束时间+耗时)
  ← JSONResponse
```

### 配置流

```
config.yaml (文件)
  → YamlConfigSettingsSource._flatten()
  → Settings.__init__()
  → settings 单例 (全应用 import)
```

### 异常流

```
Route raises BusinessException(ErrorCode.XXX)
  → business_exception_handler
  → fail(error_code, message)
  → JSONResponse(status=error.http, content={code, message, data})
```

## 4. 威胁模型

| 威胁 | 信任边界 | 缓解 |
|------|---------|------|
| 未认证请求访问受保护端点 | 网络 → 应用 | X-Token 中间件验证，白名单最小化 |
| Token 硬编码泄露 | 代码仓库 | `auth_token` 属性优先读环境变量 `API_X_TOKEN` |
| 配置泄露 (OSS密钥) | config.yaml | config.yaml 应加入 .gitignore；环境变量覆盖 |
| 异常泄露内部实现 | 应用 → 客户端 | general_exception_handler 返回通用消息 "服务器内部错误" |
| MongoDB 注入 | API 输入 → 数据库 | Motor 驱动参数化查询；不可直接拼接查询字符串 |
| 数据库未初始化访问 | 应用内部 | `db.db` property 抛出 RuntimeError |
| 日志敏感信息泄露 | 应用 → 日志文件 | uvicorn.access 设为 WARNING；需审计日志内容 |

## 5. 错误码表

| Business Code | HTTP | 枚举 | 含义 |
|---------------|------|------|------|
| 0 | 200 | OK | 成功 |
| 1000 | 400 | INVALID_REQUEST | 无效请求 |
| 1001 | 400 | BUSINESS_ERROR | 业务错误 |
| 1002 | 400 | INVALID_PARAMS | 无效参数 |
| 1003 | 429 | RATE_LIMITED | 限流 |
| 1004 | 404 | DATA_NOT_FOUND | 资源不存在 |
| 1005 | 500 | DATA_STORE_FAIL | 新增失败 |
| 1006 | 500 | DATA_UPDATE_FAIL | 更新失败 |
| 1007 | 500 | DATA_DESTROY_FAIL | 删除失败 |
| 1008 | 403 | PERMISSION_DENIED | 权限拒绝 |
| 1009 | 401 | UNAUTHORIZED | 未认证 |
| 5000 | 500 | SERVER_ERROR | 服务器繁忙 |
| 5001 | 500 | INTERNAL_ERROR | 内部错误 |
