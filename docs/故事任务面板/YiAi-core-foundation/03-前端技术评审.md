# 前端技术评审 — core-foundation

> 来源: `src/core/response.py`, `src/core/error_codes.py`, `src/core/middleware.py`
> 证据等级: A 已验证

## 1. 概述

core-foundation 是纯后端基础模块，前端不直接实现。本文档描述前端应如何对接后端契约。

## 2. API 响应契约

### 2.1 统一响应格式

所有 API 返回以下 JSON 结构：

```typescript
interface ApiResponse<T> {
  code: number;    // 业务状态码: 0=成功, 1000+客户端错误, 5000+服务端错误
  message: string; // 提示消息
  data: T | null;  // 数据载荷
}
```

### 2.2 成功响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": { "key": "abc-123" }
}
```

分页响应额外包含 `pagination` 字段：
```json
{
  "code": 0,
  "message": "success",
  "data": [...],
  "pagination": { "page": 1, "size": 20, "total": 150 }
}
```

### 2.3 错误响应处理

前端应统一处理以下错误码：

| code | HTTP | 含义 | 前端行为 |
|------|------|------|---------|
| 0 | 200 | 成功 | 正常渲染 |
| 1009 | 401 | 未认证 | 跳转登录页 / 提示重新登录 |
| 1008 | 403 | 权限拒绝 | 提示无权限 |
| 1004 | 404 | 资源不存在 | 提示或跳转 404 页 |
| 1003 | 429 | 限流 | 读取 Retry-After 头，延迟重试 |
| 1002 | 400 | 参数错误 | 展示字段级错误 (data 中含 details) |
| 5000+ | 500 | 服务端错误 | 通用错误提示 "服务器繁忙" |

### 2.4 认证

- 请求头: `X-Token: <token>`
- 401 时检查 token 是否过期，自动刷新或跳转登录
- 白名单路径无需 token: `/read-file`, `/write-file`, `/delete-file`, `/upload`, `/static/*`

### 2.5 前端请求封装建议

```typescript
// 推荐封装
async function apiRequest<T>(url: string, options?: RequestInit): Promise<T> {
  const res = await fetch(url, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      'X-Token': getToken(),
      ...options?.headers,
    },
  });

  const json: ApiResponse<T> = await res.json();

  if (json.code !== 0) {
    // 统一错误处理
    handleErrorCode(json.code, json.message);
    throw new ApiError(json.code, json.message, json.data);
  }

  return json.data as T;
}
```

## 3. 不适用声明

本模块无前端 UI 组件、无状态管理、无路由变更。前端仅需适配 §2 的响应契约和认证流程。
