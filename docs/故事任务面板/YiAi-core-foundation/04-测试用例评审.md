# 测试用例评审 — core-foundation

> 来源: `src/core/*`, `src/main.py` — 基于源码接口推导
> 证据等级: B 可推导（测试用例从接口契约推导）

## 1. 配置系统测试

### TC1.1 YAML 扁平化
- **Given** config.yaml 包含 `server: {host: "0.0.0.0", port: 10086}`
- **When** 创建 Settings 实例
- **Then** `settings.server_host == "0.0.0.0"`, `settings.server_port == 10086`

### TC1.2 环境变量覆盖
- **Given** config.yaml 中 `server_port: 8000`，环境变量 `SERVER_PORT=9999`
- **When** 创建 Settings 实例
- **Then** `settings.server_port == 9999`

### TC1.3 默认值回退
- **Given** config.yaml 缺失 `mongodb_url` 字段
- **When** 创建 Settings 实例
- **Then** `settings.mongodb_url == "mongodb://localhost:27017"`（默认值）

### TC1.4 逗号分隔列表解析
- **Given** `cors_origins: "http://a.com, http://b.com"`
- **When** 调用 `settings.get_cors_origins()`
- **Then** 返回 `["http://a.com", "http://b.com"]`

### TC1.5 属性计算
- **Given** `static_max_zip_size_mb: 100`
- **When** 访问 `settings.static_max_zip_size`
- **Then** 返回 `104857600` (100 * 1024 * 1024)

## 2. 数据库层测试

### TC2.1 单例模式
- **Given** 多次调用 `MongoDB()`
- **When** 比较两个实例
- **Then** `db1 is db2` 为 True

### TC2.2 未初始化访问
- **Given** 数据库未调用 `initialize()`
- **When** 访问 `db.db`
- **Then** 抛出 RuntimeError("Database not initialized")

### TC2.3 自动时间戳
- **Given** 文档不含 `createdTime`
- **When** 调用 `db.insert_one("test", {"name": "test"})`
- **Then** 插入的文档包含 `createdTime` 字段（UTC 格式）

### TC2.4 索引创建
- **Given** 数据库初始化
- **When** `_ensure_indexes()` 被调用
- **Then** `rss` 集合的 `link` 字段有唯一索引

### TC2.5 连接关闭
- **Given** 数据库已初始化
- **When** 调用 `db.close()`
- **Then** `db._initialized == False`, `db._client is None`

## 3. 认证中间件测试

### TC3.1 有效 Token
- **Given** `middleware_auth_enabled=true`, `API_X_TOKEN=secret`
- **When** 请求携带 `X-Token: secret`
- **Then** 请求通过，正常返回

### TC3.2 无效 Token
- **Given** `middleware_auth_enabled=true`, `API_X_TOKEN=secret`
- **When** 请求携带 `X-Token: wrong`
- **Then** 返回 401，body 中 `code: 1009`

### TC3.3 认证禁用
- **Given** `middleware_auth_enabled=false`
- **When** 请求不带 X-Token
- **Then** 请求正常通过

### TC3.4 OPTIONS 预检
- **Given** 任何配置
- **When** 发送 OPTIONS 请求
- **Then** 跳过认证，正常返回

### TC3.5 白名单路径
- **Given** `middleware_auth_enabled=true`, 有效 token
- **When** 请求 `/read-file` 不带 X-Token
- **Then** 请求通过（白名单跳过）

## 4. 响应包装测试

### TC4.1 success 无数据
- **Given** 调用 `success()`
- **When** 检查响应
- **Then** status=200, body: `{code:0, message:"success", data:null}`

### TC4.2 success 带数据
- **Given** 调用 `success(data={"key":"v"})`
- **When** 检查响应
- **Then** status=200, body: `{code:0, data:{key:"v"}}`

### TC4.3 fail
- **Given** 调用 `fail(ErrorCode.DATA_NOT_FOUND, "not found")`
- **When** 检查响应
- **Then** status=404, body: `{code:1004, message:"not found"}`

## 5. 异常处理测试

### TC5.1 Pydantic 验证异常
- **Given** 路由要求必填字段
- **When** 请求缺少该字段
- **Then** 返回 400, code=1002, data 含字段错误详情

### TC5.2 业务异常

- **Then** 返回 404, code=1004, message="xxx"

### TC5.3 HTTP 异常

### TC5.4 未处理异常
