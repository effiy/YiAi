## §2 场景详述

### 场景 1：业务模块分页查询数据

| 字段 | 内容 |
|------|------|
| 角色 | 前端页面 / 数据分析人员 |
| 触发条件 | 需要浏览某集合的文档列表 |
| 核心目标 | 通过执行引擎调用 query_documents，获取分页过滤排序后的数据 |

```mermaid
flowchart TD
    CLIENT["前端/curl"] -->|"GET /api/execution?<br/>module_name=services.database.data_service<br/>&method_name=query_documents<br/>&parameters={cname:'sessions',pageSize:20}"| EXEC["执行引擎"]
    EXEC --> WHITE["白名单校验"]
    WHITE --> IMPORT["动态导入 data_service"]
    IMPORT --> QUERY["query_documents()"]
    QUERY --> VAL["_validate_collection_name()"]
    VAL --> FILTER["_build_filter()<br/>智能解析过滤条件"]
    FILTER --> SORT["_build_sort_list()<br/>构建排序链"]
    SORT --> PROJ["字段投影<br/>sessions 排除 pageContent"]
    PROJ --> MONGO["MongoDB .find().sort().skip().limit()"]
    MONGO --> RESP["返回 {list, total, pageNum, pageSize, totalPages}"]
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造查询 | `cname`, `pageNum`, `pageSize`, 过滤参数 | — | cname 为空→ValueError |
| 2 | 解析过滤 | 各参数按类型匹配过滤策略 | filter_dict 构建 | — |
| 3 | 数据库查询 | filter + sort + skip + limit | 文档列表 | 连接失败→异常 |
| 4 | 返回结果 | — | `{list:[...], total:N, pageNum, pageSize, totalPages}` | — |

### 场景 2：RSS 数据去重创建

| 字段 | 内容 |
|------|------|
| 角色 | RSS 调度器 |
| 触发条件 | 定时拉取外部 RSS 源，新条目需入库 |
| 核心目标 | 通过 create_document 创建 RSS 文档，link 重复时自动拦截 |

```mermaid
flowchart TD
    SCHEDULER["RSS 调度器"] -->|"create_document({cname:'rss', data:{title, link, pubDate...}})"| CREATE["create_document()"]
    CREATE --> CHECK{"rss 集合?<br/>link 字段?"}
    CHECK -->|"是"| DUP["查询 link 是否已存在"]
    DUP -->|"存在"| ERR["ValueError:<br/>link 已存在"]
    DUP -->|"不存在"| GEN["生成 key/UUID<br/>createdTime/updatedTime/order"]
    CHECK -->|"否"| GEN
    GEN --> INSERT["collection.insert_one()"]
    INSERT --> RESP["返回 {key: <uuid>}"]
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造创建请求 | `cname="rss"`, `data={title, link, ...}` | — | cname 为空→ValueError |
| 2 | link 去重检查 | `link` 值 | 查询集合 | link 已存在→ValueError |
| 3 | 注入元数据 | — | key/UUID + createdTime + updatedTime + order | order 查询失败→默认 1 |
| 4 | 执行插入 | 完整文档 | insert_one | 并发冲突→ValueError |
| 5 | 返回结果 | — | `{key: <uuid>}` | — |

### 场景 3：文档更新（保护不可变字段）

| 字段 | 内容 |
|------|------|
| 角色 | 业务模块（状态服务） |
| 触发条件 | 需要修改某文档的部分字段 |
| 核心目标 | update_document 按 key 定位文档，保护 _id/key/createdTime，自动刷新 updatedTime |

```mermaid
flowchart TD
    CALLER["业务服务"] -->|"update_document({cname, data:{key, field...}})"| UPDATE["update_document()"]
    UPDATE --> FIND["collection.find_one({key})"]
    FIND -->|"不存在"| ERR2["ValueError:<br/>未找到数据"]
    FIND -->|"存在"| STRIP["移除保护字段<br/>_id / key / createdTime"]
    STRIP --> SESS_CHK{"sessions 集合?"}
    SESS_CHK -->|"是"| STRIP_SESS["移除 pageContent<br/>messages=[] 丢弃"]
    SESS_CHK -->|"否"| TIME
    STRIP_SESS --> TIME["注入 updatedTime = now()"]
    TIME --> DO_UPDATE["collection.update_one($set)"]
    DO_UPDATE --> RESP2["返回 {key, updated:true}"]
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造更新 | `cname`, `data` 含 `key` | — | key 缺失→ValueError |
| 2 | 存在性检查 | `key` | find_one | 不存在→ValueError |
| 3 | 字段保护 | — | 剥离 _id/key/createdTime/pageContent | — |
| 4 | 时间刷新 | — | updatedTime = get_current_time() | — |
| 5 | 执行更新 | `$set` 操作 | update_one | — |
| 6 | 返回结果 | — | `{key, updated:true}` | — |

### 场景 4：Upsert（存在则更新、不存在则创建）

| 字段 | 内容 |
|------|------|
| 角色 | 业务模块（会话同步） |
| 触发条件 | 不确定文档是否存在，需要幂等写入 |
| 核心目标 | 通过 upsert_document 实现 filter 匹配则更新、不匹配则创建 |

```mermaid
flowchart TD
    CALLER2["会话同步服务"] -->|"upsert_document({cname, filter:{key:'x'}, update:{...}})"| UPSERT_FN["upsert_document()"]
    UPSERT_FN --> CHK_OP{"update 含<br/>$ 操作符?"}
    CHK_OP -->|"否"| WRAP["自动包装为 $set"]
    CHK_OP -->|"是"| INJECT
    WRAP --> INJECT["注入 $set.updatedTime<br/>$setOnInsert.createdTime + key"]
    INJECT --> DO_UP["collection.update_one(filter, update, upsert=True)"]
    DO_UP --> RESP3["返回 {matched_count, modified_count, upserted_id}"]
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造 upsert | `cname`, `filter`, `update` | 参数校验 | 缺必要参数→ValueError |
| 2 | 操作符检查 | update dict | 无 `$` 前缀时包装为 `$set` | — |
| 3 | 系统字段注入 | — | `$set.updatedTime` + `$setOnInsert.{createdTime,key}` | — |
| 4 | 执行 upsert | filter + update | update_one(upsert=True) | — |
| 5 | 返回结果 | — | `{matched_count, modified_count, upserted_id}` | — |

### 场景 5：故事任务目录聚合查询

| 字段 | 内容 |
|------|------|
| 角色 | 项目管理者 / 故事面板 |
| 触发条件 | 需要查看当前所有活跃的故事任务目录 |
| 核心目标 | 通过 `list_story_task_dirs` 一键获取按 projectName/storyName 去重的目录清单 |

```mermaid
flowchart TD
    PANEL["故事面板<br/>GET /api/execution"] -->|"list_story_task_dirs({project_name:'YiAi'})"| AGG_FN["list_story_task_dirs()"]
    AGG_FN --> MATCH["$match:<br/>projectName 存在且非空"]
    MATCH --> GROUP["$group:<br/>_id={projectName, storyName}<br/>session_count: $sum 1<br/>latest_time: $max updatedTime"]
    GROUP --> SORT3["$sort:<br/>projectName ↑, storyName ↑"]
    SORT3 --> SKIP["$skip"]
    SKIP --> LIMIT["$limit"]
    LIMIT --> COUNT["计数聚合:<br/>$group → $count"]
    COUNT --> RESP4["返回 {list:[{project_name, story_name, dir_path, session_count, latest_time}], total}"]
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造查询 | `project_name?` + 分页 | — | — |
| 2 | 聚合管道 | `$match` → `$group` → `$sort` → `$skip` → `$limit` | 去重目录列表 | MongoDB 异常→传播 |
| 3 | 计数 | 轻量 count aggregation | total 值 | — |
| 4 | 返回结果 | — | `{list:[...], total, pageNum, pageSize, totalPages}` | — |
