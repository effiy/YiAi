## §2 场景详述

### 场景 1：内部服务定时调用

| 字段 | 内容 |
|------|------|
| 角色 | 内部定时任务（RSS 调度器） |
| 触发条件 | 定时触发，需拉取外部 RSS 源 |
| 核心目标 | 动态调用 RSS 抓取模块，获取最新内容 |

```mermaid
flowchart TD
    SCHEDULER["RSS 调度器<br/>cron 触发"] --> POST["POST /api/execution<br/>{module_name:'services.rss.feed_service',<br/>method_name:'fetch_all', parameters:{}}"]
    POST --> EXEC["执行器处理"]
    EXEC --> GUARD{"守卫检查"}
    GUARD --> WL{"白名单校验"}
    WL --> RUN["执行 fetch_all()"]
    RUN --> RECORD["记录到 State Store"]
    RECORD --> RESP["返回抓取结果 JSON"]
    RESP --> SCHEDULER_LOG["调度器记录日志"]
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造请求 | `module_name`, `method_name`, `parameters` | — | 参数格式校验失败→400 |
| 2 | 发送 POST | JSON body | 白名单校验 | 模块未注册→403 |
| 3 | 执行模块 | 目标函数接收 parameters | 函数执行 | 模块导入失败→400 |
| 4 | 接收结果 | — | JSON `{code:0, data:...}` | 执行异常→500 |

### 场景 5：数据查询直调（GET 示例）

| 字段 | 内容 |
|------|------|
| 角色 | 开发者 / 数据分析人员 |
| 触发条件 | 需要快速查询 MongoDB 中指定集合的文档 |
| 核心目标 | 通过浏览器地址栏或 curl 直接 GET 查询，无需构造 POST body |

```mermaid
flowchart TD
    DEV2["开发者<br/>浏览器 / curl"] --> GET2["GET /api/execution?<br/>module_name=services.database.data_service<br/>&method_name=query_documents<br/>&parameters={%22cname%22:%22sessions%22}"]
    GET2 --> MW["X-Token 中间件"]
    MW --> EXEC2["executor.execute_module()"]
    EXEC2 --> WL2{"白名单校验"}
    WL2 -->|"通过"| PARSE2["parse_parameters()<br/>解析 URL-encoded JSON"]
    PARSE2 --> IMPORT2["动态导入 data_service"]
    IMPORT2 --> RUN2["query_documents({cname:'sessions'})"]
    RUN2 --> MONGO["MongoDB 查询<br/>sessions 集合"]
    MONGO --> RESP2["返回文档列表 JSON"]
    RESP2 --> DEV2_LOG["{code:0, data:[...]}"]

    WL2 -->|"未授权"| ERR2_2["403 PERMISSION_DENIED"]
    PARSE2 -->|"JSON 非法"| ERR2_3["400 INVALID_PARAMS"]
```

**实际请求 URL**：
```
GET /api/execution?module_name=services.database.data_service&method_name=query_documents&parameters=%7B%22cname%22%3A%22sessions%22%7D
```

**参数解码**：`parameters` URL-decode → `{"cname":"sessions"}`，传入 `query_documents(params: Dict[str, Any])` 作为 `params` 实参。

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造 GET URL | `module_name`, `method_name`, URL-encoded `parameters` | — | URL 编码错误→参数解析失败→400 |
| 2 | 白名单校验 | `services.database.data_service:query_documents` | 匹配 EXEC_ALLOWLIST | 未注册→403 |
| 3 | 参数解析 | `%7B%22cname%22%3A%22sessions%22%7D` → `{"cname":"sessions"}` | 解析为 dict | 非法 JSON→400 |
| 4 | 动态导入 | `services.database.data_service` → `query_documents` | 获取 callable | 模块/函数不存在→400 |
| 5 | 执行查询 | `query_documents({"cname":"sessions"})` | MongoDB 查询 sessions 集合 | 查询异常→500 |
| 6 | 返回结果 | — | `{code:0, data:[<documents>]}` | — |

### 场景 6：故事任务目录查询（聚合管道）

| 字段 | 内容 |
|------|------|
| 角色 | 项目管理者 / 开发者 |
| 触发条件 | 需要查看当前所有活跃的故事任务目录 |
| 核心目标 | 通过 GET 执行 `list_story_task_dirs` 一键获取去重的故事任务目录清单（含会话数、最近活跃时间） |

```mermaid
flowchart TD
    PM["项目管理者<br/>浏览器 / curl"] --> GET3["GET /api/execution?<br/>module_name=services.database.data_service<br/>&method_name=list_story_task_dirs<br/>&parameters={%22project_name%22:%22YiAi%22}"]
    GET3 --> MW3["X-Token 中间件"]
    MW3 --> EXEC3["executor.execute_module()"]
    EXEC3 --> WL3{"白名单校验"}
    WL3 -->|"通过"| PARSE3["parse_parameters()"]
    PARSE3 --> IMPORT3["动态导入 data_service"]
    IMPORT3 --> RUN3["list_story_task_dirs({project_name:'YiAi'})"]
    RUN3 --> MONGO3["MongoDB 聚合查询<br/>sessions.$match:$group"]
    MONGO3 --> RESP3["返回去重目录列表 JSON"]
    RESP3 --> PM_LOG["{code:0, data:{list:[...], total:3}}"]

    WL3 -->|"未授权"| ERR3_2["403 PERMISSION_DENIED"]
```

**实际请求 URL**：
```
GET /api/execution?module_name=services.database.data_service&method_name=list_story_task_dirs&parameters=%7B%22project_name%22%3A%22YiAi%22%7D
```

| # | 步骤 | 输入 | 系统响应 | 异常分支 |
|---|------|------|---------|---------|
| 1 | 构造 GET URL | `module_name`, `method_name`, URL-encoded `parameters` | — | URL 编码错误→参数解析失败→400 |
| 2 | 白名单校验 | `services.database.data_service:list_story_task_dirs` | 匹配 EXEC_ALLOWLIST | 未注册→403 |
