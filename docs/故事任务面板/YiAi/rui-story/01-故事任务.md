# 故事任务：rui-story

## 概述

实现 `/rui-story` 技能——故事任务面板 CRUD 管理。操作边界仅限 `docs/故事任务面板/`，提供增、删、改、查、同步 5 类命令 + 7 个 REST API 端点。

- **项目**：YiAi
- **故事**：rui-story
- **类型**：backend（含 API 接口）
- **优先级**：P1 — 故事面板管理基础设施

## 故事拆分

单故事，不分拆。7 个子命令 + 7 个 API 端点内聚于同一技能定义：

| # | 子命令 | 类型 | 复杂度 |
|---|--------|------|--------|
| 1 | `/rui-story` | 只读·状态概览 | 低 |
| 2 | `/rui-story list` | 只读·进度全景 | 中 |
| 3 | `/rui-story show <name>` | 只读·单故事详情 | 中 |
| 4 | `/rui-story create <name> [--type]` | 写入·创建骨架 | 低 |
| 5 | `/rui-story delete <name>` | 写入·删除目录 | 中 |
| 6 | `/rui-story sync [name]` | 写入·委托同步 | 低 |
| 7 | `/rui-story rename <old> <new>` | 写入·重命名 | 中 |

### API 接口（新增）

为每个子命令设计对应的 REST API 端点，供客户端直接调用。

| # | 方法 | 路径 | 说明 | 复杂度 |
|---|------|------|------|--------|
| 8 | GET | `/api/story-panel/overview` | 状态概览 | 低 |
| 9 | GET | `/api/story-panel/stories` | 进度全景列表 | 中 |
| 10 | GET | `/api/story-panel/stories/{project}/{name}` | 单故事详情 | 中 |
| 11 | POST | `/api/story-panel/stories/{project}/{name}` | 创建故事 | 低 |
| 12 | DELETE | `/api/story-panel/stories/{project}/{name}` | 删除故事 | 中 |
| 13 | POST | `/api/story-panel/stories/sync` | 文档同步 | 低 |
| 14 | PUT | `/api/story-panel/stories/{project}/{name}/rename` | 重命名故事 | 中 |

## 验收标准

- [ ] `/rui-story` 无参数输出状态概览（按状态聚合 + 最近活动）
- [ ] `/rui-story list` 输出所有故事详情表格
- [ ] `/rui-story show <name>` 输出单故事详述卡
- [ ] `/rui-story create <name> [--type]` 创建仅目录骨架，不产出文档
- [ ] `/rui-story delete <name>` 需用户确认，警告 git 分支
- [ ] `/rui-story sync [name]` 委托 import-docs
- [ ] `/rui-story rename <old> <new>` 重命名目录，警告 git 分支
- [ ] 命名规范强制：`<Project>` = PascalCase，`<name>` = kebab-case
- [ ] 状态判定准确（6 态：not_started → docs_in_progress → docs_done → code_in_progress → code_done → blocked）
- [ ] `--help` / `-h` / `help` 输出帮助信息（通过 `node skills/rui-story/help.mjs`）
- [ ] 7 个 API 端点全部可用，响应格式符合 StandardResponse
- [ ] API 通过 X-Token 认证中间件
- [ ] API 路径遍历防护：project/name 仅含 `[A-Za-z0-9_-]`
- [ ] `/api/story-panel/stories/sync` 正确委托 import-docs
- [ ] rename/delete API 正确警告 git 分支（不操作 git）
