# 用户使用场景：rui-story

## 场景 1：查看故事面板概览

**角色**：开发者  
**目标**：快速了解所有故事的当前状态分布

```
输入：/rui-story
输出：
  故事任务面板 · 状态概览
  ─────────────────────────────
    code_done          2
    code_in_progress   1
    docs_done          3
    docs_in_progress   0
    not_started        1
    blocked            0
  ─────────────────────────────
    合计               7 个故事

  最近活动：
    2026-05-17  YiAi-execution-executor-doc  code_done
    2026-05-16  YiAi-rui-story               docs_in_progress
```

## 场景 2：查看所有故事详细进度

**角色**：PM / 技术负责人  
**目标**：获取每个故事的完整状态、文件数、分支信息

```
输入：/rui-story list
输出：详细表格含 Project | Story | Status | Files | Last Modified | Type | Branch
```

## 场景 3：查看单个故事详情

**角色**：开发者  
**目标**：深入了解某个故事的文件组成和元数据

```
输入：/rui-story show YiAi-execution-executor-doc
输出：
  YiAi-execution-executor-doc · docs_done

  📂 目录: docs/故事任务面板/YiAi/execution-executor-doc/
  📋 类型: backend
  📄 文件: 5 个

    文件清单:
    01-故事任务.md         2.3 KB  2026-05-17 10:30
    02-用户使用场景.md      4.1 KB  2026-05-17 10:35
    03-后端技术评审.md      8.7 KB  2026-05-17 10:40
    05-测试用例评审.md      3.2 KB  2026-05-17 10:45
    10-交互日志.md          1.1 KB  2026-05-17 11:00

  🔀 Git 分支: feat/YiAi-execution-executor-doc

  📊 元数据:
    状态: docs_done
    阶段: doc
    阻断原因: —
```

## 场景 4：创建新故事目录

**角色**：PM  
**目标**：为一个新需求创建故事目录骨架

```
输入：/rui-story create YiAi-user-login --type backend
输出：
  ✅ 故事目录已创建: docs/故事任务面板/YiAi/user-login/
  📋 类型: backend
  ⚠️  仅创建目录骨架，文档请使用 /rui doc 生成
```

- 拒绝覆盖已有目录
- 仅创建 `.memory/story-type.json`，不创建文档

## 场景 5：删除故事目录

**角色**：PM  
**目标**：清理废弃的故事

```
输入：/rui-story delete YiAi-old-feature
输出：
  ⚠️  将删除目录: docs/故事任务面板/YiAi/old-feature/
  ⚠️  注意: git 分支 feat/YiAi-old-feature 将被保留，需手动清理
  确认删除？输入 yes/no: yes
  ✅ 已删除: docs/故事任务面板/YiAi/old-feature/
```

- 必须用户输入 `yes` 确认
- git 分支不删除，仅警告

## 场景 6：同步文档到远端

**角色**：开发者  
**目标**：将故事文档同步到远端 API

```
输入：/rui-story sync YiAi-execution-executor-doc
输出：
  [import-docs] scan root: docs/故事任务面板/YiAi/execution-executor-doc/
  [import-docs] found 5 files
  [import-docs] done — created: 0, overwritten: 5, failed: 0
```

- 完全委托 import-docs，不自行实现
- 缺 token 时降级跳过

## 场景 7：重命名故事

**角色**：PM  
**目标**：修正故事命名

```
输入：/rui-story rename YiAi-old-name YiAi-new-name
输出：
  ✅ 已重命名: YiAi-old-name → YiAi-new-name
  ⚠️  git 分支 feat/YiAi-old-name 需手动重命名
```

- 仅重命名目录，不操作 git
- 目标已存在时拒绝覆盖

## HTTP API 场景（新增）

以下场景面向客户端（Web/移动端）通过 HTTP API 调用故事面板管理功能。

### 场景 8：API 获取状态概览

**角色**：前端 Dashboard  
**目标**：获取故事面板整体状态用于首页展示

```
GET /api/story-panel/overview
Headers: X-Token: ${API_X_TOKEN}

Response 200:
{
  "code": 0,
  "message": "success",
  "data": {
    "summary": {
      "code_done": 2,
      "code_in_progress": 1,
      "docs_done": 3,
      "docs_in_progress": 0,
      "not_started": 1,
      "blocked": 0,
      "total": 7
    },
    "recent": [
      {
        "project": "YiAi",
        "name": "rui-story",
        "status": "code_done",
        "modified": "2026-05-17T13:23:00"
      }
    ]
  }
}
```

### 场景 9：API 获取故事列表

**角色**：前端 StoryList 页面  
**目标**：获取所有故事的详细表格数据

```
GET /api/story-panel/stories
Headers: X-Token: ${API_X_TOKEN}

Response 200:
{
  "code": 0,
  "data": {
    "stories": [
      {
        "project": "YiAi",
        "name": "rui-story",
        "status": "code_done",
        "files": 7,
        "last_modified": "2026-05-17T13:23:00",
        "type": "backend",
        "branch": "feat/YiAi-rui-story"
      }
    ]
  }
}
```

### 场景 10：API 获取单故事详情

**角色**：前端 StoryDetail 页面  
**目标**：获取某个故事的完整信息

```
GET /api/story-panel/stories/YiAi/rui-story
Headers: X-Token: ${API_X_TOKEN}

Response 200:
{
  "code": 0,
  "data": {
    "directory": "docs/故事任务面板/YiAi/rui-story/",
    "type": "backend",
    "files": [
      { "name": "01-故事任务.md", "size": 2135, "modified": "2026-05-17T12:51:00" }
    ],
    "branch": "feat/YiAi-rui-story",
    "metadata": {
      "status": "code_done",
      "stage": "delivery",
      "block_reason": null
    }
  }
}
```

故事不存在时返回 404：
```json
{ "code": 1004, "message": "故事不存在: YiAi/nonexistent", "data": null }
```

### 场景 11：API 创建故事

**角色**：前端创建表单  
**目标**：为新需求创建故事目录骨架

```
POST /api/story-panel/stories/YiAi/new-feature
Headers: X-Token: ${API_X_TOKEN}
Body: { "type": "backend" }

Response 201:
{
  "code": 0,
  "message": "故事目录已创建",
  "data": {
    "created": true,
    "directory": "docs/故事任务面板/YiAi/new-feature/"
  }
}
```

- 格式非法（非 PascalCase-kebab-case）→ 400
- 目标已存在 → 409
- type 默认 `meta`

### 场景 12：API 删除故事

**角色**：前端管理操作  
**目标**：删除废弃的故事目录

```
DELETE /api/story-panel/stories/YiAi/old-feature
Headers: X-Token: ${API_X_TOKEN}

Response 200:
{
  "code": 0,
  "message": "success",
  "data": {
    "deleted": true,
    "directory": "docs/故事任务面板/YiAi/old-feature/",
    "warning": "git 分支 feat/YiAi-old-feature 需手动清理"
  }
}
```

- 目录不存在 → 404
- 仅删目录，不删 git 分支

### 场景 13：API 同步文档

**角色**：前端同步按钮  
**目标**：触发文档同步到远端

```
POST /api/story-panel/stories/sync
Headers: X-Token: ${API_X_TOKEN}
Body: { "name": "YiAi-rui-story" }

Response 200:
{
  "code": 0,
  "data": {
    "synced": true,
    "created": 0,
    "overwritten": 7,
    "failed": 0
  }
}
```

`name` 为空 → 同步全量 `docs/故事任务面板/`。

### 场景 14：API 重命名故事

**角色**：前端重命名操作  
**目标**：修正故事命名

```
PUT /api/story-panel/stories/YiAi/old-name/rename
Headers: X-Token: ${API_X_TOKEN}
Body: { "new_project": "YiAi", "new_name": "new-name" }

Response 200:
{
  "code": 0,
  "data": {
    "renamed": true,
    "old": "docs/故事任务面板/YiAi/old-name/",
    "new": "docs/故事任务面板/YiAi/new-name/",
    "warning": "git 分支 feat/YiAi-old-name 需手动重命名"
  }
}
```

- 旧目录不存在 → 404
- 新目录已存在 → 409

### 场景 15：API 认证失败

**角色**：未授权客户端  
**目标**：访问 API 但缺少有效 token

```
GET /api/story-panel/overview
Headers: (无 X-Token)

Response 401:
{ "code": 1009, "message": "未认证", "data": null }
```
