# 测试用例评审：rui-story

## 测试策略

rui-story 是 Claude Code 技能，测试以行为验证为主：每个子命令按场景输入，验证输出格式和文件系统副作用。

## 测试用例

### TC1：状态概览 `/rui-story`

| 项 | 内容 |
|----|------|
| **输入** | `/rui-story`（无参数） |
| **前置** | `docs/故事任务面板/` 下存在多个不同状态的故事目录 |
| **预期** | 输出按状态聚合的计数表 + 最近 5 个活动故事 |
| **验证点** | ① 状态计数准确 ② 最近活动按修改时间降序 ③ `--help` 输出帮助文本 |

### TC2：进度全景 `/rui-story list`

| 项 | 内容 |
|----|------|
| **输入** | `/rui-story list` |
| **前置** | 同上 |
| **预期** | 表格含 Project / Story / Status / Files / Last Modified / Type / Branch |
| **验证点** | ① 文件计数准确（仅 .md） ② Last Modified 为目录中最晚文件时间 ③ Type 按存在文件正确推断 ④ Branch 正确查询 git |

### TC3：单故事详情 `/rui-story show <name>`

| 项 | 内容 |
|----|------|
| **输入** | `/rui-story show YiAi-execution-executor-doc` |
| **前置** | 目标故事目录存在且含文档 |
| **预期** | 输出详述卡：目录/类型/文件清单/git分支/元数据 |
| **验证点** | ① 文件清单含大小和修改时间 ② 状态判定正确 ③ `.memory/rui-state.json` 阻断信息显示正确 |

### TC4：单故事详情——不存在

| 项 | 内容 |
|----|------|
| **输入** | `/rui-story show YiAi-nonexistent` |
| **预期** | 报错：故事不存在 |
| **验证点** | 不输出文件清单，明确报错 |

### TC5：创建故事 `/rui-story create <name> --type`

|----|------|

| **验证点** | ① 目录存在 ② `story-type.json` 含 `{"type":"backend","createdAt":"..."}` ③ 无 .md 文件 |

## 边界条件

- 空面板（`docs/故事任务面板/` 下无故事）：概览显示全 0
- 故事目录空（仅 `.memory/`）：状态 `not_started`
- `.memory/rui-state.json` 含 `blocked=true`：状态 `blocked`
- API_X_TOKEN 缺失时 sync 降级：静默跳过
- 网络不可达时 sync：告警不阻断
