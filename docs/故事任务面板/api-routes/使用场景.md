# API 路由层 — 用户使用场景

## 场景 1：通过 API 动态执行服务模块

**参与者**：前端应用/外部服务

**流程**：
1. 发送 `POST /` 携带 JSON body：
   ```json
   {"module_name": "services.ai.chat_service", "method_name": "chat", "parameters": {"user": "你好"}}
   ```
2. 执行引擎校验白名单、导入模块、调用函数
3. 同步结果直接返回 JSON；流式结果以 SSE 格式推送
4. 也可通过 GET 传递参数

## 场景 2：文件上传与读取

**参与者**：前端用户

**流程**：
1. 上传：`POST /upload` 携带 base64 内容、目标目录、文件名
2. 文件写入磁盘 static 目录
3. 读取：`POST /read-file` 指定 target_file 路径
4. 图片文件返回静态 URL，文本文件返回内容
5. 磁盘未找到自动回退 MongoDB 查询

## 场景 3：图片上传到 OSS

**参与者**：前端用户

**流程**：
1. `POST /upload-image-to-oss` 携带 data_url (base64)
2. 系统尝试上传到阿里云 OSS
3. OSS 不可用时自动降级到本地存储
4. 返回可访问的文件 URL

## 场景 4：发送企业微信消息

**参与者**：自动化流程/外部服务

**流程**：
1. `POST /wework/send-message` 携带 webhook_url 和 content
2. 校验 URL 格式（必须以 qyapi.weixin.qq.com 开头）
3. 构建 text 类型消息体发送到企业微信
4. 检查 errcode：非 0 返回错误

## 场景 5：清理未引用图片

**参与者**：运维人员

**流程**：
1. `POST /maintenance/cleanup-unused-images` 携带 `{"dry_run": true}`
2. 系统扫描 static 目录所有图片 → 从 sessions 集合提取所有引用 → 对比找出未引用图片
3. dry_run=true 仅返回统计，不实际删除
4. dry_run=false 实际删除文件并释放空间
5. cleanup_sessions=true 同时清理引用了不存在图片的 session

## 场景 6：状态记录管理

**参与者**：内部服务

**流程**：
1. 创建：`POST /state/records` → 返回 key
2. 查询：`GET /state/records?record_type=skill_execution&page_num=1&page_size=20`
3. 更新：`PUT /state/records/{key}`
4. 删除：`DELETE /state/records/{key}`

## 场景 7：故事面板同步

**参与者**：开发者

**流程**：
1. `GET /api/story-panel/overview` 查看所有故事状态分布
2. `POST /api/story-panel/stories/sync` 携带 `{"names": ["core-infra"]}` 从远端下载文档
3. 本地 `docs/故事任务面板/core-infra/` 下的文档被远端版本覆盖
