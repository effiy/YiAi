# API 路由层 — 安全审计

## 审计范围

- `src/api/routes/execution.py` — 动态模块执行
- `src/api/routes/upload.py` — 文件管理
- `src/api/routes/wework.py` — 企业微信 Webhook
- `src/api/routes/maintenance.py` — 维护接口
- `src/api/routes/state.py` — 状态记录
- `src/api/routes/observer_health.py` — 健康检查
- `src/api/routes/story_panel.py` — 故事面板

## 安全发现

### [P0] 路径遍历防护

- **发现**：upload.py 中 `_validate_path()` 和 `_resolve_static_path()` 实施了路径遍历防护
- **验证**：拒绝绝对路径、`..` 穿越、符号链接绕过（使用 realpath）、编码绕过
- **结论**：防护完整，符合 OWASP 路径遍历防护标准

### [P1] 文件上传限制

- **发现**：文件上传未限制文件大小（upload 路由），未限制文件类型
- **风险**：攻击者可上传超大文件耗尽磁盘，或上传恶意脚本
- **建议**：增加文件大小上限、扩展名白名单、内容类型校验

### [P1] 企业微信 SSRF 风险

- **发现**：wework.py 校验 URL 前缀 `https://qyapi.weixin.qq.com/`，但攻击者可能使用开放重定向或子域名绕过
- **风险**：可能被利用为 SSRF 跳板攻击内网服务
- **建议**：增加 DNS 解析校验，确保解析后的 IP 属于企业微信官方 IP 段

### [P2] 维护接口权限

- **发现**：`/maintenance/cleanup-unused-images` 可删除文件和 sessions，无额外权限控制
- **风险**：误操作或恶意调用可导致数据丢失
- **建议**：dry_run 默认 true，增加二次确认机制

### [P2] 状态记录注入

- **发现**：state.py 的查询参数（title_contains）使用 `$regex` 构造，已有 `.*` + `$options: "i"` 正则注入可能
- **验证**：用户输入的标题内容直接拼入正则表达式，需确认无 ReDoS 风险
- **建议**：使用 `re.escape()` 转义特殊字符（当前代码未使用）

### [P2] 故事面板路径校验

- **发现**：story_panel.py 中 `_validate_name()` 使用 kebab-case 正则 + basename 校验
- **验证**：`name != os.path.basename(name)` + 拒绝 `.` 开头 + 拒绝 `..` + kebab-case 正则
- **结论**：路径安全控制到位

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 路径遍历防护 | 通过 | realpath + commonpath 双重验证 |
| 文件上传安全 | 部分 | 缺少大小和类型限制 |
| SSRF 防护 | 部分 | URL 前缀校验存在绕过风险 |
| 输入校验 | 通过 | Pydantic 模型 + 自定义校验 |
| 错误信息脱敏 | 通过 | BusinessException 统一处理 |
| 操作审计 | 待实施 | 敏感操作无审计日志 |

## 审计结论

API 路由层安全控制较为完善。路径遍历防护是亮点（realpath + commonpath 双验证）。主要改进点：文件上传需增加大小/类型限制，企业微信 Webhook 需加固 SSRF 防护，敏感操作（删除文件/目录/sessions）需增加审计日志。
