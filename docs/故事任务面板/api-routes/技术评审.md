# API 路由层 — 后端技术评审

## 架构设计

### 路由注册

```
FastAPI App
├── upload.router          # tags=["Upload"]
├── execution.router       # tags=["Execution"]
├── wework.router          # tags=["WeWork"]
├── maintenance.router     # tags=["Maintenance"]
├── state.router           # tags=["State"]  (prefix="/state")
├── observer_health.router # tags=["Observer"]
├── story_panel.router     # tags=["StoryPanel"]
└── StaticFiles("/static") # 静态文件挂载
```

### 文件操作安全模型

```
请求 target_file
    ↓
_normalize_no_spaces()    # 空格→下划线
    ↓
_validate_path()           # 拒绝绝对路径、../
    ↓
_resolve_static_path()     # realpath 解析，验证在 base_dir 内
    ↓
磁盘操作 / MongoDB 操作
```

路径验证使用 `os.path.realpath` + `os.path.commonpath` 双重防护防止路径遍历。

### 执行路由 SSE 流

```
execute_module() 返回
    ├── AsyncIterator → _stream_async() → SSE bytes
    ├── Generator → _stream_sync() → SSE bytes
    └── 普通值 → success(data=result)
```

SSE 格式：`data: {"data": {...}}\n\n`，流结束标志：`{"done": true}`。

### 维护清理流程

```
scan_static_images()        → Set[str]（static 目录所有图片）
get_all_session_contents()  → Set[str]（sessions 中引用的图片）
find_unused_images()        → 差集
delete_image_files()        → 物理删除 / dry_run 计数
cleanup_sessions_with_missing_images() → 清理无效 session
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 执行入口 | GET + POST 双入口 | 兼容浏览器直接访问和 JSON 调用 |
| 流式响应 | SSE (text/event-stream) | 简单可靠，浏览器原生支持 |
| 文件存储 | 磁盘 + MongoDB 双持久化 | 磁盘为主要存储，MongoDB 为备份和跨节点同步 |
| 图片返回 | URL 而非 base64 | 减少传输量，利用浏览器缓存 |
| OSS 降级 | OSS → 本地 | 保证可用性 |
| 路径安全 | realpath + commonpath | 防止编码绕过和符号链接攻击 |

## 数据流

```
Client Request
    ↓
CORS Middleware
    ↓
Auth Middleware (X-Token)
    ↓
Observer Middlewares (Throttle → Sampler)
    ↓
Route Handler
    ├── execution → executor.execute_module()
    ├── upload → 磁盘 I/O + MongoDB
    ├── wework → aiohttp → 企业微信 API
    ├── maintenance → 文件扫描 + sessions 聚合
    ├── state → StateStoreService → MongoDB
    ├── observer_health → config + 运行时数据
    └── story_panel → 本地文件 + 远端 API
```
