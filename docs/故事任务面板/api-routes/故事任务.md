# API 路由层

## 故事概述

构建 YiAi 的 HTTP API 路由层，提供动态模块执行、文件管理、企业微信 Webhook、系统维护、状态记录 CRUD、Observer 健康检查和故事面板管理共 7 组 REST API。

## 功能范围

### 动态模块执行 (`api/routes/execution.py`)
- GET + POST 双入口：`GET /?module_name=&method_name=&parameters=` 和 `POST /` (JSON body)
- SSE（Server-Sent Events）流式响应：支持异步生成器和同步生成器
- 委托 `services.execution.executor.execute_module` 执行目标函数

### 文件管理 (`api/routes/upload.py`)
- 上传：`POST /upload`（JSON base64/文本）、`POST /upload-image-to-oss`（图片→OSS，降级本地）
- 读取：`POST /read-file`（文本/base64，图片返回 URL，磁盘优先→MongoDB 回退）
- 写入：`POST /write-file`（磁盘 + MongoDB 双持久化，upsert）
- 删除：`POST /delete-file`、`POST /delete-folder`（递归删除目录）
- 重命名：`POST /rename-file`、`POST /rename-folder`（同步 MongoDB 记录）

### 企业微信 Webhook (`api/routes/wework.py`)
- `POST /wework/send-message`：发送文本消息到企业微信机器人
- URL 格式校验（必须 `https://qyapi.weixin.qq.com/`）
- 企业微信 errcode 检查（非 0 抛出异常）
- 10 秒超时

### 维护接口 (`api/routes/maintenance.py`)
- `POST /maintenance/cleanup-unused-images`：
  - 扫描 static 目录所有图片 + 从 sessions 集合提取引用
  - 找出未引用图片，支持 dry_run 预览和实际删除
  - 可选清理引用了不存在图片的 sessions

### 状态记录 (`api/routes/state.py`)
- `POST /state/records` — 创建记录
- `GET /state/records` — 分页查询（类型/标签/标题/时间范围筛选）
- `GET /state/records/{key}` — 获取单条
- `PUT /state/records/{key}` — 更新
- `DELETE /state/records/{key}` — 删除

### Observer 健康检查 (`api/routes/observer_health.py`)
- `GET /health/observer`：返回限流/采样/沙箱/守卫各组件的启用状态和运行时数据

### 故事面板管理 (`api/routes/story_panel.py`)
- `GET /api/story-panel/overview`：六状态聚合计数 + 最近活动故事
- `GET /api/story-panel/stories`：所有故事详情（状态/文件数/修改时间/类型/分支）
- `GET /api/story-panel/stories/{name}`：单故事详情（文件清单/元数据）
- `POST /api/story-panel/stories/sync`：从远端 API 下载文档覆盖本地
- `GET /api/story-panel/remote`：远端故事查询
- 状态模型：not_started → docs_in_progress → docs_done → code_in_progress → code_done / blocked

## 验收标准
- [ ] 所有路由正确注册到 FastAPI app
- [ ] 文件操作路径遍历防护生效
- [ ] 企业微信消息发送成功（有效 webhook URL）
- [ ] 维护清理 dry_run 模式不实际删除文件
- [ ] 状态记录 CRUD 完整可用
- [ ] Observer 健康检查返回正确状态
