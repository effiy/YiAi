# API 路由层 — 测试用例评审

## 测试用例

### TC-API-001：执行模块 — GET 方式调用

| 项目 | 内容 |
|------|------|
| 目标 | GET 请求正确执行目标函数 |
| 步骤 | `GET /?module_name=services.rss.rss_scheduler&method_name=get_scheduler_status_info&parameters={}` |
| 预期 | HTTP 200，返回调度器状态 JSON |

### TC-API-002：执行模块 — POST 方式调用

| 项目 | 内容 |
|------|------|
| 目标 | POST JSON 请求正确执行 |
| 步骤 | `POST /` body: `{"module_name": "...", "method_name": "...", "parameters": {...}}` |
| 预期 | HTTP 200，返回执行结果 |

### TC-API-003：文件上传 — 文本文件

| 项目 | 内容 |
|------|------|
| 目标 | 文本文件正确写入磁盘 |
| 步骤 | `POST /upload` body: `{"target_dir": "test", "filename": "hello.txt", "content": "Hello", "is_base64": false}` |
| 预期 | 文件写入 static/test/hello.txt，返回 url |

### TC-API-004：文件读取 — 图片返回 URL

| 项目 | 内容 |
|------|------|
| 目标 | 图片文件返回静态 URL |
| 步骤 | `POST /read-file` body: `{"target_file": "test/photo.png"}` |
| 预期 | data.content 为完整静态 URL，data.type 为 "url" |

### TC-API-005：文件删除

| 项目 | 内容 |
|------|------|
| 目标 | 文件正确删除 |
| 步骤 | 1. 上传文件 2. `POST /delete-file` 删除 3. 读取该文件 |
| 预期 | 删除返回成功，再次读取返回 404 |

### TC-API-006：路径遍历防护 — ../ 攻击

| 项目 | 内容 |
|------|------|
| 目标 | 拒绝路径遍历攻击 |
| 步骤 | `POST /read-file` body: `{"target_file": "../etc/passwd"}` |
| 预期 | 返回非法路径错误，不读取文件 |

### TC-API-008：维护清理 — dry_run 模式

| 项目 | 内容 |
|------|------|
| 目标 | dry_run=true 不实际删除 |
| 前置 | static 目录存在未引用图片 |
| 步骤 | `POST /maintenance/cleanup-unused-images` body: `{"dry_run": true}` |
| 预期 | 返回统计信息，文件实际未被删除 |

### TC-API-009：状态记录 — 完整 CRUD
