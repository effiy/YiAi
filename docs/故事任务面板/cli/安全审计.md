# CLI 工具 — 安全审计

## 审计范围

- `src/cli/state_query.py` — 状态查询 CLI

## 安全发现

### [P2] sys.path 注入

- **发现**：`sys.path.insert(0, "/var/www/YiAi/src")` 硬编码绝对路径
- **风险**：如果 `/var/www/YiAi/src` 目录权限不当，攻击者可在其中放置恶意模块被导入
- **建议**：使用相对路径或基于 `__file__` 计算项目根目录

### [P2] 输出文件路径无校验

- **发现**：`--output` 参数接受任意路径，直接 `open(path, "w")`
- **风险**：可能覆盖系统关键文件（如 `--output /etc/cron.d/malicious`）
- **建议**：限制输出路径在用户目录或当前目录下

### [INFO] 服务层安全控制

- CLI 直接调用 StateStoreService，绕过 HTTP 认证中间件
- 这意味着 CLI 工具的使用者需要有服务器本地访问权限
- 这是 CLI 工具的预期行为，不是安全漏洞

### [INFO] 数据库凭据访问

- CLI 通过导入服务层间接访问 MongoDB，凭据来自 config.yaml
- 需要确保 config.yaml 的权限限制为仅授权用户可读

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 路径遍历 | 部分 | --output 无路径限制 |
| 代码注入 | 低风险 | CLI 工具仅本地使用 |
| 认证绕过 | 预期 | CLI 需本地访问 |
| 文件权限 | 待确认 | config.yaml 权限 |

## 审计结论

CLI 工具安全风险较低，因其仅限本地使用。主要改进点：--output 参数增加路径安全校验，sys.path 使用动态计算的项目路径而非硬编码。
