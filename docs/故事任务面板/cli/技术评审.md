# CLI 工具 — 后端技术评审

## 架构设计

### CLI 框架

```
typer.Typer(app)
├── list    → _run() → service.query() → 格式化输出
├── get     → _run() → service.get()   → console.print_json()
├── export  → 委托 list() with --output
└── stats   → _run() → service.query(page_size=1) → console.print()
```

### 输出格式

```
--format table (默认)
    ├── rich.Table 构建
    ├── 动态列（从第一条记录提取 keys）
    └── 底部显示 Total/Page

--format json
    ├── json.dumps(indent=2, ensure_ascii=False)
    └── --output → 文件写入

--format csv
    ├── csv.DictWriter
    └── --output → 文件写入
```

### 服务集成

CLI 通过直接导入 `StateStoreService` 使用服务层，不经由 HTTP API：
```python
sys.path.insert(0, "/var/www/YiAi/src")
from services.state.state_service import StateStoreService
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| CLI 框架 | typer | FastAPI 同源（tiangolo），类型提示驱动 |
| 终端渲染 | rich | 美观的表格和颜色输出 |
| 导出格式 | JSON / CSV | 通用数据分析格式 |
| 服务调用 | 直接导入 | 避免 HTTP 开销，命令行场景适用 |
| 异步处理 | asyncio.run() | 适配异步服务层 |

## 数据流

```
CLI 命令
    ↓
typer 解析参数
    ↓
asyncio.run(_run())
    ↓
StateStoreService.query/get
    ↓
MongoDB 查询
    ↓
格式化输出 (table/json/csv)
    ↓
终端 / 文件
```
