# 核心基础设施 — 用户使用场景

## 场景 1：服务启动与配置加载

**参与者**：运维人员

**前置条件**：config.yaml 已配置

**流程**：
1. 运维人员执行 `python main.py` 启动服务
2. Settings 单例初始化，依次从初始化参数、环境变量、.env、密钥文件、config.yaml 加载配置
3. config.yaml 中嵌套结构被扁平化（如 `server.host` → `server_host`）
4. 配置值中的 `~` 被 expanduser 展开
5. 服务监听配置的 host:port

## 场景 2：数据库连接与索引

**参与者**：系统

**前置条件**：MongoDB 服务可用

**流程**：
1. 应用启动时调用 `db.initialize()`
2. 创建 AsyncIOMotorClient 连接池（默认 min=10, max=50）
3. 对 rss 集合创建 link 唯一索引
4. 对 static_files 集合创建 target_file 唯一索引
5. 连接失败则应用启动失败，抛出异常

## 场景 3：API 认证

**参与者**：API 调用方

**前置条件**：中间件认证已启用，X-Token 已配置

**流程**：
1. 客户端发起请求，携带 X-Token 请求头
2. 中间件检查请求方法：OPTIONS 直接放行
3. 检查请求路径：白名单路径（/write-file、/read-file、/delete-file、/upload、/static/*）直接放行
4. 检查中间件开关：未启用则放行
5. 检查 X-Token 是否与配置一致：不一致返回 UNAUTHORIZED 错误
6. 认证通过，继续处理请求

## 场景 4：业务异常处理

**参与者**：开发者

**前置条件**：代码中抛出 BusinessException

**流程**：
1. 业务代码抛出 `BusinessException(ErrorCode.INVALID_PARAMS, message="参数错误")`
2. 全局异常处理器捕获
3. 返回 JSONResponse：`{"code": 1001, "message": "参数错误", "data": null}`
4. HTTP 状态码根据 ErrorCode 映射
5. 未预期的 Exception 被捕获返回 SERVER_ERROR

## 场景 5：配置热切换

**参与者**：运维人员

**流程**：
1. 修改 config.yaml 中某个配置项
2. 重启服务后新配置生效
3. 环境变量可覆盖 config.yaml 中的同名配置
