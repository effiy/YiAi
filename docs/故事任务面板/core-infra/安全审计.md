# 核心基础设施 — 安全审计

## 审计范围

- `src/core/config.py` — 配置管理
- `src/core/database.py` — 数据库连接
- `src/core/middleware.py` — 认证中间件
- `src/core/response.py` — 响应格式
- `src/core/exceptions.py` / `exception_handler.py` — 异常处理
- `src/core/logger.py` — 日志系统
- `src/core/utils.py` — 工具函数

## 安全发现

### [P0] X-Token 认证强度

- **发现**：认证基于单一 X-Token 请求头，Token 通过配置文件或环境变量静态设置
- **风险**：若 Token 泄露，攻击者可访问所有 API。Token 无过期机制，无轮换策略
- **建议**：考虑增加 Token 过期时间、支持多 Token、记录认证失败日志用于告警

### [P1] 默认认证关闭

- **发现**：`middleware_auth_enabled` 默认为 `False`
- **风险**：生产环境若未显式开启，所有 API 无认证保护
- **建议**：生产部署检查清单中强制要求确认此项配置

### [P1] 数据库连接字符串

- **发现**：`mongodb_url` 包含凭据时明文存储在 config.yaml
- **风险**：配置文件泄露导致数据库凭据泄露
- **建议**：支持通过环境变量单独设置连接字符串，config.yaml 中仅保留非敏感默认值

### [P2] 错误信息泄露

- **发现**：全局异常处理器在非生产模式下可能暴露堆栈信息
- **风险**：内部实现细节泄露给攻击者
- **建议**：生产环境统一返回通用错误消息，详细错误仅记录日志

### [P2] CORS 配置

- **发现**：默认 `cors_origins: ["*"]`，`cors_allow_any_origin: True`
- **风险**：任意来源可跨域请求 API
- **建议**：生产环境应限制为具体域名列表

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 认证机制 | 已实施 | X-Token 中间件，可配置开关 |
| 认证默认安全 | 风险 | 默认关闭，需生产部署时手动开启 |
| 敏感数据加密 | 未实施 | 配置文件中凭证明文存储 |
| 错误信息脱敏 | 部分 | 业务异常有控制，未预期异常可能泄露 |
| CORS 限制 | 宽松 | 默认允许所有来源 |
| 日志安全 | 待确认 | 需确认日志中不记录 Token 等敏感信息 |

## 审计结论

核心基础设施安全基础扎实，认证中间件、统一异常处理、错误码体系构成了安全基线。主要改进点：默认安全配置应更严格（认证默认开启、CORS 限制），凭据管理应支持环境变量注入以避免配置文件泄露风险。
