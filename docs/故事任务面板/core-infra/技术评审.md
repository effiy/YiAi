# 核心基础设施 — 后端技术评审

## 架构设计

### 配置加载链

```
init_settings → env_settings → dotenv_settings → file_secret_settings → YamlConfigSettingsSource
```

YamlConfigSettingsSource 读取 config.yaml，递归扁平化嵌套字典：`{"server": {"host": "0.0.0.0"}}` → `{"server_host": "0.0.0.0"}`。使用 pydantic-settings 的 `settings_customise_sources` 钩子注入。

### 数据库单例模式

```
MongoDB (Singleton)
├── _instance: Optional[MongoDB]     # 单例实例
├── _lock: threading.Lock            # 双重检查锁
├── _client: AsyncIOMotorClient      # motor 异步客户端
├── _db: AsyncIOMotorDatabase        # 数据库引用
└── _initialized: bool               # 初始化标记
```

`initialize()` 幂等：已初始化则直接返回。`close()` 关闭连接并重置标记。

### 中间件链路

```
Request → CORS Middleware → Auth Middleware → Route Handler
                               ├── OPTIONS → skip
                               ├── Whitelist path → skip
                               ├── Auth disabled → skip
                               ├── Token missing → skip
                               └── Token mismatch → 401
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 配置格式 | YAML + pydantic-settings | 人类可读，支持环境变量覆盖 |
| 数据库驱动 | motor (async) | 全栈异步，不阻塞事件循环 |
| 单例模式 | 双重检查锁定 | 线程安全，避免重复初始化 |
| 认证方案 | X-Token 请求头 | 简单有效，适合内部服务 |
| 错误模型 | ErrorCode 枚举 + BusinessException | 统一错误码，便于前端处理 |

## 数据流

```
config.yaml → YamlConfigSettingsSource._flatten() → Settings 单例
                                                         ├── server_host, server_port → uvicorn.run()
                                                         ├── mongodb_url, mongodb_db_name → MongoDB.initialize()
                                                         ├── middleware_auth_enabled → header_verification_middleware
                                                         └── observer_* → Observer 组件初始化
```

## 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| MongoDB 不可用 | 应用无法启动 | initialize() 中抛异常，阻断启动 |
| config.yaml 缺失 | 使用默认值 | 所有字段有 Field(default=...) |
| X-Token 泄露 | 未授权访问 | 支持环境变量 API_X_TOKEN 覆盖 |
| 连接池耗尽 | 请求超时 | waitQueueTimeoutMS=10s，maxPoolSize=50 |
