# 核心基础设施

## 故事概述

构建 YiAi 后端服务的核心基础设施层，提供配置管理、数据库连接、认证中间件、统一响应格式、错误码体系和日志系统。本模块是所有其他模块运行的基石。

## 功能范围

### 配置系统 (`core/config.py`)
- 基于 pydantic-settings 的配置管理，支持 config.yaml 扁平化加载
- 自定义 YamlConfigSettingsSource 将嵌套 YAML 映射为扁平配置键
- 覆盖链：init_settings → env_settings → dotenv_settings → file_secret_settings → yaml_settings
- 涵盖：服务器、CORS、分页、静态文件、MongoDB、OSS、RSS、启动行为、中间件、模块白名单、State Store、Observer、Ollama、日志 等配置域

### 数据库层 (`core/database.py`)
- MongoDB 单例（双重检查锁定），motor 异步驱动
- 连接池管理：minPoolSize/maxPoolSize/maxIdleTimeMS/waitQueueTimeoutMS
- 自动索引创建：rss.link（唯一）、static_files.target_file（唯一）
- 常用 CRUD 包装：insert_one、insert_many、find_one
- 自动注入 createdTime 时间戳

### 认证中间件 (`core/middleware.py`)
- X-Token 请求头验证
- OPTIONS 预检请求跳过鉴权
- 白名单路径：/write-file、/read-file、/delete-file、/upload、/static/*
- 可通过配置开关启用/禁用
- 认证失败返回标准错误响应并附加 CORS 头

### 统一响应 (`core/response.py`)
- success(data, http_code) / fail(error, message) 标准响应格式
- JSONResponse 封装，统一 code/message/data 结构

### 错误码体系 (`core/error_codes.py`)
- ErrorCode 枚举：1xxx 客户端错误（INVALID_PARAMS、UNAUTHORIZED、PERMISSION_DENIED、DATA_NOT_FOUND）／5xxx 服务端错误（INTERNAL_ERROR、SERVER_ERROR、DATA_STORE_FAIL、DATA_UPDATE_FAIL、DATA_DESTROY_FAIL）

### 异常处理 (`core/exceptions.py`, `core/exception_handler.py`)
- BusinessException：携带 ErrorCode 的业务异常
- 全局异常处理器注册：捕获 BusinessException → 返回标准错误响应；捕获未处理异常 → 返回 500

### 日志系统 (`core/logger.py`)
- 统一日志格式与级别配置
- uvicorn 日志传播控制

### 工具函数 (`core/utils.py`)
- 文本处理、时间格式化、数字判断、集合去重等通用工具

## 验收标准
- [ ] config.yaml 所有配置项可正确加载到 Settings 单例
- [ ] MongoDB 连接池正常启动，索引自动创建
- [ ] X-Token 认证中间件正确拦截未授权请求
- [ ] BusinessException 抛出后返回统一错误格式
- [ ] 日志按配置级别输出
