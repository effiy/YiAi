# 核心基础设施 — 测试用例评审

## 测试策略

单元测试覆盖配置加载、数据库初始化、中间件逻辑、异常处理、工具函数。

## 测试用例

### TC-CI-001：配置从 YAML 正确加载

| 项目 | 内容 |
|------|------|
| 目标 | 验证 config.yaml 扁平化加载 |
| 前置 | 准备测试用 config.yaml |
| 步骤 | 1. 写入 `server.port: 9999` 2. 初始化 Settings 3. 读取 settings.server_port |
| 预期 | server_port == 9999 |
| 类型 | 单元测试 |

### TC-CI-002：环境变量覆盖 YAML 配置

| 项目 | 内容 |
|------|------|
| 目标 | 验证环境变量优先级高于 YAML |
| 前置 | config.yaml 中 server_port=8000 |
| 步骤 | 1. 设置环境变量 SERVER_PORT=9000 2. 初始化 Settings |
| 预期 | server_port == 9000 |

### TC-CI-003：MongoDB 初始化幂等性

| 项目 | 内容 |
|------|------|
| 目标 | 验证多次 initialize() 不会重复创建连接 |
| 前置 | MongoDB 可用 |
| 步骤 | 1. await db.initialize() 2. await db.initialize() 3. 检查连接数 |
| 预期 | 两次调用后 _initialized=True，无异常 |

### TC-CI-004：认证中间件 — 有效 Token 放行

| 项目 | 内容 |
|------|------|
| 目标 | 有效 X-Token 通过认证 |
| 前置 | auth_enabled=True, token="secret" |
| 步骤 | 发送 GET / 携带 X-Token: secret |
| 预期 | 请求正常处理，HTTP 200 |

### TC-CI-005：认证中间件 — 无效 Token 拒绝

| 项目 | 内容 |
|------|------|
| 目标 | 无效 X-Token 被拦截 |
| 前置 | auth_enabled=True, token="secret" |
| 步骤 | 发送 GET / 携带 X-Token: wrong |
| 预期 | HTTP 200，body code=1002 UNAUTHORIZED |

### TC-CI-006：认证中间件 — 白名单路径跳过

| 项目 | 内容 |
|------|------|
| 目标 | /read-file 路径不鉴权 |
| 前置 | auth_enabled=True |
| 步骤 | POST /read-file 不携带 X-Token |
| 预期 | 请求进入路由处理（非认证错误） |

### TC-CI-007：BusinessException 全局捕获

| 项目 | 内容 |
|------|------|
| 目标 | 业务异常返回标准格式 |
| 步骤 | 1. 触发 BusinessException(ErrorCode.DATA_NOT_FOUND) 2. 检查响应 |
| 预期 | `{"code": 1004, "message": "...", "data": null}` |

### TC-CI-008：未处理异常全局捕获

| 项目 | 内容 |
|------|------|
| 目标 | 未预期异常返回 500 |
| 步骤 | 1. 触发 RuntimeError 2. 检查响应 |
