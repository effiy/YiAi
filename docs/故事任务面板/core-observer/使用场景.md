# Observer 可靠性子系统 — 用户使用场景

## 场景 1：限流保护

**参与者**：API 调用方

**前置条件**：Observer 和 Throttle 已启用，max_requests=100, window_seconds=60

**流程**：
1. 某 IP 在 60 秒内发起第 101 次请求
2. ThrottleMiddleware 检查该 IP 的请求计数
3. 超过阈值，返回 429 Too Many Requests
4. 白名单内 IP 不受此限制
5. 窗口过期后计数器自动重置

## 场景 2：慢请求监控

**参与者**：运维人员

**前置条件**：Observer 和 Sampler 已启用，slow_threshold_ms=5000

**流程**：
1. 某 API 请求耗时 8 秒
2. SamplerMiddleware 记录请求开始时间
3. 请求完成后计算耗时：8000ms > 5000ms
4. 该请求被标记为慢请求，存入 TailSampler 缓冲区
5. 运维人员可通过 health/observer 接口查看采样数据

## 场景 3：受控沙箱执行

**参与者**：模块执行引擎

**前置条件**：沙箱已启用，fs_allowlist=["/var/www/YiAi/static"]

**流程**：
1. 执行引擎调用 sandbox_context 包装目标函数
2. 目标函数尝试访问 /etc/passwd
3. 沙箱检测到路径不在 fs_allowlist 中
4. 抛出 SandboxViolation 异常
5. 违规记录被统计

## 场景 4：重入防护

**参与者**：模块执行引擎

**前置条件**：ReentrancyGuard 已启用，max_depth=3

**流程**：
1. 模块 A 调用执行引擎 execute_module
2. 执行深度 = 1，正常执行
3. 模块 A 内部又调用 execute_module 执行模块 B
4. 执行深度 = 2，正常执行
5. 模块 B 内部又调用 execute_module 执行模块 C
6. 执行深度 = 3，正常执行
7. 模块 C 内部再次调用 execute_module
8. 执行深度 = 4 > max_depth=3，抛出 ReentrancyExceeded 异常
