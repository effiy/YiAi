# Observer 可靠性子系统 — 后端技术评审

## 架构设计

### 组件关系

```
FastAPI App
├── ThrottleMiddleware     # ASGI middleware，最先执行
├── SamplerMiddleware      # ASGI middleware，记录耗时
├── SandboxMiddleware      # ASGI middleware（可选，默认关闭）
├── LazyStartManager       # 生命周期管理，非中间件
└── ReentrancyGuard        # contextvar 实现，嵌入 executor
```

### 限流算法

基于滑动窗口的 IP 计数：
- 数据结构：`Dict[str, Deque[float]]`（IP → 请求时间戳队列）
- 每次请求：清理窗口外的时间戳，检查队列长度
- 白名单：`set()` O(1) 查找

### 尾部采样设计

```
TailSampler
├── _buffer: List[SampleRecord]   # 固定容量循环缓冲区
├── max_size: int                 # 最大样本数（默认 1000）
├── slow_threshold_ms: float      # 慢请求阈值（默认 5000ms）
└── add(record) / get_slow()      # 添加样本 / 获取慢请求
```

### 沙箱实现

使用 Python 的 monkey-patch 机制：
- `builtins.open` → 检查路径白名单
- `socket.socket` → 检查网络白名单
- `sandbox_context` 上下文管理器确保退出时恢复原始函数

### 重入守卫

```
contextvar: _reentrancy_depth (默认 0)
ReentrancyGuard.acquire() → token = _reentrancy_depth.set(depth + 1)
ReentrancyGuard.release(token) → _reentrancy_depth.reset(token)
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 限流粒度 | 按 IP | 防止单 IP 滥用，实现简单 |
| 限流存储 | 内存字典 | 无需外部依赖，重启后清零可接受 |
| 采样策略 | 固定容量尾部采样 | 保留最新数据，内存可控 |
| 沙箱方式 | monkey-patch | 无需修改被调用代码，透明拦截 |
| 深度追踪 | contextvar | 异步安全，自动随 Task 传播 |

## 数据流

```
Request → ThrottleMiddleware（检查限流）
       → SamplerMiddleware（记录开始时间）
       → SandboxMiddleware（设置沙箱上下文）
       → Route Handler（业务处理）
       → SandboxMiddleware（恢复原始函数）
       → SamplerMiddleware（计算耗时，记录慢请求）
       → Response
```
