# Observer 可靠性子系统

## 故事概述

构建 API 服务的可靠性防护层，提供限流（Throttle）、尾部采样（TailSampler）、沙箱执行（Sandbox）、懒启动（LazyStart）和重入守卫（ReentrancyGuard）五个组件，确保服务在高负载和异常流量下的稳定性。

## 功能范围

### 限流中间件 (`observer/throttle.py`)
- 基于 IP 的请求频率限制
- 可配置窗口时间（window_seconds）和最大请求数（max_requests）
- IP 白名单机制：白名单内 IP 不限流
- 通过 `observer_throttle_enabled` 开关控制

### 尾部采样 (`observer/sampler.py`)
- TailSampler：固定容量缓冲区，保留最近的请求样本
- 慢请求捕获：超过 slow_threshold_ms 的请求自动标记
- SamplerMiddleware：ASGI 中间件，自动记录请求耗时

### 沙箱 (`observer/sandbox.py`)
- 文件系统访问控制：fs_allowlist 白名单
- 网络访问控制：network_allowlist 白名单
- sandbox_context 上下文管理器
- 违规记录：SandboxViolation

### 懒启动 (`observer/lazy_start.py`)
- LazyStartManager：管理组件延迟初始化
- 减少启动时间，按需加载非关键组件

### 重入守卫 (`observer/guard.py`)
- ReentrancyGuard：限制执行深度（max_depth，默认 3）
- 防止无限递归调用导致栈溢出
- contextvar 跟踪当前深度

## 验收标准
- [ ] 限流中间件正确拦截超过阈值的请求
- [ ] 白名单 IP 不受限流影响
- [ ] 尾部采样正确记录慢请求
- [ ] 沙箱阻止非白名单文件系统和网络访问
- [ ] 重入守卫在深度超限时抛出异常
- [ ] 所有组件可通过配置独立开关
