# Observer 可靠性子系统 — 测试用例评审

## 测试用例

### TC-OB-001：限流 — 正常请求放行

| 项目 | 内容 |
|------|------|
| 目标 | 阈值内请求正常通过 |
| 前置 | max_requests=100, window_seconds=60 |
| 步骤 | 同一 IP 发送 50 次请求 |
| 预期 | 全部 HTTP 200 |

### TC-OB-002：限流 — 超阈值拒绝

| 项目 | 内容 |
|------|------|
| 目标 | 超过阈值后返回 429 |
| 前置 | max_requests=5, window_seconds=60 |
| 步骤 | 同一 IP 发送 6 次请求 |
| 预期 | 前 5 次 200，第 6 次 429 |

### TC-OB-003：限流 — 白名单 IP 不限流

| 项目 | 内容 |
|------|------|
| 目标 | 白名单 IP 不限制 |
| 前置 | whitelist=["192.168.1.1"], max_requests=5 |
| 步骤 | 白名单 IP 发送 100 次请求 |
| 预期 | 全部 HTTP 200 |

### TC-OB-004：限流 — 窗口过期重置

| 项目 | 内容 |
|------|------|
| 目标 | 时间窗口过后计数器重置 |
| 前置 | max_requests=5, window_seconds=1 |
| 步骤 | 1. 发送 5 次请求 2. 等待 1 秒 3. 再发送 5 次 |
| 预期 | 两批请求均成功 |

### TC-OB-005：采样 — 慢请求捕获

| 项目 | 内容 |
|------|------|
| 目标 | 超过阈值的请求被记录 |
| 前置 | slow_threshold_ms=100 |
| 步骤 | 调用耗时 500ms 的接口 |
| 预期 | 该请求出现在慢请求列表中 |

### TC-OB-006：采样 — 快请求不记录

| 项目 | 内容 |
|------|------|
| 目标 | 低于阈值的请求不被标记 |
| 前置 | slow_threshold_ms=5000 |
| 步骤 | 调用耗时 10ms 的接口 |
| 预期 | 慢请求列表不包含该请求 |

### TC-OB-007：沙箱 — 阻止非白名单路径写入

| 项目 | 内容 |
|------|------|
| 目标 | 沙箱阻止越权文件写入 |
| 前置 | fs_allowlist=["/var/www/YiAi/static"] |
| 步骤 | 目标函数尝试写入 /etc/hosts |
| 预期 | 抛出 SandboxViolation |

### TC-OB-008：沙箱 — 白名单路径允许写入

| 项目 | 内容 |
|------|------|
| 目标 | 白名单路径正常写入 |
| 前置 | fs_allowlist=["/var/www/YiAi/static"] |
| 步骤 | 目标函数写入 /var/www/YiAi/static/test.txt |
| 预期 | 写入成功 |

### TC-OB-009：重入守卫 — 深度超限

| 项目 | 内容 |
|------|------|
| 目标 | 超过最大深度抛出异常 |
| 前置 | max_depth=3 |
| 步骤 | 嵌套调用 execute_module 4 层 |
| 预期 | 第 4 层抛出 ReentrancyExceeded |

### TC-OB-010：重入守卫 — Token 正确释放

| 项目 | 内容 |
|------|------|
| 目标 | 异常时也能正确释放深度标记 |
| 前置 | max_depth=3 |
| 步骤 | 第 2 层抛出异常，检查后续调用深度 |
| 预期 | 后续调用深度从 1 开始（非累积） |
