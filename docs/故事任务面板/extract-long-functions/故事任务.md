# 故事任务: 拆分长函数

## §1 Story

| 维度 | 内容 |
|------|------|
| 角色 | 后端维护者 |
| 场景 | 多个核心函数超过30行（最多112行），承担过多职责，难以测试和维护 |
| 价值 | 按职责拆分为小函数，提升可读性、可测试性、单一职责 |
| 范围 | `src/services/static/static_files.py`, `src/services/execution/executor.py`, `src/services/rss/feed_service.py` |
| 依赖 | 无上游依赖 |
| 变更级别 | T3（函数签名和结构变更） |

## §2 Requirements

| # | 功能点 | 文件 | 当前行数 | 目标拆分 |
|---|--------|------|---------|---------|
| R1 | 拆分 `upload_and_unzip` | `static_files.py:58` | 112 | ZIP验证→解压→路径安全→写入，各为独立函数 |
| R2 | 拆分 `execute_module` | `executor.py:154` | 82 | 守卫→导入→执行调度→错误处理→记录 |
| R3 | 拆分 `process_feed_from_url` | `feed_service.py:72` | 96 | 抓取→解析→逐条目处理→保存 |

**业务规则：**
- 拆分后行为不变（纯重构，零功能变更）
- 新辅助函数使用 `_` 前缀（私有）
- 保持现有类型提示和错误处理模式

## §3 Design

### R1: `upload_and_unzip` → 3 函数

```
upload_and_unzip(params)
  ├── _validate_and_open_zip(file_path) → (bool, str)
  ├── _extract_with_common_root(zip_ref, extract_dir) → str
  └── _safe_write_files(extract_dir, target_dir, base_dir) → int
```

### R2: `execute_module` → 4 函数

```
execute_module(module_name, parameters)
  ├── _check_guard_and_whitelist(module_name) → None|raise
  ├── _import_module(module_name) → ModuleType
  ├── _execute_callable(obj, parameters) → Any
  └── _record_execution(module_name, parameters, result, error)
```

### R3: `process_feed_from_url` → 3 函数

```
process_feed_from_url(url, source_id)
  ├── _fetch_and_parse_feed(url) → FeedParserDict
  ├── _process_feed_entry(entry, source_id) → bool
  └── process_feed_from_url(url, source_id) → int  (thin orchestrator)
```
