# 数据模型 — 用户使用场景

## 场景 1：API 请求自动校验

**参与者**：API 调用方

**流程**：
1. 客户端发送 `POST /` 但缺少必填字段 module_name
2. Pydantic 自动校验 ExecuteRequest 模型
3. 校验失败，FastAPI 返回 422 Validation Error
4. 错误响应包含具体缺失字段信息

## 场景 2：可选字段默认值

**参与者**：API 调用方

**流程**：
1. 发送文件上传请求，不指定 is_base64
2. FileUploadRequest 模型 is_base64 默认为 False
3. 文件以文本方式写入

## 场景 3：集合名引用

**参与者**：开发者

**流程**：
1. 在代码中使用 `settings.collection_sessions` 而非硬编码字符串 "sessions"
2. 集合名变更时只需修改 config.yaml，无需改动代码
3. 新增集合时在 config.yaml 添加配置项，在 Settings 类添加对应 Field
