# 数据模型 — 安全审计

## 审计范围

- `src/models/schemas.py` — Pydantic 请求/响应模型
- `src/models/collections.py` — MongoDB 集合名常量

## 安全发现

### [P2] 模型字段缺少长度限制

- **发现**：FileUploadRequest.content、WeWorkWebhookRequest.content 等字符串字段无 max_length 限制
- **风险**：攻击者可发送超大字符串导致内存耗尽
- **建议**：为字符串字段添加合理的 max_length 约束

### [P2] 模型字段缺少格式校验

- **发现**：target_file、target_dir 等路径字段仅做字符串类型校验，格式校验在业务层完成
- **风险**：如果业务层校验被绕过，恶意路径可能到达文件操作层
- **建议**：在 Pydantic 模型层添加正则格式校验作为第一道防线

### [INFO] Pydantic v2 安全特性

- Pydantic v2 默认不进行类型强制转换（严格模式），减少了类型混淆风险
- `model_dump(exclude_unset=True)` 避免了未设置的字段被意外包含

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 输入类型校验 | 通过 | Pydantic 自动类型校验 |
| 字符串长度限制 | 缺失 | 大部分字段无 max_length |
| 正则格式校验 | 部分 | 路径字段依赖业务层校验 |
| 默认值安全 | 通过 | 无危险的默认值 |

## 审计结论

数据模型层安全风险较低。Pydantic v2 自动提供类型安全。主要改进点：为字符串字段添加长度限制作为第一道防线，在模型层增加正则格式校验作为纵深防御。
