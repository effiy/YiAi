# 数据模型 — 后端技术评审

## 架构设计

### Pydantic 模型层次

```
ExecuteRequest
├── module_name: str (必填)
├── method_name: str (必填)
└── parameters: Dict | str = {}

FileUploadRequest
├── target_dir: str = ""
├── filename: str (必填)
├── content: str (必填)
└── is_base64: bool = False

StateRecord
├── key: Optional[str] = None (创建时自动生成)
├── record_type: Optional[str] = None
├── title: Optional[str] = None
├── tags: List[str] = []
├── data: Optional[Dict] = None
└── 使用 model_dump(exclude_unset=True) 忽略未设置字段
```

### 集合名管理

集合名通过 config.yaml 配置 → Settings 扁平化 → 代码中通过 `settings.collection_*` 引用：

| 配置键 | 默认值 | 用途 |
|--------|--------|------|
| collection_sessions | sessions | 会话/文档存储 |
| collection_rss | rss | RSS 条目 |
| collection_chat_records | chat_records | 对话记录 |
| collection_pet_data_sync | pet_data_sync | 数据同步 |
| collection_seeds | seeds | RSS 源配置 |
| collection_oss_file_tags | oss_file_tags | OSS 文件标签 |
| collection_oss_file_info | oss_file_info | OSS 文件信息 |
| collection_static_files | static_files | 静态文件备份 |
| collection_state_records | state_records | 状态记录 |

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 模型库 | Pydantic v2 | FastAPI 原生支持，自动校验和文档生成 |
| 集合名管理 | config.yaml + Settings | 集中管理，支持环境切换 |
| StateRecord 更新 | exclude_unset=True | 只更新客户端明确传了的字段，避免意外覆盖 |
| 分页默认值 | page_size=2000, max=8000 | 适配大量数据查询，避免性能问题 |
