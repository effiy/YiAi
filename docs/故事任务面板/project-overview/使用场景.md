# 项目总览 — 用户使用场景

## 场景 1：新人快速上手

**参与者**：新加入项目的开发者

**流程**：
1. 阅读项目画像了解项目类型、技术栈、入口文件
2. 查看技术架构全景图理解系统分层和数据流向
3. 对照模块地图定位各功能模块的源码位置
4. 查看配置体系了解可配置项和环境变量覆盖规则
5. 查阅退化对策表了解系统容错边界

## 场景 2：架构评审

**参与者**：技术负责人 / 架构师

**流程**：
1. 审查中间件链路确认请求处理顺序
2. 审查数据存储表确认数据模型设计
3. 审查安全面确认纵深防御覆盖
4. 审查退化对策确认容错策略完整
5. 对照模块地图确认模块职责边界清晰

## 场景 3：故障排查定位

**参与者**：运维 / 值班人员

**流程**：
1. 根据异常现象定位到具体模块（通过模块地图）
2. 查看配置体系确认相关开关和参数
3. 查阅退化对策表确认是否为已知降级场景
4. 查看中间件链路确认请求经过了哪些中间件处理
5. 对照安全面确认是否触发了安全防护机制

## 场景 4：依赖变更影响分析

**参与者**：开发者

**流程**：
1. 定位需要变更的模块（通过模块地图）
2. 追踪该模块的下游消费者（通过技术架构全景图）
3. 确认受影响的数据存储集合（通过数据存储表）
4. 评估变更对安全面的影响（通过安全面清单）
5. 更新退化对策表（如引入新的降级场景）

## 场景 5：安全审计准备

**参与者**：安全工程师

**流程**：
1. 从安全面清单确认当前防护机制
2. 审查 API 认证链路（X-Token 中间件位置）
3. 审查路径安全机制（realpath + commonpath）
4. 审查文件上传安全（扩展名白名单 + ZIP 安全解压）
5. 审查模块执行安全（白名单 + 沙箱 + 重入守卫）
6. 审查限流配置（滑动窗口 + IP 白名单）
