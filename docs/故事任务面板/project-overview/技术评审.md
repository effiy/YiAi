# 项目总览 — 后端技术评审

## 架构设计

### 系统拓扑

```
                        ┌─────────────────────────┐
                        │     Nginx / CDN          │
                        └───────────┬─────────────┘
                                    │
                        ┌───────────▼─────────────┐
                        │    FastAPI Application    │
                        │  ┌─────────────────────┐ │
                        │  │   CORS Middleware    │ │
                        │  ├─────────────────────┤ │
                        │  │   Auth Middleware    │ │
                        │  │   (X-Token)          │ │
                        │  ├─────────────────────┤ │
                        │  │   Observer System    │ │
                        │  │   (Throttle/Sampler) │ │
                        │  ├─────────────────────┤ │
                        │  │   API Routes (7)     │ │
                        │  ├─────────────────────┤ │
                        │  │   Service Layer (9)  │ │
                        │  ├─────────────────────┤ │
                        │  │   Core Infra         │ │
                        │  └─────────────────────┘ │
                        └───┬─────────┬───────────┘
                            │         │
                  ┌─────────▼──┐  ┌──▼──────────┐
                  │  MongoDB   │  │  Ollama      │
                  │  (motor)   │  │  (AI 模型)   │
                  └────────────┘  └─────────────┘
                            │
                  ┌─────────▼──┐
                  │  Aliyun OSS │
                  │  (文件存储) │
                  └────────────┘
```

### 中间件链路

```
Request
  → CORS Middleware (allow_origins, allow_methods, allow_headers)
  → Observer Throttle Middleware (IP 限流，可配置)
  → Observer Sampler Middleware (请求耗时记录，可配置)
  → Auth Middleware (X-Token 验证，可配置)
  → Route Handler
  → Response
```

## 配置体系

```
config.yaml (扁平化加载)
    ├── server.*          # 服务器配置（host/port/reload/uvicorn）
    ├── cors_origins      # CORS 跨域配置
    ├── mongodb_*         # MongoDB 连接池配置
    ├── collection_*      # MongoDB 集合名（9 个集合）
    ├── oss_*             # 阿里云 OSS 配置
    ├── rss_*             # RSS 调度器配置
    ├── observer_*        # Observer 组件配置（5 个组件）
    ├── ollama_*          # Ollama 连接配置
    ├── middleware_*       # 认证中间件配置
    ├── state_store_*     # 状态存储配置
    ├── static_*          # 静态文件配置
    ├── pagination_*      # 分页配置
    ├── logging_*         # 日志配置
    └── module_allowlist  # 执行白名单
```

环境变量覆盖优先级：`init_settings → env_settings → dotenv_settings → file_secret_settings → yaml_settings`

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 架构模式 | src-layout 单体 | 项目规模适中，单体架构降低运维复杂度 |
| 数据库驱动 | motor 异步 | 与 FastAPI async/await 模型一致，无事件循环冲突 |
| 配置方案 | YAML + pydantic-settings | 类型安全 + 扁平化键名兼容环境变量覆盖 |
| 存储策略 | OSS + 本地双存储 | OSS 为主要存储，本地为降级兜底 |
| 认证方案 | X-Token 头验证 | 简单可靠，适合内部服务间认证 |
| AI 引擎 | Ollama 本地部署 | 数据不出域，降低 API 调用成本 |
| Observer | 可插拔组件 | 每个组件独立开关，故障隔离 |

## 退化对策

| 场景 | 处理方式 |
|------|---------|
| MongoDB 不可用 | 初始化失败，应用无法启动 |
| OSS 不可用 | 降级到本地存储（upload.py:147） |
| Observer 组件不可用 | 跳过注册，仅打印 warning |
| Token 缺失 | 降级跳过认证 |
| RSS 系统失败 | 不阻断服务器启动 |
| 脚本执行超时 | 杀死进程，返回错误信息 |
| Ollama 不可用 | 重试 2 次后返回错误 |
