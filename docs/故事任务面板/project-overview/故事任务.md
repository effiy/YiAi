# 项目总览

## 故事概述

建立 YiAi 项目的全景知识基线，覆盖项目画像、技术架构、模块地图、中间件链路、配置体系、数据存储、退化对策和安全面共 8 个维度，为所有子故事提供统一的系统上下文参照。

## 功能范围

### 项目画像
- 项目身份：YiAi（宜 AI），后端服务，Python / FastAPI，版本 1.0.0
- 仓库：git@github.com:effiy/YiAi.git
- 入口：`main.py`（生产入口）、`src/main.py`（应用工厂 create_app）
- 架构：src-layout 单体后端
- 数据库：MongoDB（motor 异步驱动）
- AI 引擎：Ollama（qwen3.5 / qwen3-vl）
- 存储：阿里云 OSS + 本地 static 目录（双存储，OSS 不可用时自动降级）

### 技术架构全景
- Nginx/CDN → FastAPI Application → MongoDB / Ollama / Aliyun OSS
- 中间件链路：CORS → Observer Throttle → Observer Sampler → Auth (X-Token) → Route Handler
- 7 个路由模块、9 个服务模块、5 个 Observer 组件

### 模块地图
- `src/api/routes/` — 7 个路由模块（execution / upload / wework / maintenance / state / observer_health / story_panel）
- `src/core/` — 核心基础设施（config / database / middleware / response / error_codes / exceptions / logger / utils / observer）
- `src/models/` — 数据模型（schemas / collections）
- `src/services/` — 业务服务层（ai / database / execution / rss / state / static / storage / maintenance）
- `src/cli/` — CLI 工具（state_query）

### 配置体系
- `config.yaml` 扁平化加载，覆盖 server / cors / mongodb / oss / rss / observer / ollama / middleware / state_store / static / pagination / logging / module_allowlist 共 13 个配置组
- 环境变量覆盖优先级：init_settings → env_settings → dotenv_settings → file_secret_settings → yaml_settings

### 数据存储
- 9 个 MongoDB 集合：sessions / rss / chat_records / pet_data_sync / seeds / oss_file_tags / oss_file_info / static_files / state_records
- 双存储策略：OSS + 本地 static 目录

### 退化对策
- MongoDB 不可用 → 应用无法启动
- OSS 不可用 → 降级到本地存储
- Observer 组件不可用 → 跳过注册，仅 warning
- Token 缺失 → 降级跳过认证
- RSS 系统失败 → 不阻断服务器启动
- Ollama 不可用 → 重试 2 次后返回错误

### 安全面
- API 认证：X-Token 请求头验证（可配置开关）
- 路径安全：realpath + commonpath 双重路径遍历防护
- 文件上传：扩展名白名单 + 大小限制 + ZIP 安全解压
- 模块执行：白名单 + 沙箱（文件系统/网络）+ 重入守卫
- 限流：基于 IP 的滑动窗口限流 + IP 白名单
- 错误处理：BusinessException + ErrorCode 枚举 + 全局异常捕获
- 输入校验：Pydantic 模型自动校验 + 自定义参数校验
