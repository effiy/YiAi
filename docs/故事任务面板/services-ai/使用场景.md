# AI 对话服务 — 用户使用场景

## 场景 1：文本对话

**参与者**：前端用户

**流程**：
1. 通过执行 API 调用：`POST /` body: `{"module_name": "services.ai.chat_service", "method_name": "chat", "parameters": {"user": "你好，请介绍一下自己"}}`
2. 服务使用默认模型 qwen3.5 调用 Ollama
3. 返回 AI 生成的文本响应
4. 可自定义 system 提示词和 model 参数

## 场景 2：流式对话

**参与者**：前端用户

**流程**：
1. 调用 chat 并设置 `"stream": true`
2. 服务返回异步生成器
3. 执行 API 以 SSE 格式逐 token 推送给前端
4. 前端实时渲染 AI 回复内容

## 场景 3：图片识别

**参与者**：前端用户

**流程**：
1. 用户上传图片并提问
2. 调用 chat 携带 `"images": ["data:image/png;base64,..."]` 或 HTTP URL
3. 服务检测到 images 参数，自动切换模型为 qwen3-vl
4. 提取用户消息中的纯文本部分（去除系统提示前缀）
5. 将图片和文本一起发送给 qwen3-vl
6. 返回图片识别/描述结果

## 场景 4：查询可用模型

**参与者**：运维人员

**流程**：
1. 调用 `list_ollama_models`
2. 返回 Ollama 服务端已安装的所有模型列表
3. 可用于前端模型选择器或运维检查
