# AI 对话服务 — 后端技术评审

## 架构设计

### 服务组件

```
OllamaService
├── ollama_url: str           # Ollama 服务地址
├── ollama_auth: Optional[str] # basic auth (username:password)
├── _get_client() → Client    # 创建 ollama 客户端
├── generate_response()       # 同步对话生成
└── list_models()             # 获取模型列表

chat(params) → Dict | AsyncGenerator
├── 解析参数 (system, user, model, stream, images)
├── 有图片 → 切换 qwen3-vl, 提取纯文本
├── 非流式 → run_in_executor(generate_response)
└── 流式 → async gen(Queue 跨线程传递)

_resolve_images(images) → List[bytes]
├── data: URL → base64 decode
├── HTTP URL → 并发下载 (Semaphore 4)
└── 限制 10MB/张
```

### 流式对话实现

```
chat(stream=True)
    ↓
创建 asyncio.Queue
    ↓
asyncio.to_thread(_worker)  # 线程中调用 ollama 流式 API
    ↓
_worker 将每个 token 通过 queue.put() 传递
    ↓
async gen 从 queue.get() 消费
    ↓
yield {"data": {"message": token}}
    ↓
queue.get() → None → 流结束
```

### 图片处理流程

```
_resolve_images(images: List[str])
    ├── 遍历每个 image
    │   ├── 空字符串 → 跳过
    │   ├── HTTP URL → 加入 http_urls 列表
    │   ├── data: URL → 提取 base64 → decode
    │   └── 纯 base64 → decode
    ├── http_urls 并发下载 (Semaphore 4)
    │   ├── Content-Type 校验 (必须 image/*)
    │   ├── 分块读取 256KB
    │   └── 最大 10MB
    └── 返回 List[bytes]
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 同步调用 | run_in_executor (线程池) | ollama 客户端是同步的，不阻塞事件循环 |
| 流式跨线程 | asyncio.Queue + to_thread | 同步流式 API 在子线程运行，token 异步传递 |
| 多模态模型 | qwen3-vl (硬编码) | 当前唯一支持的多模态模型 |
| 图片并发 | Semaphore(4) | 限制并发 HTTP 连接数 |
| 重试策略 | 最多 2 次重试 | 应对 Ollama 临时不可用 |

## 数据流

```
用户请求 → execute_module()
         → chat(params)
         → OllamaService._get_client()
         → ollama.chat(model, messages, stream)
         → 同步返回 / 流式 token
         → SSE 推送至客户端
```
