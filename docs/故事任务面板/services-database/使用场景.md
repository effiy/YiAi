# 数据库服务层 — 用户使用场景

## 场景 1：通用集合查询

**参与者**：前端/API 调用方

**流程**：
1. 通过执行 API 调用 `query_documents`：
   ```json
   {"cname": "rss", "pageNum": 1, "pageSize": 20, "orderBy": "createdTime", "orderType": "desc"}
   ```
2. 服务构建 MongoDB 查询条件（无额外过滤），按 createdTime 降序排列
3. 返回分页结果：list、total、pageNum、pageSize、totalPages

## 场景 2：模糊搜索查询

**参与者**：前端用户

**流程**：
1. 调用 query_documents 在 sessions 集合中搜索标题含 "YiAi" 的文档：
   ```json
   {"cname": "sessions", "filter": {"title": "YiAi"}}
   ```
2. build_filter 将 "YiAi" 转为正则 `.*YiAi.*`（大小写不敏感）
3. 自动排除 pageContent 字段（sessions 特有逻辑）
4. 返回匹配的文档列表

## 场景 3：多词搜索

**参与者**：前端用户

**流程**：
1. 调用 query_documents 搜索标题含 "AI" 或 "Python" 的文档：
   ```json
   {"cname": "sessions", "filter": {"title": "AI,Python"}}
   ```
2. build_filter 检测逗号分隔，构建 $or 条件：`[{"title": /AI/}, {"title": /Python/}]`

## 场景 4：日期范围查询

**参与者**：前端用户

**流程**：
1. 调用 query_documents 查询特定日期范围的 RSS 条目：
   ```json
   {"cname": "rss", "filter": {"isoDate": "2024-01-01,2024-01-31"}}
   ```
2. 服务构建多种日期格式的正则匹配和 ISO 日期精确匹配

## 场景 5：创建文档

**参与者**：内部服务

**流程**：
1. 调用 create_document 在 sessions 集合创建新文档
2. 系统自动分配 UUID4 key、设置 createdTime/updatedTime
3. 自动计算 order = max_order + 1
4. 返回生成的 key
