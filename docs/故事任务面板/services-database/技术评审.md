# 数据库服务层 — 后端技术评审

## 架构设计

### 服务层次

```
API Routes
    ↓
data_service.py (模块级函数) / MongoDBService (类封装)
    ↓
core/database.py (MongoDB 单例)
    ↓
motor (AsyncIOMotorClient)
    ↓
MongoDB
```

data_service.py 和 mongo_store.py 提供功能等效的两套接口：前者为模块级函数（供执行引擎调用），后者为类封装（供直接导入使用）。

### 查询过滤构建 (`_build_filter`)

```
输入 query_params
    ↓
遍历每个 key-value
    ├── key == "key" → 精确匹配
    ├── key == "isoDate" → 日期范围 → regex + $in
    ├── value 是列表 → 范围查询 ($gte/$lt) 或 $in
    ├── value 是字符串 → 逗号分隔多词 $or / 单词 regex
    └── value 是数字/布尔 → 精确匹配
    ↓
输出 MongoDB filter_dict
```

### 分页排序

```
pageNum (默认 1, 最小 1)
pageSize (默认 2000, 最小 1, 最大 8000)
orderBy (默认 'order', apis 集合默认 'timestamp')
orderType (默认 'asc')
    ↓
sort_list = [(orderBy, order), (updatedTime, -1), (createdTime, -1)]
    ↓
cursor.skip((pageNum-1)*pageSize).limit(pageSize)
```

### sessions 集合特殊处理

- 查询时默认排除 pageContent（大字段，可能包含 HTML/富文本）
- 创建/更新时自动移除 pageContent
- 空 messages 数组保护（不覆盖已有数据）

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 双接口 | 模块级函数 + 类封装 | 兼容执行引擎调用和直接导入 |
| 过滤语法 | 自定义参数 → MongoDB query | 灵活但需注意注入风险 |
| 日期查询 | 多格式正则 | 兼容不同的日期存储格式 |
| 分页上限 | 8000 | 平衡查询性能和数据完整性 |
| 排序默认 | order ASC + 双时间戳 DESC | 保证分页稳定性 |

## 数据流

```
query_documents(params)
    ├── 提取 collection_name/cname
    ├── 提取 fields/excludeFields
    ├── 提取分页参数 (pageNum, pageSize)
    ├── 提取排序参数 (orderBy, orderType)
    ├── _build_filter(剩余参数)
    ├── db[collection].find(filter, projection).sort().skip().limit()
    └── 返回 {list, total, pageNum, pageSize, totalPages}
```
