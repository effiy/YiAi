# 数据库服务层

## 故事概述

在 MongoDB 单例连接之上构建通用数据服务层，提供多集合的 CRUD 操作、灵活查询过滤、分页排序和 upsert 能力。是 API 路由和业务服务访问数据库的统一入口。

## 功能范围

### 通用数据服务 (`services/database/data_service.py`)

- **query_documents()**：通用文档查询
  - 支持 cname/collection_name 双参数
  - 灵活的 filter 构建：精确匹配 key、isoDate 范围、数值范围、字符串模糊搜索（逗号分隔多词 OR）
  - 字段筛选：fields（包含）和 excludeFields（排除）
  - 分页：pageNum/pageSize（兼容旧 page/limit 参数）
  - 排序：orderBy + orderType（默认 order ASC + updatedTime DESC + createdTime DESC）
  - sessions 集合自动排除 pageContent 大字段

- **get_document_detail()**：根据 key 获取单文档详情

- **create_document()**：创建文档
  - 自动生成 key（UUID4）、createdTime、updatedTime
  - 自动计算 order 排序值（当前最大 order + 1）
  - rss 集合 link 唯一性校验

- **update_document()**：更新文档
  - 通过 key 定位，支持部分更新
  - 保护 createdTime 不被覆盖
  - sessions 集合空 messages 保护

- **upsert_document()**：upsert 操作
  - 支持 $set / $setOnInsert 原子操作
  - 自动注入时间戳和 key

- **delete_document()**：按 key 删除

- **list_story_task_dirs()**：聚合查询故事任务目录（按 projectName + storyName 分组）

### MongoDB 服务封装 (`services/database/mongo_store.py`)

- **MongoDBService 类**：面向对象的数据库服务封装
  - 提供与 data_service.py 等效的功能，但以类方法组织
  - 额外的工具方法：is_valid_date、parse_published_date、build_published_date_filter、build_filter
