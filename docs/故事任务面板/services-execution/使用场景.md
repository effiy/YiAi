# 模块执行引擎 — 用户使用场景

## 场景 1：通过 API 执行服务函数

**参与者**：前端应用

**流程**：
1. 发送 `POST /` body: `{"module_name": "services.ai.chat_service", "method_name": "chat", "parameters": {"user": "你好"}}`
2. execute_module 接收参数
3. 检查白名单：`services.ai.chat_service:chat` 是否在白名单中（或 `*` 全允许）
4. 解析 JSON 参数 → dict
5. 动态导入 `services.ai.chat_service` 模块，获取 `chat` 函数
6. 检测到 `chat` 是 async function，await 执行
7. 记录执行结果到 SkillRecorder
8. 返回结果给路由层

## 场景 2：流式响应执行

**参与者**：前端应用

**流程**：
1. 调用 chat 的 stream 模式
2. execute_module 检测到返回值是 async generator
3. 不做 await，直接返回生成器对象给路由层
4. 路由层以 SSE 格式流式推送

## 场景 3：脚本超时终止

**参与者**：运维人员

**流程**：
1. 调用 run_script("/path/to/long_script.py", timeout=30)
2. asyncio.create_subprocess_exec 启动子进程
3. 30 秒后仍未完成，process.kill()
4. 返回超时错误信息

## 场景 4：重入阻止

**参与者**：系统

**流程**：
1. 模块 A 调用 execute_module → 深度 = 1
2. 模块 A 内部再次调用 execute_module → 深度 = 2
3. 模块 B 内部再次调用 → 深度 = 3
4. 模块 C 内部再次调用 → 深度 = 4 > max_depth=3
5. 抛出 BusinessException: "Reentrancy depth 4 exceeds limit 3"
