# 模块执行引擎 — 安全审计

## 审计范围

- `src/services/execution/executor.py` — 受控模块执行器

## 安全发现

### [P0] 白名单配置 `*` 通配风险

- **发现**：`module_allowlist` 默认为 `["*"]`，允许执行任何模块的任何函数
- **风险**：攻击者可调用任意 Python 函数，包括 `os.system`、`subprocess.run` 等危险函数，实现远程代码执行（RCE）
- **建议**：
  - 生产环境必须将 `*` 改为显式的模块:函数白名单
  - 增加危险模块黑名单（os、subprocess、sys、shutil 等）
  - 增加函数签名校验，限制参数类型

### [P0] 动态导入任意模块

- **发现**：`importlib.import_module(module_path)` 无路径限制
- **风险**：结合 `*` 白名单，可导入并执行任意 Python 标准库和第三方库
- **建议**：白名单之外增加模块路径前缀限制（如只允许 `services.*`）

### [P0] 脚本执行

- **发现**：`run_script()` 执行任意 Python 脚本文件
- **风险**：如果脚本路径可控，可执行任意代码
- **建议**：限制脚本路径在特定目录下，增加脚本内容预扫描

### [P2] 参数注入

- **发现**：解析后的 parameters 直接传入目标函数 `target_function(parameters_dict)`
- **风险**：恶意参数可触发目标函数中的意外行为
- **建议**：对每个白名单函数定义参数 schema，执行前校验

### [P2] 执行记录敏感信息泄露

- **发现**：`_record_execution` 记录参数和输出摘要（截断 500 字符）
- **风险**：可能记录密码、Token 等敏感参数
- **建议**：增加敏感字段过滤（password、token、secret 等）

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 代码执行白名单 | 风险 | 默认 `*` 允许所有 |
| 模块导入限制 | 缺失 | 无路径前缀限制 |
| 脚本执行控制 | 部分 | 有超时，无路径限制 |
| 参数校验 | 缺失 | 无函数级参数 schema |
| 沙箱隔离 | 可选 | 默认关闭 |
| 重入保护 | 已实施 | max_depth=3 |

## 审计结论

模块执行引擎是系统安全的关键节点。当前默认配置 `*` 白名单存在严重的 RCE 风险（P0），必须修改为显式白名单。建议增加多层防护：白名单精确匹配 + 模块路径前缀限制 + 危险模块黑名单 + 函数参数 schema 校验，形成纵深防御。
