# 模块执行引擎 — 测试用例评审

## 测试用例

### TC-EX-001：白名单 — 允许的函数

| 项目 | 内容 |
|------|------|
| 目标 | 白名单中的函数正常执行 |
| 前置 | module_allowlist=["*"] 或包含目标函数 |
| 步骤 | execute_module("services.rss.rss_scheduler", "get_scheduler_status_info", {}) |
| 预期 | 正常返回调度器状态 |

### TC-EX-002：白名单 — 拒绝的函数

| 项目 | 内容 |
|------|------|
| 目标 | 非白名单函数被拒绝 |
| 前置 | module_allowlist=["services.rss.rss_scheduler:get_scheduler_status_info"] |
| 步骤 | execute_module("services.ai.chat_service", "chat", {"user": "hi"}) |
| 预期 | 抛出 PERMISSION_DENIED 异常 |

### TC-EX-003：白名单 — * 通配全允许

| 项目 | 内容 |
|------|------|
| 目标 | * 通配符允许所有函数 |
| 前置 | module_allowlist=["*"] |
| 步骤 | 执行任意模块/函数 |
| 预期 | 全部正常执行 |

### TC-EX-004：JSON 参数解析

| 项目 | 内容 |
|------|------|
| 目标 | JSON 字符串正确解析为 dict |
| 步骤 | parse_parameters('{"key": "value"}') |
| 预期 | 返回 {"key": "value"} |

### TC-EX-005：无效 JSON 参数

| 项目 | 内容 |
|------|------|
| 目标 | 无效 JSON 抛出异常 |
| 步骤 | parse_parameters("not json") |
| 预期 | 抛出 INVALID_PARAMS 异常 |

### TC-EX-006：非 dict JSON 参数

| 项目 | 内容 |
|------|------|
| 目标 | JSON 数组被拒绝 |
| 步骤 | parse_parameters("[1, 2, 3]") |
| 预期 | 抛出 "Parameters must be a JSON object" 异常 |

### TC-EX-007：模块不存在

| 项目 | 内容 |
|------|------|
| 目标 | 不存在的模块返回错误 |
| 步骤 | execute_module("nonexistent.module", "func", {}) |
| 预期 | 抛出 "Module or function not found" 异常 |

### TC-EX-008：函数不存在

| 项目 | 内容 |
|------|------|
| 目标 | 模块存在但函数不存在 |
| 步骤 | execute_module("services.ai.chat_service", "nonexistent", {}) |
| 预期 | 抛出 "Module or function not found" 异常 |

### TC-EX-009：脚本执行超时

| 项目 | 内容 |
|------|------|
| 目标 | 超时后进程被杀死 |
| 前置 | 准备无限循环脚本 |
| 步骤 | run_script("infinite_loop.py", timeout=1) |
| 预期 | 1 秒后返回 success:False, message 含"超时" |

### TC-EX-010：重入守卫 — 正常深度

| 项目 | 内容 |
|------|------|
| 目标 | 深度 ≤ max_depth 正常执行 |
| 前置 | max_depth=3, 当前深度=2 |
| 步骤 | execute_module(...) |
| 预期 | 正常执行，深度变为 3 |

### TC-EX-011：重入守卫 — 超限

| 项目 | 内容 |
|------|------|
| 目标 | 深度 > max_depth 抛出异常 |
| 前置 | max_depth=3, 当前深度=3 |
| 步骤 | execute_module(...) |
| 预期 | 抛出 "Reentrancy depth 3 exceeds limit 3" |
