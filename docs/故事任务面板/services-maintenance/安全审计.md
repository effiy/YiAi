# 维护服务 — 安全审计

## 审计范围

- `src/services/maintenance/session_service.py` — Sessions 维护服务

## 安全发现

### [P2] 全量查询内存风险

- **发现**：`get_all_sessions()` 无分页全量返回
- **风险**：若 sessions 集合极大，可能导致内存溢出（OOM）
- **建议**：增加分页支持或游标式遍历

### [P2] 无删除确认机制

- **发现**：`delete_session_by_key` 直接删除，无二次确认
- **风险**：误调用可能导致数据丢失
- **建议**：在 API 层增加 dry_run 机制（已在 maintenance.py 中实现）

### [INFO] 现有安全控制

- maintenance.py API 层已有 dry_run 默认 true
- cleanup_sessions 为可选功能（需显式传 cleanup_sessions=true）
- 这些控制在 API 层而非服务层，服务层应保持简单

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 数据防丢失 | 通过 | API 层 dry_run 控制 |
| 资源限制 | 风险 | 全量查询无上限 |
| 访问控制 | 间接 | 依赖认证中间件 |

## 审计结论

维护服务层安全风险较低。主要关注点：全量查询可能在大数据集下导致 OOM，建议增加分页或上限控制。API 层的 dry_run 机制有效防止误删除。
