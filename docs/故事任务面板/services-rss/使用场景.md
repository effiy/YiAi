# RSS 订阅服务 — 用户使用场景

## 场景 1：手动解析 RSS 源

**参与者**：用户/自动化流程

**流程**：
1. 通过执行 API 调用：
   `POST /` body: `{"module_name": "services.rss.feed_service", "method_name": "parse_feed", "parameters": {"url": "https://example.com/feed.xml", "name": "技术博客"}}`
2. 服务异步获取 RSS 源（60 秒超时）
3. 流式读取内容，限制 10MB
4. feedparser 解析条目
5. 逐条检查 link 是否已存在 → 新增或更新
6. 返回入库统计：saved_count、updated_count、total_items

## 场景 2：定时自动采集

**参与者**：系统

**前置条件**：seeds 集合中已配置 RSS 源

**流程**：
1. 应用启动 → init_rss_system() → 按配置间隔启动调度器
2. 调度器触发 → 从 seeds 集合查询启用的源
3. 并发解析（最多 3 个同时）
4. 入库去重
5. 记录日志

## 场景 3：动态调整调度频率

**参与者**：运维人员

**流程**：
1. 调用 set_scheduler_config：
   ```json
   {"config": {"type": "interval", "interval": 7200}}
   ```
2. 调度器停止当前任务
3. 按新间隔（7200 秒 = 2 小时）重新启动
4. 或切换为 Cron 模式：`"type": "cron", "cron": {"hour": 8, "minute": 0}`

## 场景 4：查看调度器状态

**参与者**：运维人员

**流程**：
1. 调用 `get_scheduler_status_info`
2. 返回：enabled、type、interval、cron 配置
