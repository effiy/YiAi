# RSS 订阅服务 — 安全审计

## 审计范围

- `src/services/rss/feed_service.py` — RSS 源获取与解析
- `src/services/rss/rss_scheduler.py` — 定时调度器

## 安全发现

### [P1] SSRF 风险 — RSS URL

- **发现**：`fetch_rss_feed(url)` 对用户提供的 URL 发起 HTTP 请求，无内网过滤
- **风险**：攻击者可提供内网地址探测内部服务（如 `http://localhost:6379` 探测 Redis）
- **建议**：增加 DNS 解析 + IP 校验，拒绝内网/保留 IP 段

### [P2] XML 外部实体注入 (XXE)

- **发现**：feedparser 用于解析 XML，需确认是否防御 XXE 攻击
- **风险**：恶意 RSS 源可能包含 XXE payload，读取服务器文件或发起 SSRF
- **建议**：确认 feedparser 版本是否禁用外部实体解析；测试 XXE payload

### [P2] RSS 内容注入

- **发现**：RSS 条目内容直接存入 MongoDB，未经 HTML 清洗
- **风险**：恶意 RSS 源可注入 XSS payload，前端展示时可能触发
- **建议**：对 description/content 字段做 HTML 标签过滤或转义

### [P2] 资源耗尽 — 大 RSS 源

- **发现**：已限制 10MB 上限和 60 秒超时
- **验证**：流式读取 + 大小检查，防护到位
- **结论**：当前防护有效

### [P2] 调度器 DoS

- **发现**：调度器配置通过 API 可动态修改，interval 最小 60 秒
- **风险**：若最小限制被绕过或修改，高频解析可能耗尽资源
- **建议**：增加速率限制的下限校验（当前已有 ≥60 秒检查）

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| SSRF 防护 | 缺失 | URL 无内网过滤 |
| XXE 防护 | 待确认 | 需确认 feedparser 安全配置 |
| XSS 防护 | 缺失 | RSS 内容未清洗 |
| 资源限制 | 通过 | 10MB + 60s 超时 |
| 频率限制 | 通过 | interval ≥ 60s |

## 审计结论

RSS 订阅服务存在 SSRF 和 XXE 两个需要关注的安全风险。建议优先增加 URL 的 DNS 解析 + IP 校验防止内网探测，并确认 feedparser 的 XXE 防护状态。
