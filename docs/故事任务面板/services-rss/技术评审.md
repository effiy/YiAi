# RSS 订阅服务 — 后端技术评审

## 架构设计

### RSS 处理流水线

```
RSS URL
    ↓
fetch_rss_feed(url)
    ├── aiohttp GET (60s timeout)
    ├── Content-Length 预检 (>10MB → reject)
    ├── 流式读取 (8192 bytes/chunk)
    ├── 实际大小检查 (>10MB → reject)
    └── feedparser.parse(bytes)
    ↓
process_feed_from_url(url, name)
    ├── 提取 source_name (name or feed.title)
    ├── 遍历 feed.entries
    │   ├── 无 link → skip
    │   ├── _build_entry_data() → 构建文档
    │   └── _save_or_update_entry() → 去重入库
    └── gc.collect() 释放内存
```

### 调度器架构

```
RSSSchedulerManager (Singleton)
├── _scheduler: AsyncIOScheduler
├── _running: bool
├── _config: {type, interval, cron}
│
├── start()
│   ├── 创建 IntervalTrigger / CronTrigger
│   ├── add_job(_scheduler_job)
│   └── scheduler.start()
│
├── stop()
│   ├── scheduler.shutdown(wait=False)
│   └── _scheduler = None
│
├── parse_all_sources()
│   ├── _get_enabled_sources() → seeds 集合
│   ├── asyncio.gather(*[worker(src) for src in sources])
│   └── Semaphore(3) 并发控制
│
└── set_config(config)
    ├── 验证参数
    ├── stop() → start()
    └── 无缝切换
```

### 去重策略

```
_save_or_update_entry(collection, item_data)
    ├── find_one({'link': item_data['link']})
    ├── 已存在 → update_one (保留原 key 和 createdTime)
    └── 不存在 → insert_one (生成新 key 和 UUID4)
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| RSS 解析库 | feedparser | Python RSS 解析标准库 |
| 去重字段 | link | RSS 条目标准唯一标识 |
| 调度器 | APScheduler (AsyncIOScheduler) | 支持 Cron/Interval，异步友好 |
| 并发解析 | Semaphore(3) | 限制同时解析数，避免资源争抢 |
| 源配置存储 | seeds 集合 | 动态管理，无需重启 |
| 内存管理 | gc.collect() | feedparser 内存占用大，手动回收 |

## 数据流

```
seeds 集合 (RSS 源配置)
    ↓
RSSSchedulerManager.parse_all_sources()
    ↓ (Semaphore 3)
process_feed_from_url(url, name)
    ↓
fetch_rss_feed(url) → feedparser
    ↓
MongoDB rss 集合 (按 link 去重 upsert)
```
