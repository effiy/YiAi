# RSS 订阅服务

## 故事概述

构建 RSS 源订阅与定时采集系统，支持从外部 RSS 源获取内容、解析条目、去重存储到 MongoDB，并提供基于 APScheduler 的定时调度能力。

## 功能范围

### RSS 源获取与解析 (`services/rss/feed_service.py`)

- **fetch_rss_feed()**：异步获取并解析 RSS 源
  - aiohttp 流式读取，限制最大 10MB
  - Content-Length 预检，超过限制提前拒绝
  - feedparser 解析，bozo 警告容错

- **process_feed_from_url()**：完整处理流程
  - 获取 → 解析 → 入库
  - 按 link 去重（已存在则更新，不存在则新增）
  - 自动提取 source_name、tags
  - 完成后 gc.collect() 释放 feedparser 内存

- **parse_feed()**：通过执行引擎调用的 API 接口
  - 参数校验（url 必填）
  - 返回结构化结果：success、url、source、saved_count、updated_count、total_items

### RSS 定时调度 (`services/rss/rss_scheduler.py`)

- **RSSSchedulerManager**：调度器管理单例
  - 支持 Interval 和 Cron 两种触发模式
  - 从 seeds 集合读取启用的 RSS 源配置
  - 并发解析（Semaphore 3）
  - 动态配置切换（set_config → stop → start）

- **模块级函数**（向后兼容）：
  - `start_rss_scheduler()` / `stop_rss_scheduler()`
  - `set_scheduler_config()` — 动态修改调度配置
  - `get_scheduler_status_info()` — 查询运行状态
  - `parse_all_enabled_rss_sources()` — 手动触发全量解析

- **生命周期集成**：
  - `init_rss_system()` — 应用启动时自动启动调度器
  - `shutdown_rss_system()` — 应用关闭时停止调度器

## 验收标准
- [ ] RSS 源正确获取并解析条目
- [ ] 重复 link 条目正确去重更新
- [ ] 大 RSS 源（>10MB）被拒绝
- [ ] 定时调度器按配置周期执行
- [ ] Cron 和 Interval 模式均正常工作
- [ ] 调度器配置可动态切换
