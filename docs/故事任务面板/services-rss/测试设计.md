# RSS 订阅服务 — 测试用例评审

## 测试用例

### TC-RS-001：RSS 源正常解析

| 项目 | 内容 |
|------|------|
| 目标 | 有效 RSS 源正常解析并入库 |
| 前置 | 准备测试用 RSS XML |
| 步骤 | process_feed_from_url("http://test.com/feed.xml", "Test") |
| 预期 | 返回 success:true，saved_count ≥ 0 |

### TC-RS-002：大 RSS 源拒绝

| 项目 | 内容 |
|------|------|
| 目标 | 超过 10MB 的源被拒绝 |
| 步骤 | 请求 Content-Length > 10MB 的 URL |
| 预期 | 抛出 Invalid Params 异常 |

### TC-RS-003：重复 link 去重更新

| 项目 | 内容 |
|------|------|
| 目标 | 相同 link 的条目更新而非新增 |
| 前置 | rss 集合已有 link="http://a.com/1" |
| 步骤 | 再次解析含相同 link 的 RSS |
| 预期 | updated_count=1, saved_count=0 |

### TC-RS-004：调度器 Interval 模式

| 项目 | 内容 |
|------|------|
| 目标 | Interval 调度器按周期触发 |
| 前置 | seeds 集合有启用的源 |
| 步骤 | 1. 设置 interval=60 2. 启动调度器 3. 等待 60 秒 |
| 预期 | 调度器至少触发一次解析 |

### TC-RS-005：调度器 Cron 模式

| 项目 | 内容 |
|------|------|
| 目标 | Cron 调度器按时触发 |
| 步骤 | 设置 type=cron, cron={"minute": 0} |
| 预期 | 每小时整点触发 |

### TC-RS-006：动态切换配置

| 项目 | 内容 |
|------|------|
| 目标 | set_config 无缝切换调度 |
| 前置 | 调度器正在 Interval 60s 模式运行 |
| 步骤 | set_config({"type": "interval", "interval": 120}) |
| 预期 | 调度器以新间隔 120s 运行 |

### TC-RS-007：无效 URL

| 项目 | 内容 |
|------|------|
| 目标 | 无效 URL 返回错误 |
| 步骤 | parse_feed({"url": "not-a-url"}) |
| 预期 | 返回 success:false, 包含 error |

### TC-RS-008：并发解析限制

| 项目 | 内容 |
|------|------|
| 目标 | 最多 3 个 RSS 源同时解析 |
| 前置 | seeds 集合有 10 个启用源 |
| 步骤 | parse_all_sources() → 检查日志 |
| 预期 | 同一时间最多 3 个 fetch 在执行 |

### TC-RS-009：空 seeds 集合

| 项目 | 内容 |
|------|------|
| 目标 | 无启用源时正常返回 |
| 步骤 | seeds 集合无启用源 → parse_all_sources() |
| 预期 | total_sources=0, success_count=0 |
