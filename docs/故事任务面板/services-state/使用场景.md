# 状态存储服务 — 用户使用场景

## 场景 1：记录 API 执行历史

**参与者**：模块执行引擎

**流程**：
1. execute_module 执行完成（成功或失败）
2. 调用 SkillRecorder.record_async() 记录：
   - skill_name: "services.ai.chat_service:chat"
   - status: "success" / "failed"
   - duration_ms: 1234
   - input_summary: 截断至 500 字符的参数
   - output_summary: 截断至 500 字符的结果
   - error_message: 错误信息（失败时）
3. 异步写入 state_records 集合
4. 记录失败不影响主流程

## 场景 2：查询执行历史

**参与者**：运维人员/前端

**流程**：
1. 通过 state API 查询：
   `GET /state/records?record_type=skill_execution&page_num=1&page_size=20`
2. 或查询特定标签的记录：`?tags=AI&tags=chat`
3. 或按标题模糊搜索：`?title_contains=Ollama`
4. 返回分页的最近记录（按 created_time 降序）

## 场景 3：更新状态记录

**参与者**：内部服务

**流程**：
1. 通过 state API：`PUT /state/records/{key}`
2. 只更新传入的字段（exclude_unset=True）
3. 不覆盖 key 和 created_time
4. 自动更新 updated_time

## 场景 4：删除过期记录

**参与者**：运维人员

**流程**：
1. 通过 state API：`DELETE /state/records/{key}`
2. 记录被删除
3. 若记录不存在返回 404
