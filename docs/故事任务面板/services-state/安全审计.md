# 状态存储服务 — 安全审计

## 审计范围

- `src/services/state/state_service.py` — 状态存储服务
- `src/services/state/skill_recorder.py` — 技能记录器
- `src/services/state/session_adapters.py` — 会话适配器

## 安全发现

### [P2] 标题模糊搜索 — ReDoS 风险

- **发现**：`title_contains` 参数使用 `$regex: ".*<input>.*"` 做模糊搜索，未使用 `re.escape()`
- **风险**：用户输入包含正则特殊字符（如 `.*`、`(a+)+$`）可能触发灾难性回溯（ReDoS）
- **建议**：使用 `re.escape(title_contains)` 转义特殊字符后再构建正则

### [P2] 记录数据无结构校验

- **发现**：`create()` 和 `update()` 的 `record` 参数为 `Dict[str, Any]`，无 schema 校验
- **风险**：可写入任意结构的 JSON，包括超大嵌套或恶意内容
- **建议**：对 data 字段大小做限制（如最大 1MB），对嵌套深度做限制

### [P2] 无访问控制

- **发现**：状态记录 CRUD 无权限控制，任何通过认证的调用方可读写任意记录
- **风险**：敏感状态数据可能被越权访问或篡改
- **建议**：增加记录所有权或命名空间隔离

### [INFO] 敏感数据记录

- **发现**：SkillRecorder 记录参数和输出摘要（截断 500 字符），可能包含敏感信息
- **建议**：确认截断逻辑不泄露 Token、密码等敏感信息

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 输入校验 | 部分 | 无数据大小和结构限制 |
| ReDoS 防护 | 缺失 | title_contains 正则未转义 |
| 访问控制 | 缺失 | 无记录级权限 |
| 敏感数据保护 | 部分 | 截断 500 字符 |
| 数据完整性 | 通过 | key/created_time 保护 |

## 审计结论

状态存储服务安全风险较低。主要改进点：对 title_contains 参数做 re.escape 转义防止 ReDoS，增加数据大小和嵌套深度限制。
