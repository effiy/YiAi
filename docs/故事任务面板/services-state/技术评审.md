# 状态存储服务 — 后端技术评审

## 架构设计

### StateStoreService

```
StateStoreService
├── collection_name: str     # 从 config 读取
├── max_limit: int           # 查询最大条数
│
├── create(record) → {key}
│   ├── 自动生成 key (UUID4)
│   ├── 注入 created_time / updated_time
│   └── insert_one()
│
├── query(...) → {list, total, pageNum, pageSize, totalPages}
│   ├── 构建 filter_dict
│   │   ├── record_type → 精确匹配
│   │   ├── tags → $in
│   │   ├── title_contains → $regex (大小写不敏感)
│   │   └── created_after/before → $gte/$lt
│   ├── sort: created_time DESC
│   ├── skip + limit 分页
│   └── count_documents 总数
│
├── get(key) → Optional[Dict]
│   └── find_one({"key": key})
│
├── update(key, data) → {key, updated}
│   ├── 移除 key 和 created_time
│   ├── 注入 updated_time
│   └── update_one → matched_count == 0 → ValueError
│
└── delete(key) → {key, deleted}
    └── delete_one → deleted_count == 0 → ValueError
```

### SkillRecorder

```
SkillRecorder.record_async(
    skill_name: str,
    status: str,
    duration_ms: float,
    input_summary: str,
    output_summary: str,
    error_message: str
)
    ↓
StateStoreService.create({
    "record_type": "skill_execution",
    "title": skill_name,
    "tags": [status, skill_name.split(":")[0]],
    "data": {status, duration_ms, input_summary, output_summary, error_message}
})
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 存储引擎 | MongoDB (motor) | 与项目统一，JSON 文档灵活 |
| Key 生成 | UUID4 | 全局唯一，无碰撞风险 |
| 排序 | created_time DESC | 最近优先 |
| TTL | 可选（默认 0 = 不过期） | MongoDB TTL 索引可按需启用 |
| 查询限制 | max 8000 条 | 平衡内存和完整性 |
| 记录器 | best-effort 异步 | 不影响主流程 |

## 数据流
