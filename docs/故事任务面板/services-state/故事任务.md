# 状态存储服务

## 故事概述

构建结构化的状态存储服务，提供 key-value 风格的状态记录 CRUD 能力（按类型、标签、标题、时间范围查询），为 SkillRecorder 和 Session 适配器提供底层存储支持。

## 功能范围

### 状态存储服务 (`services/state/state_service.py`)

- **StateStoreService 类**：核心状态存储引擎
  - `create(record)`：创建状态记录，自动生成 UUID4 key，自动注入时间戳
  - `query(...)`：多条件分页查询
    - 筛选条件：record_type、tags（$in）、title_contains（模糊搜索）、时间范围
    - 分页：pageNum/pageSize（受 max_limit 限制）
    - 排序：created_time DESC
  - `get(key)`：按 key 获取单条记录
  - `update(key, data)`：部分更新，保护 key/created_time
  - `delete(key)`：按 key 删除

- **配置项**：
  - `state_store_enabled`：总开关
  - `state_store_default_ttl`：默认 TTL（0 = 永不过期）
  - `state_store_query_max_limit`：查询最大条数（默认 8000）
  - `collection_state_records`：集合名（默认 "state_records"）

### 技能记录器 (`services/state/skill_recorder.py`)
- 记录模块执行的历史：技能名、状态、耗时、输入/输出摘要、错误信息

### 会话适配器 (`services/state/session_adapters.py`)
- 将状态存储服务适配为 Session 存储接口
