# 状态存储服务 — 测试用例评审

## 测试用例

### TC-ST-001：创建状态记录

| 项目 | 内容 |
|------|------|
| 目标 | 创建记录返回 key |
| 步骤 | create({"record_type": "test", "title": "Hello"}) |
| 预期 | 返回 {"key": "<uuid>"}，文档含 key、created_time、updated_time |

### TC-ST-002：按类型查询

| 项目 | 内容 |
|------|------|
| 目标 | 按 record_type 过滤 |
| 前置 | 创建 3 条 type=A，2 条 type=B |
| 步骤 | query(record_type="A") |
| 预期 | list 长度 3，total=3 |

### TC-ST-003：按标签查询

| 项目 | 内容 |
|------|------|
| 目标 | 按 tags $in 过滤 |
| 前置 | 创建带标签的记录 |
| 步骤 | query(tags=["success"]) |
| 预期 | 返回所有 tags 含 "success" 的记录 |

### TC-ST-004：模糊搜索标题

| 项目 | 内容 |
|------|------|
| 目标 | title_contains 模糊匹配 |
| 前置 | 创建 title="Ollama Chat Service" |
| 步骤 | query(title_contains="Ollama") |
| 预期 | 返回该记录 |

### TC-ST-005：时间范围查询

| 项目 | 内容 |
|------|------|
| 目标 | created_after/before 范围过滤 |
| 步骤 | query(created_after="2024-01-01", created_before="2024-12-31") |
| 预期 | 返回该时间范围内的记录 |

### TC-ST-006：分页

| 项目 | 内容 |
|------|------|
| 目标 | 分页正确计算 totalPages |
| 前置 | 创建 25 条记录 |
| 步骤 | query(page_num=1, page_size=10) |
| 预期 | list 长度 10，total=25，totalPages=3 |

### TC-ST-007：获取单条记录

| 项目 | 内容 |
|------|------|
| 目标 | 按 key 获取 |
| 前置 | 创建记录 key="test-123" |
| 步骤 | get("test-123") |
| 预期 | 返回完整记录 |

### TC-ST-008：获取不存在的记录

| 项目 | 内容 |
|------|------|
| 目标 | 不存在的 key 返回 None |
| 步骤 | get("nonexistent") |
| 预期 | 返回 None |

### TC-ST-009：更新记录

| 项目 | 内容 |
|------|------|
| 目标 | 部分字段更新 |
| 前置 | 创建记录 |
| 步骤 | update(key, {"title": "Updated"}) |
| 预期 | title 更新，key 不变，created_time 不变，updated_time 更新 |

### TC-ST-010：删除记录

| 项目 | 内容 |
|------|------|
| 目标 | 按 key 删除 |
| 前置 | 创建记录 |
| 步骤 | delete(key) → 再次 get(key) |
| 预期 | delete 返回 {deleted:true}，get 返回 None |

### TC-ST-011：更新不存在的记录

| 项目 | 内容 |
|------|------|
| 目标 | 更新不存在的 key 抛出 ValueError |
| 步骤 | update("nonexistent", {"title": "test"}) |
| 预期 | 抛出 ValueError |
