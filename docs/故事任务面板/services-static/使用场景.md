# 静态文件管理 — 用户使用场景

## 场景 1：上传 ZIP 部署静态网站

**参与者**：前端用户

**流程**：
1. 用户准备一个 ZIP 文件（如 `mysite.zip`），内部结构为 `mysite/index.html`, `mysite/style.css`
2. 上传 ZIP 到文件管理接口
3. `upload_and_unzip` 接收文件
4. 检测 ZIP 内所有文件在 `mysite/` 根目录下 → 自动剥离公共前缀
5. 解压到 `static/` 目录（如指定 project_id 则到 `static/<project_id>/`）
6. 返回解压统计：文件数、目录数

## 场景 2：路径遍历攻击防护

**参与者**：攻击者

**流程**：
1. 攻击者构造恶意 ZIP，内含 `../../../etc/passwd` 路径
2. `_is_safe_path` 检测到 `..` → 拒绝该文件
3. 该文件被跳过，不写入磁盘
4. 其他正常文件正确解压

## 场景 3：通过 API 解压本地文件

**参与者**：运维人员

**流程**：
1. 将 ZIP 文件上传到服务器 `/tmp/data.zip`
2. 通过执行 API 调用：
   ```json
   {"module_name": "services.static.archive_service", "method_name": "unzip_from_path", "parameters": {"file_path": "/tmp/data.zip", "project_id": "myproject"}}
   ```
3. 文件解压到 `static/myproject/`
