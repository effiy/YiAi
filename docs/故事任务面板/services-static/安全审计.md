# 静态文件管理 — 安全审计

## 审计范围

- `src/services/static/static_files.py` — ZIP 上传与解压
- `src/services/static/archive_service.py` — 本地 ZIP 解压

## 安全发现

### [P0] ZipSlip 路径遍历

- **发现**：`_is_safe_path()` 对每个 ZIP 条目进行路径安全校验
- **验证**：检查 `..`、绝对路径、`os.path.normpath` 后的前缀匹配
- **结论**：防护到位，符合 OWASP ZipSlip 防护标准

### [P2] 临时文件竞争

- **发现**：`NamedTemporaryFile(delete=False)` 写入后使用，存在 TOCTOU 窗口
- **风险**：在写入临时文件和读取解压之间，另一个进程可能修改临时文件
- **建议**：在写入后立即校验文件完整性（如 hash 比对）

### [P2] ZIP 炸弹

- **发现**：仅检查 ZIP 文件大小，未检查解压后总大小或压缩比
- **风险**：高压缩比的 ZIP 炸弹（如 1MB → 100GB）可耗尽磁盘
- **建议**：限制解压后总大小、限制压缩比（如 > 100:1 拒绝）

### [P2] 符号链接攻击

- **发现**：ZipFile.extractall 默认不保留符号链接，但需确认
- **风险**：恶意 ZIP 包含符号链接可能指向系统关键文件
- **建议**：显式禁用符号链接提取，仅提取普通文件

### [P2] unzip_from_path 路径任意访问

- **发现**：`unzip_from_path` 接受任意本地路径 `file_path`
- **风险**：如果调用者可控制 file_path，可解压系统任意 ZIP 文件
- **建议**：限制 file_path 在特定目录下

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| ZipSlip 防护 | 通过 | `..` 和绝对路径检测 |
| 文件类型校验 | 通过 | 仅允许 .zip |
| 文件大小限制 | 通过 | MAX_ZIP_SIZE |
| ZIP 炸弹防护 | 缺失 | 未检查解压后大小 |
| 符号链接防护 | 待确认 | 需确认 ZipFile 行为 |
| 临时文件安全 | 部分 | 存在 TOCTOU 窗口 |

## 审计结论

静态文件管理的 ZipSlip 防护是亮点。主要改进点：增加 ZIP 炸弹防护（限制解压后大小和压缩比），确认符号链接处理策略，加固临时文件安全。
