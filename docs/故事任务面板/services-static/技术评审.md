# 静态文件管理 — 后端技术评审

## 架构设计

### ZIP 上传解压流程

```
upload_and_unzip(file, project_id)
    ├── 格式校验: .zip only
    ├── 大小校验: > MAX_ZIP_SIZE → reject
    ├── 确定目标目录: static/ 或 static/<project_id>/
    ├── 写入临时文件 (NamedTemporaryFile)
    ├── ZipFile 打开
    │   ├── _find_common_root(): 检测单一公共根目录
    │   └── _extract_zip_entries():
    │       ├── 遍历 infolist
    │       ├── _decode_filename(): UTF-8 / cp437
    │       ├── 剥离公共前缀 (strip_prefix)
    │       ├── _is_safe_path(): 拒绝 .. 和绝对路径
    │       └── 安全解压到目标目录
    └── finally: os.unlink(temp_zip_path)
```

### 路径安全模型

```
_is_safe_path(path, base_dir)
    ├── os.path.normpath(path)
    ├── 拒绝 ".." in path
    ├── 拒绝 path.startswith("/")
    ├── full_path = os.path.join(base_dir, path)
    └── full_path.startswith(base_dir) → True/False
```

### 公共根目录检测

```
_find_common_root(file_list, project_id)
    ├── 提取所有文件的第一级目录名
    ├── 去重 → first_level_dirs
    ├── len == 1 → 返回该目录 + "/"
    └── len != 1 → 返回 "" (不剥离)
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 解压方式 | 逐文件安全校验 | 防止 ZipSlip 攻击 |
| 临时文件 | NamedTemporaryFile + finally 清理 | 保证磁盘空间释放 |
| 编码兼容 | UTF-8 → cp437 回退 | 兼容 Windows/macOS 创建的 ZIP |
| 公共根目录 | 自动检测并剥离 | 用户体验优化 |
| Python 3.11+ | metadata_encoding='utf-8' | 解决中文文件名问题 |

## 数据流

```
UploadFile (.zip)
    ↓
读取全部字节 → 大小校验
    ↓
写入临时文件
    ↓
ZipFile 打开 → 遍历条目
    ↓
逐文件安全校验 → 解压到 static/
    ↓
清理临时文件 → 返回统计
```
