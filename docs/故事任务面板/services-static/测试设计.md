# 静态文件管理 — 测试用例评审

## 测试用例

### TC-SF-001：正常 ZIP 上传解压

| 项目 | 内容 |
|------|------|
| 目标 | 有效 ZIP 正确解压 |
| 前置 | 准备含 3 个文件的 test.zip |
| 步骤 | upload_and_unzip(file, project_id="test") |
| 预期 | static/test/ 下有 3 个文件，返回 extracted_files_count=3 |

### TC-SF-002：非 ZIP 文件拒绝

| 项目 | 内容 |
|------|------|
| 目标 | .txt 文件被拒绝 |
| 步骤 | 上传 filename="test.txt" 的文件 |
| 预期 | 抛出 "只支持ZIP格式文件" 异常 |

### TC-SF-003：超大 ZIP 拒绝

| 项目 | 内容 |
|------|------|
| 目标 | 超过限制的 ZIP 被拒绝 |
| 前置 | MAX_ZIP_SIZE = 1MB |
| 步骤 | 上传 2MB 的 ZIP |
| 预期 | 抛出大小超限异常 |

### TC-SF-004：路径遍历防护

| 项目 | 内容 |
|------|------|
| 目标 | 恶意 ZIP 中的 ../../../ 路径被拒绝 |
| 前置 | 构造含 `../etc/passwd` 条目的 ZIP |
| 步骤 | upload_and_unzip(file) |
| 预期 | 恶意条目被跳过，不写入磁盘 |

### TC-SF-005：公共根目录剥离

| 项目 | 内容 |
|------|------|
| 目标 | ZIP 内所有文件在单一目录下时自动剥离 |
| 前置 | ZIP 结构：`myapp/index.html`, `myapp/style.css` |
| 步骤 | upload_and_unzip(file, project_id="test") |
| 预期 | static/test/index.html, static/test/style.css（无 myapp/ 前缀） |

### TC-SF-006：无公共根目录不剥离

| 项目 | 内容 |
|------|------|
| 目标 | 多个顶级目录时不剥离 |
| 前置 | ZIP 结构：`dir1/a.txt`, `dir2/b.txt` |
| 步骤 | upload_and_unzip(file) |
| 预期 | 保留原始目录结构 |

### TC-SF-007：中文文件名

| 项目 | 内容 |
|------|------|
| 目标 | 中文文件名正确解码 |
| 前置 | ZIP 内含 `说明文档.txt` |
| 步骤 | upload_and_unzip(file) |
| 预期 | 文件正确解压，文件名不出现乱码 |

### TC-SF-008：本地解压

| 项目 | 内容 |
|------|------|
| 目标 | unzip_from_path 正常解压 |
| 前置 | /tmp/test.zip 存在 |
| 步骤 | unzip_from_path({"file_path": "/tmp/test.zip", "project_id": "demo"}) |
| 预期 | 文件解压到 static/demo/ |

### TC-SF-009：本地解压 — 文件不存在

| 项目 | 内容 |
|------|------|
| 目标 | 文件不存在时返回错误 |
| 步骤 | unzip_from_path({"file_path": "/tmp/nonexistent.zip"}) |
| 预期 | 抛出 ValueError: "File not found" |
