# OSS 存储服务 — 用户使用场景

## 场景 1：上传图片到 OSS

**参与者**：前端用户

**流程**：
1. 通过 `POST /upload-image-to-oss` 上传图片（base64 data URL）
2. 服务调用 `upload_bytes_to_oss(content, filename, directory)`
3. 校验文件扩展名是否在允许列表中
4. 校验文件大小
5. 生成时间戳文件名，上传到 OSS
6. 返回可访问的 HTTPS URL
7. 若 OSS 不可用，自动降级到本地存储

## 场景 2：管理文件标签

**参与者**：用户/管理员

**流程**：
1. `set_file_tags("images/photo.jpg", ["vacation", "2024", "beach"])`
2. 标签去重后保存到 MongoDB oss_file_tags 集合
3. `get_file_tags("images/photo.jpg")` → `["vacation", "2024", "beach"]`
4. `get_all_tags()` → `[{"name": "vacation", "count": 5}, {"name": "2024", "count": 12}, ...]`

## 场景 3：浏览文件列表

**参与者**：前端用户

**流程**：
1. `list_files(directory="images/2024/")`
2. 遍历 OSS 目录下的对象
3. 为每个文件附加标签和信息
4. 可选按标签过滤：`list_files(directory="images/", tags="vacation")`
5. 返回文件列表含：名称、大小、修改时间、URL、标签、标题、描述

## 场景 4：删除文件

**参与者**：用户/管理员

**流程**：
1. `delete_oss_file("images/old_photo.jpg")`
2. 检查文件是否存在
3. 删除 OSS 对象
4. 清理 MongoDB 中的标签记录
5. 清理 MongoDB 中的文件信息记录
