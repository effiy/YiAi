# OSS 存储服务 — 安全审计

## 审计范围

- `src/services/storage/oss_client.py` — OSS 客户端

## 安全发现

### [P1] OSS 凭证明文存储

- **发现**：`access_key_id`、`access_key_secret` 通过 config.yaml 明文配置
- **风险**：配置文件泄露直接导致 OSS 凭据泄露，攻击者可通过 OSS API 访问所有文件
- **建议**：使用环境变量或密钥管理服务（如阿里云 KMS）存储凭据

### [P1] URL 构建 — 虚拟主机风格风险

- **发现**：`build_oss_url` 用 endpoint 构造 URL，未验证 endpoint 格式
- **风险**：若 endpoint 被恶意修改，URL 可能指向攻击者服务器
- **建议**：校验 endpoint 为合法 OSS endpoint 格式

### [P2] 文件扩展名绕过

- **发现**：白名单基于扩展名字符串匹配，但未检查文件实际内容（魔数）
- **风险**：攻击者可重命名 .exe 为 .jpg 绕过上传限制
- **建议**：增加文件魔数检测，确认扩展名与实际内容类型一致

### [P2] 对象名注入

- **发现**：`object_name` 由 directory + timestamp + ext 拼接，directory 来自用户输入后经过滤
- **验证**：directory 仅使用 `_normalize_no_spaces` 去除空格
- **风险**：如果攻击者控制 directory，可能构造路径穿越的对象名
- **建议**：对 directory 做路径安全校验

### [INFO] OSS 降级安全

- OSS 不可用时降级到本地存储（upload.py:147）
- 降级时文件存储在 static 目录，需确保本地存储有同等的扩展名和大小限制

## 合规检查

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 凭据管理 | 风险 | config.yaml 明文存储 |
| 文件类型校验 | 部分 | 扩展名白名单，缺魔数检测 |
| 文件大小限制 | 通过 | 可配置限制 |
| 路径安全 | 部分 | directory 仅去空格 |
| 访问控制 | 通过 | OSS Bucket 策略控制 |

## 审计结论

OSS 存储服务的主要安全风险是凭证明文存储（P1）。建议将 access_key/secret_key 迁移到环境变量或密钥管理服务。文件类型校验建议增加魔数检测以加固。
