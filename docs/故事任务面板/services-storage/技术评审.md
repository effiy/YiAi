# OSS 存储服务 — 后端技术评审

## 架构设计

### OSS 集成架构

```
OSSConfig (配置加载)
    ↓
oss2.Auth(access_key_id, access_key_secret)
    ↓
oss2.Bucket(auth, endpoint, bucket_name)
    ↓
业务操作:
├── bucket.put_object(object_name, content)
├── bucket.delete_object(object_name)
├── bucket.object_exists(object_name)
└── oss2.ObjectIterator(bucket, prefix)
```

### 标签与信息存储

标签和文件信息存储在 MongoDB（而非 OSS 元数据），理由：
- 灵活查询（聚合计数、模糊搜索）
- 不受 OSS 元数据 API 限制
- 与项目统一的 MongoDB 数据层一致

```
oss_file_tags 集合: {object_name, tags[], updatedTime}
oss_file_info 集合: {object_name, title, description, createdTime, updatedTime}
```

### URL 构建

```
build_oss_url(bucket, endpoint, object_key)
    ↓
clean_endpoint = endpoint.replace('http://', '').replace('https://', '')
    ↓
f"https://{bucket}.{clean_endpoint}/{object_key}"
```

### 文件上传安全流程

```
upload_bytes_to_oss(content, filename, directory)
    ├── 提取扩展名 → ALLOWED_EXTENSIONS 白名单校验
    ├── 大小校验 → max_file_size
    ├── 生成 object_name: {directory/}timestamp{ext}
    ├── bucket.put_object(object_name, content)
    └── 返回 {url, filename, object_name}
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| OSS SDK | oss2 (aliyun-oss-python-sdk) | 阿里云官方 SDK |
| 标签存储 | MongoDB | 灵活查询，聚合方便 |
| 文件命名 | 时间戳 + 原始扩展名 | 避免冲突，保留类型信息 |
| 扩展名白名单 | 可配置 | 灵活控制可上传文件类型 |
| URL 格式 | HTTPS 虚拟主机风格 | 标准 OSS URL 格式 |

## 数据流

```
upload_bytes_to_oss(content, filename, directory)
    ↓
OSSConfig() → get_bucket()
    ↓
校验扩展名 + 大小
    ↓
生成 object_name → bucket.put_object()
    ↓
build_oss_url() → 返回可访问 URL
    ↓
(可选) 创建 MongoDB 标签/信息记录
```
