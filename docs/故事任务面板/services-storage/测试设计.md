# OSS 存储服务 — 测试用例评审

## 测试用例

### TC-OS-001：上传文件到 OSS

| 项目 | 内容 |
|------|------|
| 目标 | 文件正确上传并返回 URL |
| 前置 | OSS 配置完整 |
| 步骤 | upload_bytes_to_oss(b"test", "test.png", "images") |
| 预期 | 返回 url 以 https:// 开头，含 filename 和 object_name |

### TC-OS-002：不支持的文件类型

| 项目 | 内容 |
|------|------|
| 目标 | .exe 文件被拒绝 |
| 步骤 | upload_bytes_to_oss(b"test", "malware.exe") |
| 预期 | 抛出 "Unsupported file type" 异常 |

### TC-OS-003：超大文件拒绝

| 项目 | 内容 |
|------|------|
| 目标 | 超过限制的文件被拒绝 |
| 前置 | max_file_size = 1MB |
| 步骤 | upload_bytes_to_oss(b"x" * 2MB, "large.png") |
| 预期 | 抛出 "File too large" 异常 |

### TC-OS-004：OSS 配置不完整

| 项目 | 内容 |
|------|------|
| 目标 | 配置不完整时抛出异常 |
| 前置 | OSS 配置为空 |
| 步骤 | get_bucket(OSSConfig()) |
| 预期 | 抛出 RuntimeError |

### TC-OS-005：设置文件标签

| 项目 | 内容 |
|------|------|
| 目标 | 标签正确设置并去重 |
| 步骤 | set_file_tags("test.png", ["a", "b", "a"]) |
| 预期 | 标签为 ["a", "b"]（去重后） |

### TC-OS-006：获取文件标签

| 项目 | 内容 |
|------|------|
| 目标 | 获取已设置的标签 |
| 前置 | 已设置标签 ["tag1", "tag2"] |
| 步骤 | get_file_tags("test.png") |
| 预期 | 返回 ["tag1", "tag2"] |

### TC-OS-007：删除文件标签

| 项目 | 内容 |
|------|------|
| 目标 | 标签正确删除 |
| 前置 | 已设置标签 |
| 步骤 | delete_file_tags("test.png") → get_file_tags("test.png") |
| 预期 | 删除返回 True，再次获取返回 [] |

### TC-OS-008：聚合标签统计

| 项目 | 内容 |
|------|------|
| 目标 | get_all_tags 正确聚合计数 |
| 前置 | 文件 A: [tag1, tag2], 文件 B: [tag1] |
| 步骤 | get_all_tags() |
| 预期 | [{"name": "tag1", "count": 2}, {"name": "tag2", "count": 1}] |

### TC-OS-009：更新文件信息

| 项目 | 内容 |
|------|------|
| 目标 | 标题和描述正确更新 |
| 步骤 | update_file_info("test.png", "My Photo", "A beautiful sunset") → get_file_info("test.png") |
| 预期 | title="My Photo", description="A beautiful sunset" |

### TC-OS-010：列出目录文件

| 项目 | 内容 |
|------|------|
| 目标 | 列出指定目录下的文件 |
| 前置 | OSS 目录 images/ 下有文件 |
| 步骤 | list_files(directory="images") |
| 预期 | 返回文件列表，每个文件含 name、size、url、tags 等 |
