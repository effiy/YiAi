# 项目结构规范

本文档定义 YiAi 项目的目录结构规范。

---

## 整体结构

```
YiAi/
├── main.py                    # 兼容性入口文件
├── config.yaml                # 主配置文件
├── requirements.txt           # Python 依赖
├── CLAUDE.md                  # Claude Code 项目说明
├── .gitignore                 # Git 忽略文件
│
├── docs/                      # 文档目录
│   ├── README.md             # 项目文档首页
│   ├── 项目结构.md           # 项目结构规范
│   ├── 核心功能/             # 核心功能文档
│   ├── 开发规范/             # 开发规范文档
│   └── 复盘管理/             # 复盘管理文档
│
├── logs/                      # 日志目录
│
├── tests/                     # 测试目录（如需要）
│
└── src/                       # 源代码目录
    ├── main.py               # FastAPI 应用入口
    ├── api/                  # API 层
    ├── core/                 # 核心基础设施
    ├── models/               # 数据模型
    └── services/             # 业务逻辑层
```

---

## 源代码目录详解

### API 层 (src/api/)

```
src/api/
└── routes/
    ├── __init__.py
    ├── execution.py      # 动态模块执行端点
    ├── upload.py         # 文件上传端点
    ├── wework.py         # 企业微信集成端点
    └── maintenance.py    # 维护端点
```

**职责：**
- 处理 HTTP 请求和响应
- 参数验证
- 调用业务逻辑层
- 返回统一格式的响应

### 核心基础设施层 (src/core/)

```
src/core/
├── config.py           # 配置管理
├── database.py         # MongoDB 单例
├── logger.py           # 日志设置
├── exceptions.py       # 自定义异常
├── error_codes.py      # 错误码定义
├── response.py         # 统一响应格式
├── middleware.py       # 认证中间件
└── exception_handler.py # 全局异常处理器
```

**职责：**
- 提供配置管理
- 数据库连接管理
- 日志系统
- 异常处理
- 中间件

### 数据模型层 (src/models/)

```
src/models/
└── schemas.py          # Pydantic 模型定义
```

**职责：**
- 定义请求/响应数据模型
- 数据验证

### 业务逻辑层 (src/services/)

```
src/services/
├── execution/
│   └── executor.py     # 模块执行器
├── rss/
│   ├── scheduler.py    # RSS 调度器
│   └── parser.py       # RSS 解析器
├── ai/
│   └── ollama_client.py # Ollama 客户端
├── storage/
│   └── oss_client.py   # OSS 存储客户端
├── static/
│   └── file_manager.py # 文件管理器
└── database/
    └── crud.py         # 通用 CRUD 操作
```

**职责：**
- 实现核心业务逻辑
- 调用外部服务
- 数据处理和转换

---

## 新增模块规范

### 添加新 API 端点

1. 在 `src/api/routes/` 下创建新文件
2. 定义 APIRouter
3. 在 `src/api/routes/__init__.py` 中导出
4. 在 `src/main.py` 中注册路由

### 添加新服务

1. 在 `src/services/` 下创建新目录
2. 实现业务逻辑
3. 提供清晰的公共接口
